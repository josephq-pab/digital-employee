## 四、原型运行方式

原型**不是**网站或 Web 服务，而是命令行 Python 脚本。

解压后，在本地运行：

```bash
cd 数字员工原型完整包/project
python3 scripts/run_prototype.py
```

**运行要求：**
- Python 3.6+
- 无需安装额外依赖（原型使用 Python 内置库）

## 五、"让 Agent 访问服务器"是什么意思

OpenClaw 是一个运行在这台服务器（`47.112.211.98`）上的 AI Agent 框架。

如果有人说"让我的 Agent 去这台服务器"：

| 他们的意思 | 实际说明 |
|-----------|---------|
| 让另一个 Agent 也来运行原型 | 同一台服务器上可以开多个 OpenClaw session |
| 让 Agent 自动化运行原型 | 可以通过 cron job 让 agent 定期运行 `run_prototype.py` |
| 让外部 Agent 调用原型 API | 原型是脚本，不是 API 服务 |

**当前 Agent 访问方式：** 通过这个 OpenClaw 对话界面，直接发送指令即可。

## 六、MVP 反馈表单与原型的区别

| 服务 | 地址 | 说明 |
|------|------|------|
| MVP 反馈表单（Web 页面） | `http://47.112.211.98:3000/` | 页面化 MVP，用户可提交试用反馈 |
| 原型下载包 | `http://47.112.211.98:3000/prototype_package.zip` | Python 原型完整代码包 |

两者运行在不同层级：
- 反馈表单是**已经搭建好的 Web 页面**（可在线访问）
- 原型是**待运行的 Python 脚本**（下载后本地运行）

## 七、如果下载后运行遇到问题

| 问题 | 解决方法 |
|------|---------|
| `python3: command not found` | 安装 Python 3.6+：`sudo apt install python3` |
| 找不到样本文件 | 确认解压后目录结构完整，脚本内使用绝对路径 |
| 7/7 不全 PASS | 检查 Python 版本是否为 3.6+ |
| 想要修改原型 | 必须在 c1e85bc 基线上修改，修改后重新验证 7/7 |
