## 二、各目录详解

### 2.1 代码目录（src/）

**路径：** `project/src/`

| 子目录 | 内容 | 接手者需要做什么 |
|--------|------|----------------|
| adapters/ | normalizer.py（归一化） | 了解即可，无需修改 |
| generators/ | 领导摘要/台账/风险清单生成器 | 了解即可，无需修改 |
| loaders/ | 周报/台账/映射加载器 | 了解即可 |
| rules/ | 规则文件（当前未激活） | 未来扩展用 |

### 2.2 输入目录（samples/）

**路径：** `samples/samples/`

| 子目录 | 文件 | 说明 |
|--------|------|------|
| weekly_reports/ | weekly_report_series_latest3.json | 3期周报样本 |
| project_ledger/ | project_ledger_latest3.json | 台账样本 |
| mapping/ | mapping_latest3.json | 映射关系 |

**注意：** 这是固定样本，不是真实业务数据。

### 2.3 输出目录（project/data/output/）

**路径：** `project/data/output/`

| 文件格式 | 命名规则 | 说明 |
|---------|---------|------|
| 领导摘要 | `领导摘要_YYYYMMDD_HHMMSS.txt` | 每次运行生成新文件 |
| 事项总台账 | `事项总台账_YYYYMMDD_HHMMSS.txt` | 每次运行生成新文件 |
| 风险清单 | `风险清单_YYYYMMDD_HHMMSS.txt` | 每次运行生成新文件 |

### 2.4 中间结果目录（project/data/tmp/）

**路径：** `project/data/tmp/`

| 文件格式 | 命名规则 | 说明 |
|---------|---------|------|
| 归一化结果 | `normalized_YYYYMMDD_HHMMSS.json` | 可选查看，不影响输出 |

### 2.5 文档目录

**关键文档（必读）：**

| 文档 | 路径 | 用途 |
|------|------|------|
| RUNBOOK.md | `/` | 操作手册（运行/测试/故障排查） |
| 新Agent开工清单 | `project/docs/新Agent开工清单.md` | 新接手第一份必读 |
| 稳定基线版本统一说明 | `docs/稳定基线版本统一说明.md` | 基线版本定义 |

**完整文档索引：**
```
project/docs/docs总索引与阅读路径.md
```

### 2.6 日志/问题记录目录

**日志文件位置：**

| 内容 | 路径 | 说明 |
|------|------|------|
| 内存记录 | `memory/*.md` | 按日期记录 agent 工作历史 |
| GitHub 问题 | `docs/GITHUB_SYNC_ISSUE.md` | 远程同步问题留痕 |

## 三、快速定位指南

| 想做的事 | 去哪里 |
|---------|-------|
| 运行原型 | `python3 project/scripts/run_prototype.py` |
| 查看最新输出 | `ls -t project/data/output/*.txt \| head -3` |
| 了解原型能力 | `RUNBOOK.md` |
| 新接手从哪里开始 | `project/docs/新Agent开工清单.md` |
| 确认当前稳定基线版本 | `git log --oneline -1` |
| 了解项目文档全貌 | `project/docs/docs总索引与阅读路径.md` |
