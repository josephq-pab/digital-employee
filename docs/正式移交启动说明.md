## 五、基线版本不得擅自漂移

### 5.1 版本漂移定义

以下情况属于版本漂移，**禁止擅自执行**：

| 漂移行为 | 说明 |
|---------|------|
| 切换到非 c1e85bc 的 commit | 禁止 `git checkout` 到其他版本 |
| 修改代码未重新验证 7/7 | 禁止在未验证的情况下认为代码正常 |
| 修改输入样本文件 | 禁止修改 samples/ 下的 JSON 文件 |
| 修改规则/术语文件 | 禁止修改已冻结的判断规则 |
| 把样本说成真实数据 | 禁止在任何文档/对话中混淆 |

### 5.2 版本确认机制

每次接手或切换分支后，**必须**先执行：

```bash
git log --oneline -1
# 确认输出为 c1e85bc 或本项目的正式后续版本

python3 scripts/run_prototype.py
# 确认 7/7 PASS，方可开始工作
```

### 5.3 如果需要修改代码

```
Step 1：在 c1e85bc 基线上修改
Step 2：修改后重新运行 run_prototype.py 确认 7/7 仍 PASS
Step 3：如果通过，新版本从 c1e85bc 出发，记录为 c1e85bc-xxx
Step 4：如果失败，恢复到 c1e85bc，重新修改
Step 5：禁止在未验证的情况下直接替换原型代码
```
