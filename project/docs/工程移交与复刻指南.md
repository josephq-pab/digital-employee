# 工程移交与复刻指南

**版本：** v1.0
**这是本项目最关键的文档之一。**

**目的：** 确保任何新Agent接手时，不需要依赖"当前Agent知道但文档没写"的隐性知识，仅通过阅读docs目录即可完整继续推进项目。

---

## 一、新Agent接手流程（强制顺序）

### 第一步：读《一期原型总览》

**目的：** 理解项目全貌（做什么、不做什么、为什么）

**读完应知道：**
- 一期核心问题是什么
- 为什么这样做（一期只支持人工填报的原因）
- 完整链路是什么
- 项目边界在哪里

### 第二步：读《一期术语与判断口径》

**目的：** 掌握所有业务口径基准

**读完应知道：**
- 什么叫"事项"、"进展"、"完成"、"延期"、"无明确进展"、"需领导介入"
- 每个术语的判定标准是什么
- 哪些由系统判断，哪些必须人工确认

### 第三步：读《输入输出数据契约》

**目的：** 掌握数据结构

**读完应知道：**
- 输入字段有哪些，哪些必填
- 支持哪些格式（周报/月报、台账）
- 输出JSON结构是什么
- 三种输出（领导摘要/事项台账/风险清单）的区别

### 第四步：读《判断规则与状态机说明》

**目的：** 掌握核心判断逻辑

**读完应知道：**
- 规则优先级顺序（为什么优先级高必须先匹配）
- 5条核心规则分别是什么
- 状态机状态转换关系
- 哪些判断必须人工确认

### 第五步：读《原型模块拆解与目录设计》

**目的：** 掌握代码实现单元

**读完应知道：**
- 有哪几个模块，每个模块输入输出是什么
- 目录结构是什么
- 模块之间的依赖关系
- 哪些模块是阶段4必须实现，哪些是阶段5扩展

### 第六步：读《样本与测试用例规范》

**目的：** 掌握测试要求

**读完应知道：**
- 样本命名规范是什么
- 如何脱敏真实数据
- 冒烟测试有几条，必须达到什么标准
- 扩展测试目标是多少条

### 第七步：读《阶段推进总表》

**目的：** 掌握当前阶段门控状态和全部阶段推进条件（唯一真相源）

**读完应知道：**
- 当前在哪个阶段，阶段门控状态如何
- 哪些条件已满足，哪些未满足
- 哪些事项在阻断后续阶段

**重要：** 阶段推进条件以本文档为准，不以各阶段进展报告中的描述为准。

### 第八步：读《决策记录与待确认事项》

**目的：** 掌握已知问题和未决事项

**读完应知道：**
- 已做出的关键设计决策有哪些
- 当前仍有哪些未决事项
- 哪些事项阻断后续阶段

### 第八步：读《工程移交与复刻指南》（即本文档）

**目的：** 掌握如何继续工作

## 三、可调整内容（但需走变更流程）

| 编号 | 内容 | 调整条件 |
|------|------|---------|
| R01 | 新增术语或修改术语口径 | 书面确认，同步更新EVAL_RULES.md |

**调整流程：**
1. 提出变更申请（写清变更内容、原因、影响范围）
2. 确认影响：是否需要修改测试用例、数据契约、规则说明
3. 相关文档同步更新
4. 记录到《决策记录与待确认事项》的变更日志

## 五、如何在另一个工作区复刻本项目

### 5.1 最小复刻步骤

```bash
# 1. 创建新的workspace
mkdir new-workspace
cd new-workspace

# 2. 复制整个project目录
cp -r original-workspace/project ./project

# 3. 复制docs目录到工作区根
cp -r original-workspace/project/docs ./docs

# 4. 复制scripts目录
cp -r original-workspace/scripts ./scripts

# 5. 验证目录结构
find . -name "*.md" | grep -v __pycache__ | sort
```

### 5.2 复刻后必须验证的事项

| 检查项 | 验证方式 |
|--------|---------|
| 冒烟测试可通过 | bash scripts/run_smoke.sh |
| 演示可运行 | bash scripts/run_dev.sh |
| 文档完整性 | docs目录中11份文档均存在 |
| 无绝对路径引用 | 代码中无硬编码路径 |

### 5.3 复刻后必须更新的配置

- 项目根目录下的工作区ID（如有）
- 脚本中的绝对路径（如PROJECT_DIR）
- 任何外部服务地址（如有）

## 六、如何在不依赖会话记忆的情况下继续工作

### 6.1 每次会话开始前必须检查

1. 读SOUL.md和AGENTS.md（身份约束）
2. 读USER.md（用户偏好）
3. 读阶段进展报告（了解当前阶段）
4. 读《决策记录与待确认事项》（了解卡点）

### 6.2 当前会话上下文不包含的内容

以下内容**必须从文档获取**，不得假设会话中提到过：

- 术语口径定义（见《一期术语与判断口径》）
- 输入输出字段定义（见《输入输出数据契约》）
- 规则优先级（见《判断规则与状态机说明》）
- 模块划分（见《原型模块拆解与目录设计》）
- 测试用例要求（见《样本与测试用例规范》）
- 当前阶段门控（见《阶段1文档一致性修正记录》）

### 6.3 发现文档与实际不符时

- 立即停止执行（不要按错误文档继续）
- 在《决策记录与待确认事项》中记录差异
- 提出修正申请
- 等修正完成后继续

## 七、Agent工作边界检查清单

每次执行前，Agent必须自检：

- [ ] 是否在处理数字员工项目（而非AI雷达站）
- [ ] 是否在按阶段推进（未跳阶段）
- [ ] 是否在写文档而非写代码（本阶段）
- [ ] 判断结论是否附了依据
- [ ] 是否依赖了会话上下文中的隐性知识（不应依赖）
- [ ] 是否有文档未更新就引入了新内容
