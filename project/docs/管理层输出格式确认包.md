# 管理层输出格式确认包

**版本：** v1.0
**发出方：** 数字员工项目组
**接收方：** 管理层审阅人
**用途：** 请管理层确认"领导摘要"的格式和内容是否满足使用需求

---

## 一、确认包概述

本包用于请求管理层对"领导摘要"输出格式进行确认。
确认后，《输入输出数据契约》中的领导摘要格式将被正式冻结。

---

## 二、领导摘要固定结构

### 2.1 模板A（推荐模板）

```
【智慧园区一期 · 张明 · 2026年4月第1周】

■ 综合状态：关注（风险等级：中）

■ 进展摘要：
基础平台部署完成80%，用户权限模块开发完成，数据看板开发滞后约2周。

■ 完成度判断：
承诺6月30日前交付，存在无法按时完成的风险。

■ 风险定位：
主要风险为数据源接口对接进度不及预期，可能影响看板按时上线。

■ 建议动作：
1. 分析滞后根因，制定缓释措施；
2. 对风险自述进行深入分析，制定缓释措施。

■ 是否升级：否（系统建议），最终由人工判断。

■ 判断依据（供复核）：
- 触发"关注"风险：进度描述含"滞后"，且有风险自述
- 原始关键词：数据看板开发滞后约2周
```

### 2.2 模板B（备选模板，更简洁）

```
【智慧园区一期】状态：关注（风险：中）
进展：基础平台80%，权限模块完成，看板滞后2周。
风险：接口对接不及预期，可能影响上线。
建议：分析根因，制定缓释措施。
升级建议：暂不需要升级。
```

## 三、字段必须项 vs 可选项

### 必须项（不可删除）

| 字段路径 | 说明 |
|---------|------|
| project_name | 项目名称（透传） |
| owner | 责任人（透传） |
| status_summary.overall | 总体状态：正常/关注/危险 |
| status_summary.risk_level | 风险等级：低/中/高 |
| status_summary.progress_text | 进展摘要（≤200字） |
| risk_assessment.level | 风险等级（与overall对应） |
| risk_assessment.summary | 风险摘要（一句话） |
| suggested_actions | 建议动作列表 |
| escalation_recommended | 是否建议升级（true/false） |
| reasoning.raw_signal | 判断原始关键词（可人工复核） |

### 可选项（可删除或简化）

| 字段路径 | 说明 |
|---------|------|
| report_id | UUID（系统自动生成） |
| generated_at | 时间戳（系统自动生成） |
| status_summary.completion_estimate | 完成度估算（可选项） |
| status_summary.no_clear_progress | 无明确进展标识（可选项） |
| risk_assessment.triggered_by | 触发的风险条件列表（可选项） |
| risk_assessment.risk_points_present | 是否有风险自述（可选项） |
| escalation_indicator | 升级来源：system_suggested/user_requested（可选项） |
| reasoning.facts_extracted | 事实清单（可选项） |
| reasoning.rules_applied | 规则名称列表（可选项） |

---

## 四、确认后会冻结什么

管理层确认本格式后，以下内容将被正式冻结：

| 冻结项 | 冻结文件 | 冻结后果 |
|--------|---------|---------|
| 领导摘要JSON结构 | 输出层冻结 | 后续实现不得删减必须项字段 |
| 字段必须项/可选项定义 | 输出层冻结 | 实现可删减可选项，不得删减必须项 |
| 文本摘要模板 | 输出层冻结 | 后续实现不得改变必须项的表述逻辑 |

---

## 五、未确认会阻断什么

若管理层未给出确认或提出修改意见：

| 阻断项 | 影响 |
|--------|------|
| 阶段2无法正式关闭 | 影响后续阶段推进 |
| 输出层设计无法冻结 | 阶段4开发缺乏稳定基准 |
| 演示版无法正式制作 | 影响阶段5进入 |
