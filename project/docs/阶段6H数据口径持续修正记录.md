## 四、数据口径表述规范

所有文档和页面中，数据问题的表述必须统一：

| ✅ 正确 | ❌ 错误 |
|---------|--------|
| 样本数据待更新 | 系统错误 |
| 数据缺失 | 数字不对 |
| 样本问题 | Bug |
| 口径偏差 | 系统坏了 |
| 待验证 | 错的 |

---

## 五、数据问题处理SOP

1. **发现** → 立即登记到本表
2. **分类** → 判断是代码问题还是样本数据问题
3. **影响评估** → 确定影响范围和优先级
4. **处理方案** → 显式标注（样本问题）或修复代码（代码问题）
5. **验证** → 确认修复后无新问题
6. **关闭判断** → 满足关闭条件后标记"✅ 已关闭"
