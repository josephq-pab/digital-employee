## 一、当前样本接入方式

### 1.3 映射匹配逻辑（已有fuzzy匹配）

```python
def find_ledger_record(ledger_index, project_name):
    """在台账中模糊匹配项目名称"""
    name_lower = project_name.lower()
    for rid, rec in ledger_index.items():
        if name_lower in rec['project_name'].lower():
            return rec
    return None
```

**当前方式总结：** 文件系统加载（本地JSON文件），项目名称fuzzy匹配映射

## 二、真实数据接入方式可选项

### 方案A：文件替换（推荐，最简）

**方式：** 将真实数据JSON文件替换到 `samples/samples/` 目录

| 字段 | 内容 |
|------|------|
| **实现成本** | 极低（仅替换文件路径或文件名不变直接替换内容） |
| **代码改动** | 零改动或极少量改动（loader路径不变） |
| **适用场景** | 业务方可定期导出JSON文件并替换 |
| **优点** | 最简单，最快可用，零技术风险 |
| **缺点** | 需要人工定期替换，不适合高频更新 |

**操作步骤：**
1. 业务方将周报数据导出为指定JSON格式
2. 替换 `samples/samples/weekly_reports/weekly_report_series_latest3.json`
3. 替换 `samples/samples/project_ledger/project_ledger_latest3.json`
4. 运行 `python3 scripts/run_prototype.py` 验证

### 方案B：目录分离（推荐，隔离）

**方式：** 保持samples目录不变，新建 `realtime/` 目录存放真实数据

| 字段 | 内容 |
|------|------|
| **实现成本** | 低（新增loader路径配置） |
| **代码改动** | 在run_prototype.py中增加路径选择逻辑 |
| **适用场景** | 需要同时保留样本数据用于对比 |
| **优点** | 样本和真实数据隔离，可切换对比 |
| **缺点** | 需要配置切换逻辑 |

**操作步骤：**
1. 新建 `samples/realtime/` 目录结构
2. 配置环境变量或命令行参数选择使用 samples 或 realtime 路径
3. 运行验证

### 方案C：API对接（中期方案）

**方式：** 通过API实时获取周报和台账数据

| 字段 | 内容 |
|------|------|
| **实现成本** | 高（需开发API适配器） |
| **代码改动** | 新增API loader，修改run_prototype.py |
| **适用场景** | 有稳定的内部系统API可用 |
| **优点** | 数据实时，无需人工操作 |
| **缺点** | 需要API开发，不适合当前阶段 |

**前置条件：**
- 内部系统有可用的API
- API返回格式与当前JSON格式兼容或可转换

## 三、建议采用的接入方式

**当前阶段建议：方案A（文件替换）**

理由：
1. **成本最低**：零代码改动，只需替换文件
2. **风险最小**：不影响现有稳定运行的系统
3. **最快可用**：业务方今天就可以导出文件并替换
4. **验证最简**：直接运行冒烟测试验证

**为什么不是方案C（API对接）：**
- 当前阶段是"准备阶段"，不是"开发阶段"
- API开发成本高，不适合作为准备阶段的首要任务
- B3的最终验证只需要有数据可跑，不依赖API

## 四、文件替换方式的操作流程

### 步骤1：确认JSON格式

业务方导出的JSON必须与当前样本格式兼容：

**周报JSON必要字段：**
```json
{
  "report_meta": {
    "report_period": "0414-0418",
    "report_date": "2026-04-14"
  },
  "project_overview": {
    "total_items": 11,
    "status_light": "green"
  },
  "items": [
    {
      "item_id": "P001",
      "project_name": "XX项目",
      "content": "事项内容",
      "status": "进行中"
    }
  ]
}
```

**台账JSON必要字段：**
```json
{
  "records": [
    {
      "record_id": "R001",
      "project_name": "XX项目",
      "阶段": "二阶段",
      "status": "green"
    }
  ]
}
```

### 步骤2：替换文件

```bash
# 备份当前样本
cp -r samples/samples samples/samples_backup

# 替换周报文件（文件名不变）
cp 真实周报.json samples/samples/weekly_reports/weekly_report_series_latest3.json

# 替换台账文件（文件名不变）
cp 真实台账.json samples/samples/project_ledger/project_ledger_latest3.json
```

### 步骤3：验证

```bash
cd project
python3 scripts/run_prototype.py
```
