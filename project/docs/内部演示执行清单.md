## 一、演示前（执行人：项目总控Agent）

| # | 动作 | 负责方 | 完成标准 | 状态 |
|---|------|--------|---------|------|
| P1 | 确认演示时间 | 项目总控Agent | 管理层已确认具体时间 | ☐ |
| P2 | 确认参会人名单 | 项目总控Agent | 业务方代表+管理层代表已确认 | ☐ |
| P3 | 发送演示执行页给参会人 | 项目总控Agent | 参会人收到《内部演示执行页.md》 | ☐ |
| P4 | 发送会前提醒 | 项目总控Agent | 演示前24h发出《内部演示会前提醒_正式版.md》 | ☐ |
| P5 | 发送边界声明 | 项目总控Agent | 演示前告知"这是历史样本预览，不是生产系统" | ☐ |
| P6 | 准备演示设备 | 项目总控Agent | 投影/投屏/浏览器测试完毕 | ☐ |
| P7 | 打印/准备结论模板纸质版 | 项目总控Agent | 演示主持手边有《内部演示会后结论模板.md》 | ☐ |

**演示前检查完成标准**：P1-P7全部打勾，才能执行演示。

## 二、演示中（执行人：演示主持）

| # | 时间 | 动作 | 完成标准 | 状态 |
|---|------|------|---------|------|
| D1 | 0-3min | 开场+背景说明 | 参会人知道项目当前阶段 | ☐ |
| D2 | 3-10min | 领导摘要Tab讲解 | 参会人理解整体判断逻辑 | ☐ |
| D3 | 10-16min | 事项总台账Tab讲解 | 参会人理解逐项复核逻辑 | ☐ |
| D4 | 16-21min | 风险清单Tab讲解 | 参会人理解黄灯判断规则 | ☐ |
| D5 | 21-26min | 样本与结论对照Tab讲解 | 参会人理解规则可解释性 | ☐ |
| D6 | 26-29min | 路线图+决策选项讲解 | 参会人理解3个分支走向 | ☐ |
| D7 | 29-30min | 收口三问 | 参会人对3个问题有明确口头意向 | ☐ |

**演示中检查完成标准**：D1-D7全部执行，3个问题有口头结论。

## 三、演示后（执行人：演示主持 + 项目总控Agent）

| # | 动作 | 负责方 | 完成标准 | 状态 |
|---|------|--------|---------|------|
| A1 | 填写结论模板 | 演示主持 | 3项结论全部填写，有决策人签字 | ☐ |
| A2 | 记录口头反馈 | 演示主持 | 反馈记录到《内部演示反馈登记表.md》 | ☐ |
| A3 | 确定分支走向 | 项目总控Agent | 根据3项结论确定分支A/B/C | ☐ |
| A4 | 发出分支确认通知 | 项目总控Agent | 相关方收到分支走向通知 | ☐ |
| A5 | 更新路线图（如涉及） | 项目总控Agent | 路线图反映最新分支状态 | ☐ |
| A6 | 归档演示材料 | 项目总控Agent | 演示结论模板+反馈记录存档 | ☐ |

**演示后检查完成标准**：A1-A6全部完成，演示才算正式收口。

---

## 四、演示执行约束

| 约束 | 内容 |
|------|------|
| 不提前假设结论 | 3项结论必须当场确认，不预设答案 |
| 不跳过边界声明 | 演示前必须发出边界声明 |
| 不跳过结论模板 | 演示后必须当场填写结论模板 |
| 不代替决策人决策 | 结论模板必须由决策人确认，不能由Agent代填 |
