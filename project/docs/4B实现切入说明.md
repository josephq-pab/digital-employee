## 三、4B开工第1-3步

### 第1步：确认包收口（确认完成的当天）

```
1. 打开《确认结果回填模板.md》
2. 登记所有确认结果
3. 更新《阶段4B进入门控卡.md》（G1/G2/G3 → 🟢）
4. 如G4/G5自动升级为🟢，确认门控总体变为🟢
5. 发出通知："确认链已闭合，G1-G7全部🟢，可进入4B"
```

### 第2步：工作区就绪最后确认（确认完成当天）

```
执行《4B开工前最后检查表.md》32项检查
优先检查：
  - engine.py 是否存在且未修改
  - smoke_tests.json 是否存在
  - 样本文件是否在指定目录
  - 文档是否已冻结（检查变更记录）
```

### 第3步：正式开工（确认完成第2天）

```
执行T01（样本装载与适配层）
  → 将周报JSON适配到INPUT_SCHEMA
  → 将台账JSON适配到INPUT_SCHEMA
  → 运行demo_case验证连通性

同时启动T02（适配层扩展）
  → 适配层处理ADPT-01~04的解决方案
  → 不修改规则引擎
```

## 八、风险预案

| 风险 | 预案 |
|------|------|
| 确认完成后engine.py被意外修改 | 重新执行smoke_tests.json全部用例 |
| 样本文件路径变化 | 以registry中的file字段为准 |
| 确认包反馈中出现"不同意" | 暂停进入4B，按《确认反馈合并规则》处理 |
| G4/G5未能自动升级为🟢 | 检查Q01是否真的完成，更新手控卡 |
