# 扩大评审结论待命清单

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4M《扩大评审执行与真实样本接入启动》

---

## 一、三种分支结论

| 分支 | 触发条件 | 综合结论 |
|------|---------|---------|
| 分支A | 评审认可或有条件认可（条件可满足）| 通过（附条件或不附条件）|
| 分支B | 评审认可，但要求先做真实影子运行 | 先影子运行，后扩大 |
| 分支C | 存在暂缓或拒绝意见 | 暂缓，维持当前范围 |

---

## 二、分支A：评审通过，进入扩大试点准备

**触发条件：** 评审方全部认可或有条件认可（条件可满足）

**最小动作集：**

| 动作 | 说明 | 执行人 |
|------|------|--------|
| A1 | 记录通过结论（FB-001结论标记"通过"）| 技术侧 |
| A2 | 确认扩大评审参与方名单 | 业务方 |
| A3 | 确认真实样本提供计划（时间/数量）| 业务方 |
| A4 | 确认人工确认流程和责任人 | 业务方 |
| A5 | 启动"真实周报样本接入"准备工作 | 技术侧 |
| A6 | 通知相关方进入扩大试点准备阶段 | 技术侧 |

---

## 三、分支B：评审通过，要求先做真实影子运行

**触发条件：** 评审方认可技术能力，但希望先用真实数据验证再扩大

**最小动作集：**

| 动作 | 说明 | 执行人 |
|------|------|--------|
| B1 | 记录结论（FB-001结论标记"先影子"）| 技术侧 |
| B2 | 业务方提供 ≥ 3期 真实周报样本 | 业务方 |
| B3 | 技术侧执行影子运行（按SOP）| 技术侧 |
| B4 | 业务方人工对照验证 | 业务方 |
| B5 | 评估差异率（按登记表结论栏）| 双方 |
| B6 | 差异率 ≤ 20% → 通知业务方，进入分支A |
| B7 | 差异率 > 20% → 记录 → 评估修正可行性 | 技术侧 |

---

## 四、分支C：评审暂缓，维持当前范围

**触发条件：** 评审方认为验证仍不充分，或有其他顾虑

**最小动作集：**

| 动作 | 说明 | 执行人 |
|------|------|--------|
| C1 | 记录暂缓结论（FB-001结论标记"暂缓"）| 技术侧 |
| C2 | 整理评审反馈中的具体顾虑点 | 技术侧 |
| C3 | 针对顾虑点制定补充验证计划 | 技术侧 |
| C4 | 补充验证执行（不等待真实样本，用现有历史数据）| 技术侧 |
| C5 | 补充验证完成后，重新申请评审 | 技术侧 |

---

## 五、三种分支的共同最小动作

无论哪种分支，均需执行：

| 动作 | 说明 |
|------|------|
| 记录 | 所有反馈正式登记在《扩大评审反馈登记表》 |
| 告知 | 结论通知所有相关方（PMO/管理层/技术侧）|
| 记录变更 | 更新本清单的"当前分支"状态 |
