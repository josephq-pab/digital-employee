## 二、输出命名规范

```
领导摘要_<YYYYMMDD_HHMMSS>.txt
事项总台账_<YYYYMMDD_HHMMSS>.txt
风险清单_<YYYYMMDD_HHMMSS>.txt
normalized_<YYYYMMDD_HHMMSS>.json
```

示例：`领导摘要_20260408_050528.txt`

## 三、输出验收标准

| # | 输出文件 | 验收项 | 通过条件 |
|---|---------|-------|---------|
| OUT-1 | 领导摘要 | 文件存在 | os.path.exists() = True |
| OUT-2 | 领导摘要 | 非空 | 文件大小 > 100 bytes |
| OUT-3 | 领导摘要 | 格式完整 | 包含"总事项数"关键字 |
| OUT-4 | 事项总台账 | 文件存在 | os.path.exists() = True |
| OUT-5 | 事项总台账 | 非空 | 文件大小 > 200 bytes |
| OUT-6 | 事项总台账 | 事项数 | ≥ 预期事项数 * 0.9 |
| OUT-7 | 风险清单 | 文件存在 | os.path.exists() = True |
| OUT-8 | 风险清单 | 非空 | 文件大小 > 100 bytes |
| OUT-9 | 风险清单 | 风险数 | ≥ 1 |
| OUT-10 | 归一化JSON | JSON语法正确 | json.load()不报错 |
| OUT-11 | 归一化JSON | 字段完整 | 包含normalized_items和engine_results |
