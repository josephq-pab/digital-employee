### 第1步：文件传输

**方式A：Git clone（推荐）**
```bash
cd ~/
git clone https://github.com/josephq-pab/digital-employee.git
```
> 克隆后文件在 `~/digital-employee/`

**方式B：打包传输**
```bash
# 在源服务器执行
cd /home/admin/.openclaw/workspace-digital-employee
tar -czvf /tmp/mvp-migration.tar.gz \
  project/docs/阶段6A_页面化MVP建设/pages/index.html \
  project/docs/mvp-feedback-form.html \
  project/data/slices/ \
  project/docs/阶段6F当前共识落版.md \
  project/docs/阶段6F迁移必读说明.md \
  project/docs/阶段6F数据口径修正清单.md \
  project/docs/W17战略客户数据待更新说明.md

# 传输到目标服务器
scp /tmp/mvp-migration.tar.gz user@目标服务器IP:/tmp/

# 在目标服务器解压
tar -xzvf /tmp/mvp-migration.tar.gz -C /目标路径/
```

### 第3步：启动服务

```bash
cd <迁移后pages目录>

# 启动HTTP服务
python3 -m http.server 3000 > /tmp/mvp3000.log 2>&1 &

# 验证启动
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:3000/
# 应返回 200
```

---

## 三、回滚方案

若迁移后出现问题：
1. 停止新服务：`kill $(cat /tmp/mvp3000.pid)`（或 `pkill -f http.server`）
2. 旧服务器继续运行，不关机
3. 修复后重新传输 index.html 覆盖即可
