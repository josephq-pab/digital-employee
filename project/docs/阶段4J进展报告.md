## 三、W3-P2 根因与修复

### 3.1 问题现象
1117-1121 request_lines 仅含标题行"一、一个项目申请延期，请您审批："，实际内容在 raw_text 下一行。适配器未实现标题-内容配对，导致"数字经营"延期申请完全丢失。

### 3.2 根因
`adapt_request_lines_to_closure_requests()` 只处理 request_lines 内容，未实现：
1. 识别"标题行"（正则与实际格式不匹配：`[审批批示]` 匹配单个汉字而非"审批"双字词）
2. 跳过标题行后的具体内容行（内容行单独处理）
3. 当 request_lines 仅标题时，从 raw_text 提取紧跟内容

### 3.3 修复方案

**修复1 - 标题识别正则修正：**
```python
# 错误：[审批批示] 只匹配1个汉字
r"^一.+?[审批批示][：:]?\s*$"
# 正确：(审批|批示) 匹配双字词
r"^一.+?(?:请您)?(审批|批示).{0,15}[：:]\s*$"
```

**修复2 - 标题行跳过逻辑：**
```python
skip_current = False
if is_header and i < len(request_lines):
    next_line = request_lines[i].strip()
    if next_line and (next_line[0].isdigit() or next_line.startswith(("1、", "2、", "3、"))):
        skip_current = True  # 跳过标题行，内容行单独处理
```

**修复3 - raw_text 配对提取：**
```python
if is_header and raw_text and not skip_current:
    m = re.search(r"审批[：:]\s*\n([^\n]{2,60})", raw_text)
    if m:
        line = m.group(1).strip()  # 从raw_text提取标题后紧跟行
```

**修复4 - 项目名引号清理：**
```python
pn = pn.replace('"').replace("'").strip()  # 去除ASCII引号
```

**修复文件：** `project/src/loaders/november_adapter.py`

### 3.4 验证结果
- 1117-1121 `数字经营` ✅ 正确识别为延期申请（从丢失变为正常识别）
- 项目名无残留引号：`"数字经营"规划推进` → `数字经营`

## 六、工程教训

### 教训1：heredoc 中 `\n` 被 Python 解释为真实换行符
**错误做法：** 通过 Python 字符串写入含 `\n` 的正则表达式
**后果：** `\n` 被解释为真实换行符，导致多行字符串截断，产生 unclosed string
**正确做法：** 用 `sed -i` 精确替换，或完整重写整个函数，避免部分写入

### 教训2：`[审批批示]` 是字符类，匹配单个汉字
**错误做法：** 正则 `[审批批示]` 只匹配1个汉字（"审"、"批"、"指"、"示"之一）
**正确做法：** 用 `(审批|批示)` 匹配"审批"或"批示"双字词组

### 教训3：同一文件反复打补丁无效时，应完整重写
**现象：** 第125行 triple-quote 问题导致文件语法错误，修复后再次写入又损坏
**根因：** 写入机制有 `\n` 转义问题，同一文件反复修复时每次都重新损坏
**正确做法：** 识别到机制问题后，用完整重写（heredoc 一次性写入整个函数）替代反复打补丁
