## 靶子说明

以下是现有HTML演示页中最核心的4个页面截图/内容说明。

每个页面下方标注了**"这一页要您判断什么"**。

---

## 页面1：首页/总览

**对应Tab**：Tab 0 — 首页总览

**核心内容**：
- 顶部声明：本页面为演示预览，非生产系统界面
- 项目阶段进度条（9步，已完成7步，当前在"演示实物化"）
- 本期样本说明（1208-1212期，某总行PMO条线）
- 4个快捷入口按钮：领导摘要 / 事项总台账 / 风险清单 / 样本与结论对照

**这一页要您判断什么**：

> 整体框架和定位说明是否清晰？参会人打开页面后是否知道自己在看什么？

---

## 页面2：领导摘要

**对应Tab**：Tab 1 — 领导摘要

**核心内容**：
- 三大数字看板：申请结项3项 / 黄灯警示2项 / 推进中18项
- 整体判断文字："本期以结项申请为主，黄灯持续偏长，建议从提示升级为节点约束"
- 结项事项列表（3项）：事项名称 / 主责部门 / 结项说明 / 判断标签
- 黄灯警示事项（2项）：学校行业（逾期近3个月）/ 汽车行业（未达小试目标）

**这一页要您判断什么**：

> 领导看到这页，能否在30秒内知道"本期整体盘面怎么样、谁需要关注什么"？
> 这套摘要结构和表达方式，是否达到您愿意给管理层看的水平？

---

## 页面3：事项总台账

**对应Tab**：Tab 2 — 事项总台账

**核心内容**：
- 标签栏：结项3项 / 推进中18项 / 黄灯警示2项 / 会议待办2项
- 完整表格：事项名称 / 类型 / 主责部门 / 计划节点 / 系统判断 / 风险 / 管理建议
- 每行记录右侧有颜色标签（绿色=结项，蓝色=推进中，黄色=黄灯）
- 共约20行记录，覆盖创新类、改革规划类、会议待办类等

**这一页要您判断什么**：

> PMO或业务方看到这页，会不会觉得"这就是我们自己的周报台账，只是被结构化了"？
> 还是说，看起来像另一个系统强加的格式，和实际工作脱节？

---

## 页面4：风险清单

**对应Tab**：Tab 3 — 风险清单

**核心内容**：
- 三格统计：黄灯2项 / 逾期1项（双链融合汇报）/ 结构性压力1项（并购贷款）
- 风险详情表（4行）：风险ID / 事项 / 风险类型 / 风险说明 / 判断依据 / 处置建议
- 黄灯历史追踪：展示学校行业和汽车行业过去4周的状态演化
- 每条风险都有明确的"原计划 vs 当前"对比

**这一页要您判断什么**：

> 黄灯和逾期事项是否被清晰识别出来，还是被淹没在大量正常事项中？
> 判断依据是否足够透明，让人相信"系统不是乱判断的"？
