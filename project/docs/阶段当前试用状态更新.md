## 五、D-01/D-02/D-03为何暂不先做

**原因一：无具体案例支撑**
D类决策涉及管理决策权威性认定，必须基于具体案例，不能基于假设讨论。

**原因二：优先收集真实反馈**
在真实试用前，D类问题的描述都是假设的。等业务方试用后，有具体分歧案例，再触发D类才有实际意义。

**原因三：符合"先小后大"原则**
先验证"反馈→迭代"短闭环是否跑通，再考虑需要D类决策才能解决的问题。

**触发条件（已固化）**：

| 信号 | 触发 |
|------|------|
| 多个反馈："不知道系统结论是怎么来的" | D-01 |
| 多个反馈："系统结论与人工判断不符" | D-02 |
| 具体案例："系统建议A，人工决定B" | D-03 |
