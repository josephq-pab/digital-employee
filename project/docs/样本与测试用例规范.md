# 样本与测试用例规范

**版本：** v1.0
**用途：** 定义一期所需的样本数据、测试用例集合及后续维护规范

---

## 一、样本数据规范

### 1.1 样本分类

| 类别 | 用途 | 存放位置 | 来源 |
|------|------|---------|------|
| 演示样例 | run_dev.sh演示 | data/samples/demo_case_01.json | 人工构造 |
| 开发验证样例 | 开发过程验证 | data/samples/dev_*.json | 人工构造/脱敏 |
| 真实样本 | 阶段4/5验证 | data/samples/real_*.json | PMO提供（脱敏后） |

### 1.2 样本命名规范

```
{类型}_{序号}_{格式类型}.{扩展名}

类型：
  demo    - 演示样例
  dev     - 开发验证样例
  real    - 真实样本（脱敏后）

序号：
  01, 02, 03...  两位数字自增

格式类型：
  weekly  - 周报/月报格式
  ledger  - 项目台账格式

示例：
  demo_01_weekly.json       # 演示样例，周报格式
  dev_01_weekly.json        # 开发验证样例，周报格式
  real_01_ledger.json       # 真实样本，台账格式
```

### 1.3 样本脱敏规范

**必须脱敏的字段：**
- 项目名称 → 使用"XX项目N"替代
- 责任人姓名 → 使用"张三"等通用姓名
- 日期 → 使用相对日期（如"报告日期：{基准日-7d}）
- 具体公司名、技术细节 → 通用化处理

**不得脱敏的内容（保持判断逻辑完整性）：**
- 承诺事项的描述结构
- 进度描述中的关键词（滞后、完成等）
- 风险自述中的关键词
- 字段长度和格式结构

### 1.4 样本质量要求

| 检查项 | 要求 |
|--------|------|
| 字段完整性 | 必填字段必须存在 |
| 格式正确性 | JSON格式合法，无解析错误 |
| 场景覆盖 | 至少覆盖5种核心场景（正常/滞后/风险/升级/严重） |
| 标签准确性 | 每个样本须标注预期输出（risk_level, escalation等） |
| 可运行性 | 样本须能通过InputParser，无报错 |

## 二、测试用例规范

### 2.1 测试用例分类

| 类别 | 用途 | 执行时机 |
|------|------|---------|
| 冒烟测试（Smoke Tests） | 核心规则验证 | 每次run_smoke.sh |
| 单元测试（Unit Tests） | 模块级验证 | 阶段4开发中 |
| 集成测试（Integration Tests） | 全链路验证 | 阶段5前 |

### 2.2 冒烟测试用例集（5条核心用例）

**文件位置：** `data/test_cases/smoke_tests.json`

| 用例ID | 场景 | 输入特征 | 预期risk_level | 预期escalation | 预期overall |
|--------|------|---------|---------------|----------------|-------------|
| TC01 | 正常无风险 | 无滞后词，risk_points空 | 低 | false | 正常 |
| TC02 | 滞后+风险自述 | 含"滞后"+风险自述 | 中 | false | 关注 |
| TC03 | 请求介入 | escalation_requested=true | 高 | true | 危险 |
| TC04 | 严重滞后 | 含"严重滞后"关键词 | 高 | true | 危险 |
| TC05 | 无风险正常进度 | 含"正常"类词，risk空 | 低 | false | 正常 |

**用例文件格式（smoke_tests.json）：**

```json
[
  {
    "id": "TC01",
    "description": "正常无风险",
    "input": { ... },
    "expected": {
      "risk_level": "低",
      "escalation_recommended": false,
      "overall": "正常"
    }
  }
]
```

### 2.3 扩展测试用例集（目标：20条）

| 用例ID | 场景 | 说明 |
|--------|------|------|
| TC06 | 空risk_points | risk_points="" |
| TC07 | escalation=false显式 | escalation_requested=false |
| TC08 | 进度停滞 | 含"停滞"/"卡住" |
| TC09 | 进度未完成 | 含"未完成" |
| TC10 | 风险自述但进度正常 | 有风险词但无滞后词 |
| TC11 | 滞后+无风险自述 | 有滞后词但risk_points空 |
| TC12 | 严重滞后+请求介入 | 两者同时存在 |
| TC13 | 纯完成描述 | 只有完成类关键词 |
| TC14 | 模糊进度描述 | "进行中"等 |
| TC15 | 很长文本 | progress超过500字 |
| TC16 | 格式B-台账格式 | items数组 |
| TC17 | 台账+有风险 | ledger+overall_risk_points |
| TC18 | 台账+escalation | ledger+escalation |
| TC19 | 无明确进展-空 | current_progress为空 |
| TC20 | 无明确进展-等 | 含"等"类无进展词 |

### 2.4 测试用例执行要求

**run_smoke.sh必须满足：**
- [ ] TC01-TC05 全部通过（冒烟门槛）
- [ ] TC06-TC20 全部通过（扩展门槛）
- [ ] 每个失败用例输出：实际值 vs 预期值
- [ ] 测试结果写入 eval/smoke_test_results.json（含时间戳）

## 三、测试样本存放规范

```
data/
├── samples/                      # 样本数据
│   ├── README.md                  # 样本命名与脱敏规范
│   ├── demo_01_weekly.json       # 演示样例（必须存在）
│   ├── dev_01_weekly.json        # 开发样例：正常
│   ├── dev_02_weekly.json        # 开发样例：滞后
│   ├── dev_03_weekly.json        # 开发样例：严重滞后
│   ├── dev_04_weekly.json        # 开发样例：风险自述
│   ├── dev_05_weekly.json        # 开发样例：空risk_points
│   ├── real_01_weekly.json       # 真实样本1（PMO提供，脱敏）
│   ├── real_02_weekly.json       # 真实样本2（PMO提供，脱敏）
│   └── real_01_ledger.json       # 真实样本-台账格式
│
└── test_cases/
    ├── smoke_tests.json          # 冒烟测试用例集
    ├── extended_tests.json       # 扩展测试用例集（目标20条）
    └── eval_results.json         # 测试结果存档（自动生成）
```

## 四、样本数据获取责任人

| 数据类型 | 来源 | 提供方 | 脱敏处理 | 预计到位时间 |
|---------|------|--------|---------|------------|
| 演示样例 | 人工构造 | Agent | N/A | 已完成 |
| 开发样例 | 人工构造 | Agent | N/A | 阶段2完成时 |
| 真实样本 | PMO提供 | 用户/PMO | PMO负责 | 阶段1关闭后 |

**重要：** 真实样本数据必须由PMO提供，Agent不自行从任何渠道获取。一期不使用爬虫、API抓取等方式获取数据。
