# 正式试跑期问题处理规则

**版本：** v1.0
**日期：** 2026-04-08
**适用范围：** 正式MVP试跑期间（Q01+Q03+Q04全部确认后）

---

## 一、问题分类

| 类型 | 说明 | 典型场景 |
|------|------|---------|
| **阻断型** | 试跑无法继续，必须立即修复 | 脚本报错/无输出/判断逻辑根本性错误 |
| **继续观察型** | 不阻断试跑，但需持续观察 | 输出轻微偏差/边界情况 |
| **展示型** | 不影响核心功能，只影响展示 | 格式/截断/用词 |
| **数据补充型** | 样本问题，需补充数据 | 样本缺失/映射不完整 |
| **规则确认型** | 规则本身是否正确，需人工判断 | 误判/漏判/规则升级 |

---

## 二、各类型处理规则

### 2.1 阻断型（P0）

**判断标准：** 试跑无法继续，输出完全不可用

**处理流程：**
```
发现问题 → 运行负责人立即停止试跑
    → 登记P0问题
    → 通知阶段判断人
    → 触发回退机制
    → 修复后重新验证
    → 恢复试跑
```

**处理时限：** 立即（不超当天）

---

### 2.2 继续观察型（P2）

**判断标准：** 不影响核心功能，但需持续跟踪

**处理流程：**
```
发现问题 → 登记为P2观察项
    → 在《正式MVP试跑周报》中记录
    → 持续观察3次
    → 如3次中有2次以上重复出现 → 升级为P1
    → 否则关闭为观察
```

**处理时限：** 观察3个周期后判断

---

### 2.3 展示型（P1）

**判断标准：** 不影响判断逻辑，只影响可读性

**处理流程：**
```
发现问题 → 登记为P1展示问题
    → Agent评估是否可立即修复
    → 可立即修复 → 修复后继续
    → 需下次Q01/Q03确认 → 保留到下次确认周期
```

**处理时限：** 可立即修复的立即处理；需确认的等确认周期

---

### 2.4 数据补充型（P1）

**判断标准：** 样本缺失或映射不完整导致输出不完整

**处理流程：**
```
发现问题 → 登记为P1数据问题
    → 通知样本负责人补充数据
    → 补充后重新执行
    → 更新映射文件（如需要）
```

**处理时限：** 样本补充后当日或次日

---

### 2.5 规则确认型（P1）

**判断标准：** 规则判断可能存在误判/漏判，需人工复核判断

**处理流程：**
```
发现问题 → 登记为P1规则问题
    → 人工复核原始样本和输出
    → 判断是误判还是漏判
    → 误判 → 记录但不修改规则（规则已由Q01确认）
    → 漏判 → 记录，评估是否需更新规则
    → 如需更新规则 → 暂停试跑 → 走Q01重新确认流程
```

**处理时限：** 人工复核后1个工作日内给出判断

## 三、问题登记格式

| 字段 | 说明 |
|------|------|
| 编号 | RUN-P0/P1/P2-YYYYMMDD-NNN |
| 发现日期 | YYYY-MM-DD |
| 发现人 | 角色代称 |
| 问题类型 | 阻断/继续观察/展示/数据补充/规则确认 |
| 问题描述 | 具体描述 |
| 影响程度 | 高/中/低 |
| 试跑影响 | 暂停/继续/观察 |
| 处理动作 | 立即修/等确认/观察/升级 |
| 状态 | 处理中/已关闭 |

---

## 四、周报中的问题汇总

每周在《正式MVP试跑周报》中汇总：
1. 本周新增问题数量（P0/P1/P2）
2. 本周关闭问题数量
3. 累计未关闭问题数量
4. 是否影响继续建议

---

## 五、禁止事项

| 禁止 | 说明 |
|------|------|
| 不得在正式试跑中擅自修改规则 | 规则已由Q01确认，修改必须重新走确认流程 |
| 不得将P0降级 | P0必须立即处理 |
| 不得跳过人工复核 | 每期输出必须经过人工复核 |
| 不得隐瞒问题 | 问题必须登记，不得压低严重度 |
