# 阻断项状态看板定义

**版本：** v1.0
**用途：** Q01-Q05的统一状态机，应用于所有确认包和样本跟踪

---

## 一、状态定义

| 状态 | 含义 | 颜色 | 下一步动作 |
|------|------|------|---------|
| **未发出** | 确认包尚未发送给接收方 | 🔴 红色 | 立即发出 |
| **已发出** | 确认包已发送，等待签收 | 🟡 黄色 | 等待签收 |
| **已签收** | 接收人已确认收到 | 🟡 黄色 | 等待反馈 |
| **待反馈** | 接收人在处理中 | 🟡 黄色 | 继续等待 |
| **反馈中** | 接收人已口头反馈，正在走内部流程 | 🟡 黄色 | 等待书面确认 |
| **已通过** | 确认包已通过审阅 | 🟢 绿色 | 更新门控卡 |
| **已通过（有修改）** | 确认通过但有修改记录 | 🟢 绿色 | 执行修改，更新文档 |
| **被退回** | 确认包被退回，需重新发出 | 🔴 红色 | 重新处理后发出 |
| **暂缓确认** | 接收方暂时无法确认，需后续跟进 | 🟡 黄色 | 记录原因，设定跟进日期 |
| **已关闭** | 确认项已完成，后续不再跟踪 | ✅ 完成 | 归档 |

---

## 二、每种状态的进入条件

| 状态 | 如何进入 |
|------|---------|
| 未发出 | 初始状态 |
| 已发出 | 确认包发送给接收人 |
| 已签收 | 接收人书面/口头确认收到 |
| 待反馈 | 接收人表示"收到，正在处理" |
| 反馈中 | 接收人给出初步反馈，需等待最终确认 |
| 已通过 | 接收人签字确认，无修改 |
| 已通过（有修改） | 接收人确认，但有少量修改意见（记录修改内容） |
| 被退回 | 接收人拒绝确认，需重新发出 |
| 暂缓确认 | 接收人表示当前无法确认 |
| 已关闭 | 相关联的所有后续门控已更新 |

---

## 三、每种状态的退出条件

| 状态 | 退出到 | 退出条件 |
|------|--------|---------|
| 未发出 | 已发出 | 确认包实际发出 |
| 已发出 | 已签收 | 接收人确认签收 |
| 已发出 | 待反馈 | 超过期望反馈时间且无回应 |
| 已签收 | 待反馈 | 进入等待反馈阶段 |
| 待反馈 | 反馈中 | 接收人给出初步反馈 |
| 待反馈 | 已通过 | 接收人直接确认通过 |
| 待反馈 | 被退回 | 接收人拒绝确认 |
| 反馈中 | 已通过 | 书面最终确认通过 |
| 反馈中 | 已通过（有修改） | 确认通过但有修改 |
| 反馈中 | 被退回 | 接收人拒绝确认 |
| 反馈中 | 暂缓确认 | 接收人暂停确认 |
| 已通过 | 已关闭 | 所有关联门控已更新 |
| 已通过（有修改） | 已关闭 | 修改已执行并确认 |
| 被退回 | 未发出 | 重新处理后再次发出 |
| 暂缓确认 | 待反馈 | 接收人恢复处理 |

---

## 四、状态更新规则

1. **更新时机：** 状态变化后1个工作日内更新本看板
2. **更新人：** Agent（或协调人）
3. **更新记录：** 每次状态变化须记录日期和原因
4. **超期标记：** 超过期望反馈时间5个工作日，自动标红

## 六、状态机流程图

```
未发出 ──发出──→ 已发出 ──签收──→ 已签收
                    │                │
                    │ 超期无回应     │ 接收人在处理
                    ▼                ▼
                待反馈 ←──反馈── 反馈中
                    │    │          │
                    │    │ 拒绝      │ 最终确认
                    ▼    ▼          ▼
                被退回  已通过 → 已通过（有修改）
                    │         │
                    │ 重新处理  │ 修改执行
                    ▼         ▼
                未发出      已关闭
```
