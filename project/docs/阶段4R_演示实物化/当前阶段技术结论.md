## 三、后续替换真实数据的方式

### 替换原则
- **只换数据，不换结构**
- INPUT_SCHEMA不变 → OUTPUT_SCHEMA不变 → 页面不动
- 判断规则不变 → 逻辑不动

### 替换路径
```
真实周报 → 按INPUT_SCHEMA整理 → 放入 /normalized/weekly_reports/by_period/
     ↓
运行 run_prototype.py（路径锁定）
     ↓
自动生成结构化JSON + 文本摘要
     ↓
业务方审阅 → 管理决策
```

### 数据契约
| 文件类型 | 存放路径 | 格式 |
|----------|----------|------|
| 原始周报 | `normalized/weekly_reports/by_period/YYYY-MM-DD_YYYY-MM-DD.json` | JSON |
| 台账快照 | `normalized/ledgers/snapshots/YYYY-MM-DD_YYYY-MM-DD.json` | JSON |
| 指标快照 | `normalized/metrics/snapshots/YYYY-MM-DD.json` | JSON |
