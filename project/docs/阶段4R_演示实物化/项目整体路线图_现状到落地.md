## 常见问题回答

**Q：这套系统已经上线了吗？**  
A：没有，目前是原型验证阶段，还没接真实生产数据。

**Q：什么时候能用上？**  
A：等真实样本到位后做影子运行→评审通过→真实试运行。按当前进度，最快路径还需要1-2轮。

**Q：现在能看什么？**  
A：打开 `演示总览页_可跳转.html`，30分钟看完项目全貌。

**Q：如果不想用这套工具怎么办？**  
A：完全没问题，这只是一期原型。最终决策权在管理层和业务方。

**Q：如果系统判断错了怎么办？**  
A：系统输出必须经人工审阅才能用于管理决策，不存在"系统自动决定"的情况。

## 演示后分支路径（阶段4S-A产出）

演示完成后，根据《内部演示决策记录模板》中的3项决策，选择分支走向：

```
演示完成 → 填决策记录（A结构认可 / B影子运行授权 / C评审推进）
    ↓
├── 分支A：演示认可 → 直接进入影子运行（A同意 且 B同意/有条件）
│
├── 分支B：演示认可，但先修小问题 → 修完后进入分支A（A/B有条件，小问题可修复）
│
└── 分支C：先推进扩大评审 → 评审通过后进入分支A（C同意，评审优先）
```

| 分支 | 最短时间 | 核心阻塞 |
|------|---------|---------|
| 分支A | 样本到位即可 | 真实样本（FB-001/S-001）未到位 |
| 分支B | +1-3天（修小问题） | 展示层/口径小问题 |
| 分支C | +评审周期 | 评审方时间确认 |

详细说明见：`project/docs/阶段4S-A_内部演示执行/演示后分支说明.md`
