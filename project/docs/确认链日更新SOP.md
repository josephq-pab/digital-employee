# 确认链日更新SOP

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 每天执行确认链状态更新的标准操作流程
**适用对象：** 用户或Agent

---

## 一、每日更新顺序（不可打乱）

```
① 发出登记检查
  ↓
② 签收状态检查
  ↓
③ 反馈状态检查
  ↓
④ 回填模板更新
  ↓
⑤ 门控卡更新
  ↓
⑥ 周报模板更新（如需要）
```

---

## 二、逐项操作说明

### ① 发出登记检查（每日开始时）

**动作：**
1. 打开《确认包发出登记表.md》
2. 对比"当前状态"与"实际状态"
3. 如有变化，立即更新

**判断标准：**
- 包状态为"待发出"但用户已发出 → 更新为"已发出"，填日期
- 等待超期（超7个工作日）→ 触发催办

---

### ② 签收状态检查

**动作：**
1. 确认已发出的包是否已签收
2. 如已签收，更新"实际签收日期"，状态→"待反馈"
3. 如未签收且超期，触发催办

---

### ③ 反馈状态检查

**动作：**
1. 检查是否有新反馈到达
2. 判断反馈来源（哪个包）
3. 判断反馈结果类型（同意/修改后同意/不同意/暂缓）

---

### ④ 回填模板更新

**动作：**
1. 打开《确认结果回填模板.md》
2. 找到对应确认编号的行
3. 填写：收到日期、结果、具体内容
4. 填写"简化汇总栏"

---

### ⑤ 门控卡更新

**动作：**
1. 打开《阶段4B进入门控卡.md》
2. 根据回填结果，判断门控变化
3. 如有门控项变化，更新颜色和状态
4. 记录"最后更新时间"

**变化判定：**
- 反馈为"同意" → 对应G→🟢
- 反馈为"暂缓" → 对应G维持🟡，标注"暂缓"
- 反馈为"不同意" → 对应G维持🔴，发起升级

---

### ⑥ 周报模板更新（如需要）

**动作：**
1. 如门控有重大变化（如总体从🔴变为🟡），更新《阻断项推进周报》
2. 如无变化，无需操作

---

## 三、每日更新快速检查清单（5分钟）

```
□ 打开《确认包发出登记表》
□ 检查4个包的当前状态
□ 如有发出/签收/反馈变化 → 立即更新
□ 打开《确认回填待命清单》
□ 如有新反馈 → 按清单执行5步动作
□ 打开《阶段4B进入门控卡》
□ 如有门控变化 → 更新门控
□ 确认G1/G2/G3当前状态
□ 确认总体状态（🟢/🟡/🔴）
□ 确认距进入4B还差什么
```

## 四、不同角色的操作权限

| 操作 | 用户 | Agent | 说明 |
|------|------|-------|------|
| 更新发出日期 | ✅ | ✅ | 收到用户确认后执行 |
| 更新签收日期 | ✅ | ✅ | 收到用户确认后执行 |
| 填回填模板 | ✅ | ✅ | 收到用户反馈后执行 |
| 更新门控卡 | ✅ | ✅ | 回填完成后执行 |
| 发出催办 | ✅ | ❌ | 仅用户可对外发消息 |
| 宣布进入4B | ✅ | ❌ | 仅用户可触发阶段切换 |

## 五、哪些状态变化必须同步写回docs

| 变化类型 | 必须同步的文档 | 时限 |
|---------|-------------|------|
| 包状态变化 | 确认包发出登记表 | 1个工作日内 |
| 收到反馈 | 确认结果回填模板 | 1个工作日内 |
| 门控颜色变化 | 阶段4B进入门控卡 | 1个工作日内 |
| 门控总体变化 | 决策记录与待确认事项 | 1个工作日内 |
| 新决策形成 | 决策记录与待确认事项 | 1个工作日内 |

## 六、异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 超过7个工作日无签收 | 触发催办，记录催办次数 |
| 超过14个工作日无反馈 | 升级处理，用户决定是否电话催办 |
| 多个包同时收到反馈 | 按优先级处理：MGT-01 > PMO-01/02 > PILOT-01 |
| 反馈内容涉及多个确认项 | 拆分为多个确认项，逐项回填 |
| 反馈与默认建议冲突 | 暂停，进入《确认反馈合并规则》中的冲突解决流程 |

---

## 七、新Agent接手后的第一动作

```
Day 1 第1件事：
  1. 读《确认包发出登记表》→ 确认当前4个包的状态
  2. 读《阶段4B进入门控卡》→ 确认G1/G2/G3当前颜色
  3. 读《确认回填待命清单》→ 确认6项的状态
  4. 执行"每日更新快速检查清单"（5分钟）

  → 完成上述4步后，即可立即上手，无需额外记忆
```

## 八、与《确认反馈合并规则》的关系

| 本SOP | 《确认反馈合并规则》 |
|-------|-----------------|
| 每日执行的操作流程 | 收到反馈后的处理逻辑 |
| 执行层面（怎么做） | 决策层面（怎么判断） |
| 顺序性操作 | 分支判断逻辑 |

**两者配合使用：本SOP管"每日做什么"，《合并规则》管"收到反馈后怎么判断"。**
