# 减负与价值判断标准

**版本**：v1.0  
**日期**：2026-04-09  
**阶段**：阶段5A《并行运行与价值验证》  
**用途**：明确什么叫"减轻了实际工作"、"替代了部分人工判断"、"已产生业务价值"

---

## 一、"减轻了实际工作"的判断标准

**定义**：数字员工的输出直接减少了人工处理时间或工作量。

**判断条件（满足任一即可）**：

| 场景 | 具体标准 |
|------|---------|
| 信息整理 | 原来需要X小时整理的内容，现在只需Y分钟核对（Y < X/2） |
| 风险识别 | 原来需要逐条阅读判断的风险，现在系统已给出判断，人工只需复核 |
| 摘要生成 | 原来需要人工写的领导摘要，现在系统生成初稿，人工只需修改 |
| 台账维护 | 原来需要人工维护的台账，现在系统自动更新，人工只需确认 |
| 对照检查 | 原来需要人工自查的判断，现在系统已给出对照结果，人工只需确认差异 |

**不计入"减轻工作"的情况**：
- 数字员工生成了输出，但人工需要全部重新核对才能使用
- 数字员工的输出反而增加了人工需要处理的信息量
- 数字员工生成的内容与业务场景不匹配，无法直接使用

---

## 二、"替代了部分人工判断"的判断标准

**定义**：原本需要人工基于经验做的判断，现在系统能给出合理的初步判断。

**判断条件（满足任一即可）**：

| 场景 | 具体标准 |
|------|---------|
| 风险等级判断 | 系统判断"黄灯/逾期"与人工判断一致率超过80% |
| 结项申请判断 | 系统判断"同意结项/不同意结项"与人工判断一致率超过80% |
| 量化指标评估 | 系统计算的指标偏离幅度与人工评估结果基本一致 |
| 优先级判断 | 系统给出的优先级排序与人工优先级认知基本匹配 |

**不计入"替代人工判断"的情况**：
- 数字员工只是做了数据整理，没有做判断（如：只是列出了事项清单，没有判断风险）
- 数字员工的判断与人工判断经常不一致，需要人工全部重新判断
- 判断场景过于主观，无法用规则描述（如：需要经验积累才能判断的事项）

---

## 三、"已产生业务价值"的判断标准

**定义**：数字员工的输出对业务决策产生了正面影响。

**判断条件（满足任一即可）**：

| 场景 | 具体标准 |
|------|---------|
| 管理效率提升 | 管理层每周用于了解项目状态的时间减少（原来1小时，现在20分钟） |
| 风险发现 | 系统识别出的人工未注意到的风险项 |
| 决策支持 | 管理层在决策时参考了系统输出的摘要或风险清单 |
| 信息对齐 | 不同参会人对同一事项的状态判断通过系统输出一致化 |
| 历史追踪 | 系统输出的风险历史追踪帮助人工发现了被忽视的延期项 |

**不计入"业务价值"的情况**：
- 系统输出了很多内容，但无人查阅
- 系统输出被查阅但未被采纳
- 系统输出反而引起了不必要的讨论或质疑

---

## 四、综合价值判断

**每周期由业务方填写，判断标准如下**：

| 综合结论 | 判断条件 |
|---------|---------|
| **明显价值** | 减轻工作√ + 替代判断√ + 业务价值√ |
| **部分价值** | 满足上述任两类，且无明显负反馈 |
| **无价值** | 三项均不满足，或出现明显负反馈 |
| **负价值** | 系统输出反而增加了工作负担或导致决策偏差 |

---

## 五、偏差率计算方式

**偏差率 = 偏差点数 / 总判断点数 × 100%**

| 偏差类型 | 说明 |
|---------|------|
| 完全偏差 | 系统判断与人工完全相反（如：人工判黄灯，系统判正常） |
| 部分偏差 | 系统判断与人工基本一致，但理由或程度不同 |
| 口径偏差 | 判断本身一致，但用的分类名称或表述方式不同（不影响使用） |

**注**：口径偏差不计入偏差率，但需记录以便后续优化表述。
