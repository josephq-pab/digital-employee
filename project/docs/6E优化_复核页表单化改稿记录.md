## 一、改前问题

| 问题 | 具体表现 |
|------|---------|
| 复核页像浏览页 | 标题是"人工复核与ABC决策"，但没有明确的填写引导 |
| ABC选择混杂 | 三个按钮并排，选完没有明确的状态反馈 |
| 无问题填写引导 | "具体问题填写区"没有显式提示必须填具体内容 |
| 偏差记录是表格 | 先展示空表格，用户不知道该填什么 |
| 不像操作页 | 用户进来不知道这是需要"操作"的页面 |

## 二、改前/改后对比

### 改前复核页结构

```
✅ 人工复核与ABC决策
记录一致/偏差，登记ABC决策结果

[card: 本期运行概要 - 纯展示]
[card: 一致率汇总 - 纯展示]
[card: 偏差记录 - 表格形式，空状态]
[card: ABC决策登记 - 说明文字+三个按钮]
[card: 历史决策记录 - 表格]
```

### 改后复核页结构（6E优化版）

```
✅ 复核与ABC决策
PMO协调人专用：复核系统输出 → 填写判断 → 做出决策

[绿色横幅：本页是操作页，不是浏览页]

[card Q1: 本期系统是否跑通？]
  ✅ 是，跑通了 / ⚠️ 部分跑通 / ❌ 否
  → 选中后显示结果

[card Q2: 系统输出对管理是否有价值？]
  ✅ 有价值 / ⚠️ 部分有价值 / ❌ 无价值

[card Q3: 系统输出是否减轻了你的工作？]
  ✅ 减轻了 / ⚠️ 部分减轻 / ❌ 没减轻

[card Q4: 本期是否存在偏差？]
  ✅ 无偏差 / ⚠️ 少量偏差 / ❌ 较多偏差
  → 选"少量/较多"时出现偏差填写区（强制填写）

[card Q5: 选择本期ABC决策]
  [A 接受]  [B 修改后接受]  [C 不接受]
  → 选B/C时出现具体问题填写区（强制填写）
  [✅ 提交复核结论]  → 全部回答后激活
```

## 三、核心优化：表单化

### 3.1 Q1-Q4固定问题结构

每道题固定结构：
1. 问题文字（明确问什么）
2. 三选一按钮（✅ / ⚠️ / ❌）
3. 结果反馈（选中后显示）
4. Q4有特殊逻辑：选"少量偏差/较多偏差"时，强制展开偏差填写区

### 3.2 ABC选择卡片化

**改前：**
```
[button: A（接受）] [button: B（修改后接受）] [button: C（不接受）]
```

**改后：**
```
┌────────────┐  ┌────────────┐  ┌────────────┐
│    A      │  │    B      │  │    C      │
│   接受    │  │ 修改后接受 │  │  不接受   │
│ 系统可直接 │  │需人工修改  │  │ 重大偏差  │
│  使用     │  │  部分内容  │  │ 不适合参考 │
└────────────┘  └────────────┘  └────────────┘
```

卡片点击后高亮，选B/C时自动展开具体问题填写区。

### 3.3 提交按钮激活逻辑

- 5道题全部回答 → 按钮变亮，cursor变pointer
- 任意题未回答 → 按钮变灰，disabled
- 防止用户跳着填或漏填

## 四、如何降低填写门槛

| 改前 | 改后 |
|------|------|
| 偏差记录是空表格 | Q4直接问"是否存在偏差"，选是才出现填写区 |
| 偏差说明没有格式要求 | 占位符给出格式："事项\|系统判断\|人工判断\|说明" |
| ABC说明是长文字 | 三个卡片，每个卡片一句话说明 |
| 提交按钮总可点 | 未填完时按钮禁用，防止无效提交 |
| 没有"本页是操作页"的提示 | 顶部绿色横幅明确告知 |
