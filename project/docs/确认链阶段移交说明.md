## 三、第一优先级任务（接手后立即执行）

### 3.1 第1件事：检查门控卡

打开《阶段4B进入门控卡.md》，确认G1-G7当前状态：

```
G1 Q01 = 🔴/🟡/🟢 ？
G2 Q03 = 🔴/🟡/🟢 ？
G3 Q04 = 🔴/🟡/🟢 ？
G4 S1样本 = 🟡已到位（待确认）？
G5 S2样本 = 🟡已到位（待确认）？
G6 文档冻结 = 🟡部分冻结？
G7 冒烟测试 = 🟢已就绪？
```

### 3.2 第2件事：检查确认包发出状态

打开《确认包签收与反馈登记表.md》，确认：
- Q01/Q03/Q04确认包是否已发出？
- 签收日期是什么时候？
- 预计反馈时间？

### 3.3 第3件事：确认当前最高优先级的下一步

| G1/G2/G3状态 | 下一步动作 |
|-------------|----------|
| 全部🔴未发出 | 用户发出确认包（这是用户的责任，Agent只能催） |
| 全部🟡已发出待反馈 | 等待反馈（可催办） |
| 任意🟢 | 更新门控卡，继续盯剩余项 |

## 四、发确认包的标准流程

### 4.1 发包前

1. 打开《确认包发出清单.md》
2. 核对发出前检查项（5项）
3. 确认发出对象、发出方式、发出时间

### 4.2 发包时

1. 使用《管理层确认外发版.md》发给管理层
2. 使用《PMO确认外发版.md》发给PMO
3. 使用《试点范围确认外发版.md》发给PMO+管理层

### 4.3 发包后

1. 登记到《确认包签收与反馈登记表.md》
2. 设定预计反馈时间提醒
3. 按《阻断项催办话术》安排催办节奏

---

## 五、收到反馈后的标准动作

```
Step 1：打开《确认结果回填模板.md》
Step 2：逐项填写反馈结果
Step 3：判断结果类型（同意/修改后同意/不同意/暂缓）
Step 4：按《确认反馈合并规则》处理
Step 5：更新《阶段4B进入门控卡.md》
Step 6：如全部确认完成 → 执行《4B实现切入说明》
```

## 六、禁止抢跑规则

即使Agent认为"条件已成熟"，以下情况绝对不能抢跑：

| 场景 | 禁止原因 |
|------|---------|
| Q01未确认但Agent认为规则已稳定 | 规则未经PMO签字，存在法律/业务风险 |
| 样本已到位但未确认适配方案 | 适配方案未经确认，可能导致判断错误 |
| G4/G5已🟡但G1仍🔴 | G1是核心阻断项，未清零不得进入4B |
| 用户催促进度但门控未清零 | 门控卡是唯一真实源，不得绕过 |
