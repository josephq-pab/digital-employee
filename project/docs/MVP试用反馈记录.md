## 五、反馈处理原则

1. **只记录具体问题，不记录抽象评价**
   - ❌ "系统不好用" → ✅ "点击台账页，筛选功能无响应"
   - ❌ "输出不够清晰" → ✅ "风险清单中M-001的阶段目标15亿来源不明确"

2. **分类归入四大类之一**
   - 体验问题 → 进入优化清单P2级
   - 表达问题 → 进入优化清单P1/P2级
   - 流程问题 → 进入流程优化框架
   - 非问题建议 → 记录为观察项

3. **明确是否可复现**
   - 可复现 → 进入问题登记
   - 不可复现 → 记录为一次性观察
