# 演示反馈收口机制

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 将演示反馈转化为后续行动，不是只停留在"记录"

---

## 一、反馈收口总流程

```
演示结束
    ↓
收集反馈（填写演示版验收记录表.md）
    ↓
按类型分类（演示反馈归并规则.md）
    ↓
本机制：类型→文档→阶段→处理人
    ↓
产出：行动项清单（不是报告）
    ↓
执行/等待确认
    ↓
更新门控（如涉及Q01/Q03/Q04）
```

---

## 二、反馈→行动映射

### 类型1：展示问题

| 环节 | 内容 |
|------|------|
| 反馈来源 | 演示版验收记录表.md |
| 归入文档 | 展示问题清单.md（新建，如不存在则建） |
| 处理阶段 | 正式演示前（下次演示前处理） |
| 处理人 | Agent |
| 行动类型 | 修改输出格式/标签/顺序 |
| 是否需要确认 | 否（Agent自行处理） |
| 更新门控 | 否 |

### 类型2：规则问题（涉及Q01-Q05）

| 环节 | 内容 |
|------|------|
| 反馈来源 | 演示版验收记录表.md |
| 归入文档 | 规则问题待确认清单.md（新建） |
| 处理阶段 | 等Q01-Q05确认后处理 |
| 处理人 | PMO/管理层（需要外部确认） |
| 行动类型 | 规则修改或维持provisional |
| 是否需要确认 | 是（必须等原确认人回复） |
| 更新门控 | 是（如涉及G1/G2/G3） |

### 类型3：规则问题（不涉及Q01-Q05）

| 环节 | 内容 |
|------|------|
| 反馈来源 | 演示版验收记录表.md |
| 归入文档 | QUAL清单.md（在现有QUAL基础上追加） |
| 处理阶段 | 阶段5或未来阶段 |
| 处理人 | Agent |
| 行动类型 | 记录为改进项 |
| 是否需要确认 | 否 |
| 更新门控 | 否 |

### 类型4：数据问题

| 环节 | 内容 |
|------|------|
| 反馈来源 | 演示版验收记录表.md |
| 归入文档 | 数据问题反馈表.md（新建） |
| 处理阶段 | 下期样本准备时处理 |
| 处理人 | PMO（数据提供方） |
| 行动类型 | 反馈给数据方，映射问题触发MAP确认 |
| 是否需要确认 | 否（给数据方的建议） |
| 更新门控 | 否 |

### 类型5：需求扩展

| 环节 | 内容 |
|------|------|
| 反馈来源 | 演示版验收记录表.md |
| 归入文档 | 需求池.md（在现有需求池基础上追加） |
| 处理阶段 | 一期结束后评估 |
| 处理人 | 用户/管理层决定 |
| 行动类型 | 记录为未来需求 |
| 是否需要确认 | 否 |
| 更新门控 | 否 |

---

## 三、反馈处理时限

| 反馈类型 | 记录时限 | 初步响应时限 | 处理完成时限 |
|---------|---------|------------|------------|
| 展示问题 | 演示当天 | 1天 | 下次演示前 |
| 规则问题（涉Q01-Q05） | 演示当天 | 1天 | 等外部确认 |
| 规则问题（不涉Q01-Q05） | 演示当天 | 3天 | 记录即可 |
| 数据问题 | 演示当天 | 1天 | 下期样本前 |
| 需求扩展 | 演示当天 | 1天 | 一期结束后 |

---

## 四、门控联动规则

当反馈涉及Q01/Q03/Q04时：

| 反馈内容 | 门控影响 | 动作 |
|---------|---------|------|
| 涉及MGT-01（管理层格式） | G2可能变化 | 记录，不自行更新门控 |
| 涉及PMO-01/02（术语/规则） | G1可能变化 | 记录，不自行更新门控 |
| 涉及PILOT-01（试点范围） | G3可能变化 | 记录，不自行更新门控 |

**规则：门控更新必须基于正式确认反馈，不基于演示口头反馈。**

---

## 五、行动项清单格式

每次演示后，Agent必须产出以下清单（写入演示版验收记录表.md的"下一步跟进"部分）：

```
## 演示后行动项

| # | 类型 | 具体内容 | 处理人 | 截止日期 | 状态 |
|---|------|---------|--------|---------|------|
| 1 | 展示问题 | （具体描述） | Agent | （日期） | 待处理 |
| 2 | 规则问题（涉Q01） | （具体描述） | PMO | 等确认 | 待确认 |
| ... | ... | ... | ... | ... | ... |
```

---

## 六、反馈收口检查点

每次处理完反馈后，检查：

```
□ 反馈已记录到对应文档
□ 行动项已写入演示版验收记录表.md
□ 涉及门控的已标记（不自行更新）
□ 已通知相关责任人（如需外部确认）
□ 下次演示前已完成展示问题处理
```

---

## 七、闭环确认

反馈处理的闭环标志：

| 类型 | 闭环标志 |
|------|---------|
| 展示问题 | 下次演示验证通过 |
| 规则问题（涉Q01-Q05） | Q01-Q05门控状态变化 |
| 规则问题（不涉Q01-Q05） | 记录进入QUAL清单/需求池 |
| 数据问题 | PMO确认数据已更新 |
| 需求扩展 | 记录进入需求池 |
