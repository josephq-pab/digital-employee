## 一、角色总览

| 角色 | 人员/职位 | 职责 |
|------|----------|------|
| 数据提供人 | PMO | 通过原渠道转发周报数据 |
| 运行责任人 | Agent（数字员工） | 接收数据、生成输出、执行运行链 |
| 复核责任人 | PMO试点协调人 | 对照现有机制复核系统输出 |
| 问题登记责任人 | PMO试点协调人 | 记录偏差案例，登记具体问题 |
| 收口主持人 | PMO试点协调人（或指定） | 主持周五收口会 |

---

## 二、每角色具体职责

### 2.1 数据提供人

| 职责 | 具体动作 |
|------|---------|
| 提供周报数据 | 每周一上午通过原渠道（邮件/群/文件共享）转发PMO周报 |
| 确认数据格式 | 确认数据为JSON格式或可解析格式 |
| 通知运行责任人 | 数据转发后通知Agent（数字员工）可接入 |

**约束**：数据提供人不负责人工复核，只负责提供原数据。

### 2.2 运行责任人（Agent）

| 职责 | 具体动作 |
|------|---------|
| 接收数据 | 确认数据已接入 |
| 生成输出 | 按标准模板生成领导摘要/台账/风险清单 |
| 记录运行日志 | 记录运行时间、输入内容、输出路径 |
| 归档输出 | 将输出文件放入指定目录 |

**约束**：Agent不负责人工复核，不做人工判断。

### 2.3 复核责任人

| 职责 | 具体动作 |
|------|---------|
| 接收系统输出 | 在周一/周二接收Agent生成的输出文件 |
| 对照现有机制复核 | 与现有台账/判断方式逐项对照 |
| 记录一致点 | 记录系统判断与人工一致的项 |
| 记录偏差点 | 记录系统判断与人工不一致的项 |
| 填写价值验证记录 | 在周四17:00前填写《首周期价值验证记录》 |

**约束**：复核责任人必须对照实际业务判断，不能主观评价。

### 2.4 问题登记责任人

| 职责 | 具体动作 |
|------|---------|
| 收集问题 | 从复核过程和业务沟通中收集具体问题 |
| 筛选问题 | 按《哪些问题需要决策哪些不需要》判断是否需要决策 |
| 登记问题 | 将需要决策的问题填入《具体问题ABC决策表》 |
| 判断紧迫性 | 标注是否需要当周决策 |

**约束**：只登记具体问题（必须有案例），不登记抽象评价。

### 2.5 收口主持人

| 职责 | 具体动作 |
|------|---------|
| 准备收口材料 | 汇总本周输入/输出/偏差/价值记录 |
| 主持收口会 | 控制时间，引导讨论 |
| 确认ABC决策项 | 确认本周是否有需要决策的问题 |
| 记录收口结论 | 记录下周计划和待确认事项 |
