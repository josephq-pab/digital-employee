# 一期范围说明

## 一、一期做什么

**只做这一件事：**
> 给定一份结构化的重点项目周报/月报/项目台账，输出标准化的执行状态摘要（包含状态等级、风险判断、建议动作、升级建议），供管理层审阅。

**一期数据来源：** 人工填报，支持周报格式、月报格式、项目台账格式。**一期不做系统自动采集。**

**一期必须包含的最小内容集：**

| 内容 | 说明 |
|------|------|
| 单项目输入 | 支持单个项目台账/周报的完整处理链路 |
| 结构化输出 | 按《一期术语与判断口径》输出的标准化摘要 |
| 事实/判断/建议分离 | 输出中明确区分事实提取、判断依据、建议动作 |
| 人工审阅节点 | 输出后必须有人的审阅环节，系统不替代人做决策 |
| 冒烟测试 | 5条核心规则的自动化验证 |
| 可演示样例 | 至少1份可运行的真实格式样例 |

---

## 二、一期不做什么

**明确不引入以下内容：**

| 类别 | 不做的事项 | 原因 |
|------|-----------|------|
| 数据源 | 接入 OA、项目管理平台 | 一期以人工填报为唯一数据源 |
| 自动化 | 自动催报、自动通知、自动升级 | 一期允许人工确认，不做自动闭环 |
| 展示层 | Dashboard、图表、前端界面 | 一期只输出结构化文本 |
| 聚合 | 多项目横向对比、自动排名 | 一期只做单项目纵向跟踪 |
| 决策 | 自动判断延期可接受性 | 最终决策权归人工 |
| 生态 | Agent间协作、多Agent调度 | 一期单Agent运行 |

---

## 三、暂不做什么（后续考虑）

以下内容在PMO试点成功后，可纳入后续阶段评估：

| 事项 | 前提条件 |
|------|---------|
| 接入飞书/企微，获取真实周报数据 | 飞书机器人/企微机器人已配置，PMO同意接入 |
| 接入项目管理平台（PMIS） | PMO提出明确需求，数据格式可标准化 |
| 多项目汇总报告 | 至少完成3个以上项目的单项目原型验证 |
| 自动催报 | 接入消息通道后，作为增值功能评估 |
| 自动升级推送 | 升级建议经人工确认有效后，作为半自动功能评估 |

---

## 四、一期原型与MVP的关系

**一期原型（Phase 4产出）：**
- 目标：验证核心逻辑可运行
- 形态：本地脚本 + JSON输入/输出
- 验收标准：冒烟测试100%通过 + 演示用样例可正常运行
- 数据：使用模拟/脱敏样例

**一期MVP（Phase 5产出）：**
- 目标：在PMO真实试点中可用
- 形态：HTTP API服务（可选）+ 真实样例数据验证
- 验收标准：PMO实际使用并给出反馈
- 数据：接入真实周报/台账格式

**差别：** 原型验证逻辑闭环，MVP验证业务价值闭环。

---

## 五、一期试点边界

**限定范围：**
- 仅限PMO团队重点跟踪的项目
- 仅限周报/月报/项目台账格式的结构化数据
- 仅限管理层作为最终受众
- 仅限单项目串行处理（暂不支持批量）

**后续扩展边界由阶段3试点运行机制设计时再定义。**
