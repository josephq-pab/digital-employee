# 真实生产数据影子运行SOP

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4L《扩大评审准备与真实试运行待命》

---

## 一、影子运行定义

**影子运行 = 仅做输入→输出比对，人工确认每条输出的正确性，不替代任何人工流程，不写回任何生产系统。**

---

## 二、影子运行约束（不可违反）

| 约束 | 说明 | 违反处理 |
|------|------|---------|
| 不替代人工判断 | 所有输出仅供人工参考 | 立即暂停 |
| 不写回生产系统 | 任何情况下不修改生产数据 | 立即停止+调查 |
| 完全可回退 | 关闭系统即可，无任何遗留 | - |
| 逐条人工确认 | 每条输出均需人工确认才能标记为"已确认" | 流程要求 |

---

## 三、影子运行步骤

### 第0步：样本接入（执行1次）

1. 接收业务方提供的样本（≥ 3期）
2. 检查字段完整性
3. 存入 `openclaw_unified_sample_pack/openclaw_unified_sample_pack/real_sample/`
4. 反馈业务方"样本已接入，开始影子运行"

---

### 第1步：适配验证（每期样本执行）

1. 将样本文件路径加入适配器
2. 执行适配器，输出 adapted_items
3. 检查适配成功率：适配成功的项 / 总项
4. 记录适配失败项及其原因
5. 适配成功率 < 60% → 暂停 → 评估适配器修正可行性

---

### 第2步：归一化与规则判断（每期样本执行）

1. 执行 normalizer，输出 normalized_items
2. 执行 rule_engine，输出 engine_results
3. 执行 generator，输出领导摘要/台账/风险清单
4. 全流程无报错 → 继续；有阻断性报错 → 暂停 → 记录 → 修正

---

### 第3步：人工对照（每期样本执行）

1. 将系统输出提交给业务方对接人
2. 业务方逐条核对"系统输出 vs 人工判断"
3. 在《真实试运行结果登记表》中记录：
   - 事项识别正确（系统判断 = 人工判断）
   - 事项识别错误（系统判断 ≠ 人工判断）
   - 事项遗漏（人工判断有，系统未识别）
   - 事项误报（系统识别，人工判断不存在）

---

### 第4步：差异评估（每期样本执行后）

1. 统计差异类型和比例
2. 评估差异原因：
   - 适配器问题（格式未覆盖）→ 修正适配器
   - 规则问题（判断逻辑错误）→ 修正规则（核心引擎，慎重）
   - 样本问题（数据质量问题）→ 反馈业务方
3. 差异率 > 20% → 暂停 → 修正 → 再验证
4. 差异率 ≤ 20% → 可接受 → 记录 → 继续

---

### 第5步：结果登记（每期样本执行后）

1. 填写《真实试运行结果登记表》
2. 结论标记：
   - 可接受（差异率 ≤ 20%）
   - 需修正（差异率 > 20%，可修复）
   - 暂停（存在核心适配缺陷）

---

## 四、影子运行输出

| 输出 | 内容 | 用途 |
|------|------|------|
| 领导摘要 | TXT文件 | 人工核对用 |
| 事项总台账 | TXT文件 | 人工核对用 |
| 风险清单 | TXT文件 | 人工核对用 |
| 归一化结果 | JSON文件 | 技术分析用 |
| 差异记录 | 填入《真实试运行结果登记表》| 决策用 |

---

## 五、终止条件

以下任一条件满足，终止影子运行：

1. 适配成功率 < 60%，且适配器无法快速修正
2. 差异率 > 20%，且核心适配逻辑存在缺陷
3. 业务方主动要求暂停
4. 发现违反约束的情况（不替代人工/不写回系统）
