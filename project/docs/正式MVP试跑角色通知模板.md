# 正式MVP试跑角色通知模板

**版本：** v1.0
**日期：** 2026-04-08
**适用范围：** 正式MVP试跑启动/暂停/周结论通知

---

## 一、通知类型总览

| 通知类型 | 触发条件 | 发送方 | 接收方 |
|---------|---------|-------|-------|
| 试跑启动通知 | Q01+Q03+Q04全部确认 | Agent+联络负责人 | 全体参与者 |
| 试跑暂停通知 | 发现P0/规则错误/管理层要求 | Agent+联络负责人 | 全体参与者 |
| 试跑恢复通知 | 问题修复后恢复 | Agent+联络负责人 | 全体参与者 |
| 试跑周结论通知 | 每周最后一个工作日 | 阶段判断人 | 管理层/PMO |

---

## 二、试跑启动通知

```
【正式MVP试跑正式启动】

日期：[YYYY-MM-DD]
版本：[MVP-RUN-01]

各位：

在Q01（PMO术语口径）、Q03（管理层输出格式）、Q04（试点项目范围）已正式确认的基础上，正式启动组织执行力数字员工MVP试跑。

一、试跑目的
验证数字员工原型在真实试点场景中的业务价值闭环。

二、试点范围
[试点项目名称]

三、试跑周期
预计[XX]周

四、角色分工
- 样本负责人：[角色代称/姓名]
- 运行负责人：[角色代称/姓名]
- 复核负责人：[角色代称/姓名]
- 问题登记人：[角色代称/姓名]
- 阶段判断人：[角色代称/姓名]

五、本期样本
周报期间：[YYYY-WXX]
台账记录：[N]条

六、产出说明
每周产出《正式MVP试跑周报》，供管理层和PMO参考。
所有产出均为provisional版本，正式口径以Q01/Q03确认内容为准。

七、下一步
[下一步具体动作]

请各角色按《正式MVP试跑首周SOP》做好准备。

[Agent签名]
日期：[YYYY-MM-DD]
```

## 六、通知发送规则

| 通知类型 | 发送时间 | 发送方式 | 存档位置 |
|---------|---------|---------|---------|
| 启动通知 | Q01+Q03+Q04全部确认当天 | 邮件/飞书 | project/docs/notifications/ |
| 暂停通知 | 暂停触发当天 | 邮件/飞书 | project/docs/notifications/ |
| 恢复通知 | 修复验证通过当天 | 邮件/飞书 | project/docs/notifications/ |
| 周结论通知 | 每周最后一个工作日 | 邮件/飞书 | project/docs/notifications/ |

---

## 七、通知存档要求

| # | 要求 |
|---|------|
| 1 | 所有通知必须存档到 `project/docs/notifications/` |
| 2 | 存档文件名格式：`[类型]_[YYYYMMDD]_[版本].txt` |
| 3 | 不得以口头通知替代书面通知 |
| 4 | 通知内容不得包含未确认信息 |
| 5 | 不得在通知中擅自去除provisional标注 |
