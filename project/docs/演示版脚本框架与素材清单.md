# 演示版脚本框架与素材清单

**版本：** v1.0
**目的：** 定义演示版的框架，不制作实际演示成品（演示成品在阶段5制作）

---

## 一、演示版核心目标

> 向PMO/管理层展示：给定真实项目数据，系统能生成什么样的摘要，以及这个摘要如何支持审阅决策。

---

## 二、演示对象

| 角色 | 关注点 |
|------|-------|
| PMO负责人 | 摘要格式是否符合工作习惯？判断口径是否合理？ |
| 管理层 | 结论是否清晰？风险是否到位？是否可直接使用？ |
| PMO执行人员 | 操作成本有多高？填报有多复杂？ |

---

## 三、演示路径

### 3.1 演示三步法

```
步骤1：展示输入
  → 给演示观众看一份真实格式的周报/台账（脱敏后）
  → 说明：我们是这样接收数据的
  
步骤2：展示输出
  → 展示对应的领导摘要JSON和文本摘要
  → 逐字段解释：这是怎么看出来的
  
步骤3：展示判断依据
  → 展示reasoning字段（facts_extracted + rules_applied + raw_signal）
  → 说明：每一个结论都可以人工复核
```

### 3.2 演示时间控制

| 步骤 | 建议时长 | 说明 |
|------|---------|------|
| 步骤1（输入） | 2分钟 | 展示样本，解释填报要求 |
| 步骤2（输出） | 3分钟 | 展示完整输出，逐字段解释 |
| 步骤3（依据） | 3分钟 | 展示reasoning，说明可复核性 |
| Q&A | 5-10分钟 | 回答观众问题 |

---

## 四、演示必须展示的输入

### 4.1 必须有的输入场景

| 场景 | 说明 |
|------|------|
| 场景A：正常无风险 | progress无滞后词，risk_points空 |
| 场景B：有风险需关注 | 有风险自述，需要说明判断逻辑 |
| 场景C：建议升级 | escalation=true或严重滞后 |

### 4.2 输入展示要求

- 原始输入（JSON格式，字段名可见）
- 每份输入要有**脱敏记录备注**（说明哪些字段被脱敏）

---

## 五、演示必须展示的输出

### 5.1 领导摘要（必须展示）

每份演示场景必须展示：
- JSON格式（字段结构可见）
- 文本摘要（可读版本）
- `reasoning.raw_signal`（判断依据，可人工复核）

### 5.2 内部测试产物（仅供PMO看）

| 产物 | 展示场景 | 展示给谁 |
|------|---------|---------|
| 事项总台账 | 格式B台账演示 | PMO负责人 |
| 风险清单 | 有风险自述的演示场景 | PMO负责人 |

---

## 六、演示必须讲清的边界

每次演示必须主动说明以下边界，**不得让观众自行理解**：

| 边界项 | 必须说明的内容 |
|--------|--------------|
| 边界1：人工审阅不可跳过 | 系统输出后必须人工审阅，不可全自动使用 |
| 边界2：判断依据可查 | reasoning字段记录了原始关键词，任何人都可复核 |
| 边界3：系统不修正输入 | 输入错则输出错，系统不对数据做修正 |
| 边界4：规则有限 | 目前只有5条核心规则，无法覆盖所有复杂场景 |
| 边界5：升级是建议 | escalation_recommended是建议，最终由人工决定 |
| 边界6：不替代人决策 | 系统提供参考，不替代管理层做决策 |

---

## 七、演示与MVP的区别

| 对比项 | 演示版 | MVP |
|--------|--------|-----|
| 目标 | 可讲可看可验证 | 可用可靠可运行 |
| 数据 | 脱敏真实样本 | 真实试点数据 |
| 场景数 | 3个固定场景 | 不限场景 |
| 交互 | 单次展示，无反馈 | PMO持续使用，有反馈 |
| 环境 | 本地脚本 | API服务（可选） |
| PMO反馈 | 口头/书面均可 | 书面，正式 |

## 八、演示素材清单

### 8.1 演示前必须准备好的素材

| # | 素材 | 状态 | 说明 |
|---|------|------|------|
| M1 | 演示样本A（正常无风险） | 待准备 | S1级，≥1份 |
| M2 | 演示样本B（有风险需关注） | 待准备 | S1级，≥1份 |
| M3 | 演示样本C（建议升级） | 待准备 | S1级，≥1份 |
| M4 | 每份样本的预期输出JSON | 待生成 | 阶段4实现后生成 |
| M5 | 演示PPT/文档（可选） | 可选 | 不用PPT亦可纯演示 |

### 8.2 演示脚本模板

```
【开场】1分钟
"我们今天演示的是：给定一份项目周报，系统如何在30秒内生成标准化摘要。"

【步骤1：展示输入】2分钟
"这是PMO提交的一份周报（脱敏后），包含项目名称、承诺事项、当前进展、风险点。"

【步骤2：展示输出】3分钟
"系统在10秒内生成了这份领导摘要，包括状态等级、风险判断、建议动作。"

【步骤3：展示判断依据】3分钟
"每一个结论背后都有原始关键词支撑，这是触发'关注'风险等级的原始信号……"

【边界说明】2分钟（主动说，不等提问）
"需要说明几点：①人工审阅不可跳过 ②系统不修正输入错误 ③目前只支持5条核心规则……"

【Q&A】5-10分钟
```

## 九、演示准备检查清单

**演示前，Agent必须确认：**

- [ ] M1/M2/M3三份样本已通过S1验收
- [ ] 三份样本覆盖了"正常/关注/升级"三种场景
- [ ] 每份样本的预期输出已由人工确认
- [ ] 演示脚本已演练过（时长控制在12分钟以内）
- [ ] reasoning字段可逐条解释
- [ ] 边界说明（6条）已准备
- [ ] 备用Q&A已准备（如：为什么不接飞书？为什么要人工审阅？）
