# 真实影子运行触发规则

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4M《扩大评审执行与真实样本接入启动》

---

## 一、触发规则总则

**唯一触发条件：样本到位 + 校验通过 + 人工确认流程已确认**

三个条件必须同时满足，缺一不可。

---

## 二、触发条件明细

| # | 条件 | 状态 | 说明 |
|---|------|------|------|
| T1 | 真实周报样本到位（≥ 3期）| ❌ 未满足 | 业务方尚未提供 |
| T2 | 样本校验通过（《真实样本接入校验清单》全部通过）| ❌ 未满足 | 待样本到位后执行 |
| T3 | 人工确认对接人已指定 | ❌ 未满足 | 待业务方确认 |

---

## 三、触发决策树

```
样本到位检查
    │
    ├── 否 → 退回业务方，说明不到位原因
    │
    └── 是 → 执行《真实样本接入校验清单》
                  │
                  ├── 全部通过 → 执行《真实影子运行前检查表》
                  │                      │
                  │                      ├── 全部通过 → 【触发影子运行】
                  │                      │
                  │                      └── 任一不通过 → 退回业务方
                  │
                  └── 任一不通过 → 退回业务方，说明不合格项
```

---

## 四、如果样本不合格

| 不合格类型 | 处置 |
|-----------|------|
| 格式问题（JSON解析失败）| 退回，要求重新提供符合格式的样本 |
| 字段缺失 | 退回，说明缺失字段，要求补充 |
| 项目名未脱敏 | 退回，要求脱敏后重新提供 |
| 内容为空（无事项）| 退回，要求提供有实际内容的周报 |

---

## 五、触发后动作

| 步骤 | 动作 | 执行人 |
|------|------|--------|
| 1 | 《真实影子运行前检查表》全部通过 | 技术侧 |
| 2 | 正式记录触发时间（《真实样本接入启动记录》更新）| 技术侧 |
| 3 | 执行影子运行（按SOP）| 技术侧 |
| 4 | 业务方人工对照验证 | 业务侧 |
| 5 | 差异率评估，结论记入登记表 | 双方 |
