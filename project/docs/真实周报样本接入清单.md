# 真实周报样本接入清单

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4L《扩大评审准备与真实试运行待命》

---

## 一、业务方需要提供什么

### 1.1 最小可启动样本标准

| 属性 | 要求 | 说明 |
|------|------|------|
| 样本数量 | ≥ 3期 | 连续或不连续均可 |
| 时间跨度 | 任意 | 最近即可 |
| 格式 | JSON | 结构化数据，字段见下 |
| 脱敏要求 | 至少项目名脱敏 | 其他字段可协商 |

### 1.2 样本字段要求

样本必须为 JSON 格式，包含以下关键字段（字段名可协商，但结构需对应）：

```json
{
  "report_id": "周报唯一标识",
  "report_period": "期间标识（如：0531-0604）",
  "start_date": "2025-06-01",
  "end_date": "2025-06-04",
  "report_date": "2025-06-05",
  "reporting_unit": "报送单位",
  "service_audience": "服务对象",
  "source_form": "原表单类型",
  "raw_text": "周报正文全文（重要）",
  "overview": "周报概述",
  "request_lines": ["申请事项行列表"],
  "alert_lines": ["风险/黄灯行列表"],
  "in_progress_count": "在施项目数"
}
```

**说明：**
- `request_lines` 和 `alert_lines` 是最关键字段，直接影响识别率
- `raw_text` 是兜底数据源，当 `request_lines` 为空时从中解析
- 如实际字段名与上述不同，请提供字段映射表

---

## 二、样本提供方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 文件传输 | 提供JSON文件，可加密压缩 | 少量样本 |
| 共享目录 | 指定共享目录，读取后删除 | 持续提供 |
| API接入 | 通过内部API拉取（需提前对接）| 频繁更新 |

---

## 三、脱敏要求

| 字段 | 脱敏要求 | 优先级 |
|------|---------|--------|
| 项目名 | 至少做别名替换（如"A项目"）| 必须 |
| 单位名 | 可保留真实名称 | 建议 |
| 人名 | 删除或替换 | 必须 |
| 具体金额/日期 | 可保留真实值 | 可选 |

---

## 四、样本接收流程

1. **业务方准备**：按上述格式准备样本，通知对接人
2. **样本接收**：读取样本，存入 `openclaw_unified_sample_pack/openclaw_unified_sample_pack/real_sample/` 目录
3. **格式确认**：检查字段完整性，反馈是否需要补充
4. **影子运行**：按SOP执行影子运行
5. **样本清理**：验证完成后删除原始样本（除非业务方同意保留）

---

## 五、对接人

| 角色 | 职责 |
|------|------|
| 样本提供方（业务方）| 提供样本，确认脱敏 |
| 样本接收方（技术侧）| 接收样本，执行影子运行 |
| 结果确认方（业务侧）| 人工对照验证，确认结果可接受 |
| 问题登记方（技术侧）| 记录差异项，追踪修正 |

---

## 六、样本接入检查清单

业务方在提供样本前，请确认：

- [ ] 样本数量 ≥ 3期
- [ ] 格式为 JSON
- [ ] 关键字段（request_lines / alert_lines / raw_text）完整
- [ ] 项目名已脱敏
- [ ] 对接人已指定
- [ ] 人工确认流程已确认
