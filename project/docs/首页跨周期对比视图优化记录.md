## 一、改了什么

在首页结论卡下方，新增「**M-001 五期进度趋势**」独立模块，对 W16~W20 五个推演样本的 M-001 指标（汽车零部件贷款余额净增）进行横向进度条可视化对比。

**具体改动**：
- 新增 HTML 区块：`#home-m001-trend`（`display:none`，推演样本时显示）
- 每期一行，包含：周期标签 / gap文字 / 当前值 / 横向进度条 / 目标线标记
- 五期进度条颜色按状态语义编码：红→橙→黄→绿
- 底部图例说明颜色含义和目标线
- 静态写入 W16~W20 五期数据（current/gap），不依赖 DATA 对象动态更新

**JS 联动**：
- `renderHomepage()` 新增 `isSimulation` 判断
- `display:none` ↔ `display:block` 切换，随样本切换实时响应

---

## 二、为什么这项优化最有价值

**跨周期验证**是判断数字员工价值的核心场景。

在没有任何实际生产数据的情况下，W16~W20 五期本身构成了一条完整的故事线：

> W16 有缺口 → W17 在改善 → W18 接近达标 → W19 超额完成 → W20 持续稳定

这是最直观展示"数字员工可以跟踪趋势、预判风险"能力的窗口。没有这个视图，业务方看到 W16 的缺口结论，只能得到"有问题"的信息；有了五期趋势对比，还能看到"在好转"。**一个图表，同时说明"问题是什么"和"趋势是什么"**。

---

## 三、优化后能帮助用户看懂什么

1. **趋势是否在改善**：W16→W20 五期，颜色从红到绿，趋势一目了然
2. **当前离目标多远**：gap文字（缺X亿 / 超X亿）快速定位差距
3. **哪个周期是转折点**：W18 是"从黄灯转绿灯"的临界期
4. **生产接入价值**：如果真实 W16 走同样的路径，用户可以提前介入，而不用等 W19 才知道

---

## 四、技术说明

- 目标值固定为 **15亿**（来自原始样本定义）
- 进度条最大刻度按 W20 最大值（17.1亿）设置为 100% 宽度
- W19/W20 超额部分延伸至 100% 宽度（视觉上超出目标线）
- 目标线（深色竖线）通过 `calc(100% * 15 / 17.1)` 定位在 W20 条内的正确位置
- 非推演样本（6BLIVE、5CR、5CS）不显示此区块，不干扰真实数据展示
