## 二、阶段边界

### 2.1 当前不是生产系统

**明确声明：当前版本不是生产系统。**

当前版本的定位：
- 最小可运行原型
- 演示版（供演示和反馈收集）
- 条件性实现（规则未完全冻结）

### 2.2 一期不做

| 不做的事 | 原因 |
|---------|------|
| 不接生产平台 | 一期目标验证判断能力，不做系统集成 |
| 不做平台化 | MVP前不扩展，做完最小闭环再考虑 |
| 不做大模型替换 | 规则引擎在一期够用，LLM是未来选项 |
| 不做全自动判断 | 一期原则：允许人工确认 |
| 不做数据自动采集 | 周报/台账人工提供，不接OA |

### 2.3 一期做

| 做的事 | 说明 |
|--------|------|
| 纯规则推理 | 确定性、可测试、可复核 |
| 最小闭环 | 输入→归一化→判断→输出，全链路打通 |
| 真实样本验证 | 3期真实周报数据，端到端测试 |
| 人工确认节点 | 输出后有人工审阅节点 |
| 规则追溯机制 | 每项判断附规则名称和原始信号 |

**provisional意味着：判断逻辑是正确的，但尚未获得官方背书。**

## 四、责任边界

| 责任方 | 责任内容 |
|--------|---------|
| Agent（数字员工） | 执行规则推理，生成结构化输出 |
| 管理者 | 审阅AI输出，有权确认、修改或推翻 |
| PMO | 提供术语口径和映射关系确认 |
| 管理层 | 提供输出格式确认，指定试点范围 |
| 用户（发起人） | 协调各方确认，推动试点启动 |

**数字员工不替代人的决策，输出是参考，人是最终决策者。**
