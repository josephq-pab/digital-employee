# 真实影子运行后续动作分支

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4Q-T《真实样本影子运行准备》

---

## 分支概览

| 分支 | 条件 | 下一阶段 |
|------|------|---------|
| 分支A：影子运行通过 | 无阻断差异 + 完整准确 | 进入下一阶段 |
| 分支B：影子运行小修 | 无阻断差异 + 有可接受差异 | 记录后继续 |
| 分支C：影子运行暂停 | 有阻断差异 | 修复后重新运行 |

---

## 分支A：影子运行通过

**条件：** 无阻断性差异，抽取完整且准确

**最小动作集：**
1. 登记运行结果（样本ID + 抽取数量 + 结论）
2. 更新《真实样本受理记录》状态为"已运行"
3. 整理运行产物归档
4. 判断是否满足下一阶段进入条件

**下一步：** 进入阶段4Q-U《影子运行结果评审》

---

## 分支B：影子运行小修

**条件：** 无阻断性差异，但存在可接受差异

**最小动作集：**
1. 登记所有差异（类型+描述+是否阻断）
2. 判断差异是否在适配器层可修复
   - 是 → 记录修复方案 → 适配器打补丁
   - 否 → 记录为观察项
3. 记录后继续，不阻塞推进
4. 重复运行以验证修复效果（如已打补丁）

**修复后：**
- 补丁有效 → 重新运行 → 重新判断
- 补丁无效 → 降级为观察项，继续推进

**下一步：** 差异修复后重新运行 → 重新判断

---

## 分支C：影子运行暂停

**条件：** 存在阻断性差异（B1~B5）

**最小动作集：**
1. 立即停止，登记阻断项
2. 分析阻断根因：
   - 适配器层问题 → 修复适配器
   - 归一化层问题 → 修复归一化
   - 样本格式问题 → 退回业务方补充
3. 修复后重新执行启动卡
4. 重新判断

**阻断项修复责任：**
| 阻断项 | 责任方 | 动作 |
|--------|--------|------|
| B1/B2 | 适配器/normalizer | 修复映射规则 |
| B3 | 规则引擎 | 修复判断逻辑 |
| B4 | 工程 | 修复脚本/环境 |
| B5 | 样本/适配器 | 确认样本+适配器 |

**下一步：** 阻断项修复后 → 重新执行启动卡

---

## 通用动作

所有分支必须执行：
1. 更新《真实样本校验结果模板》中的运行记录
2. 归档本次运行的所有输出文件
3. 记录本次运行的样本版本哈希
