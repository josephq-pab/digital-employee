## 一、当前样本台账格式（参考基准）

**样本路径：** `samples/samples/project_ledger/project_ledger_latest3.json`

**数据结构：**
```json
{
  "records": [
    {
      "record_id": "ledger-31",
      "worksheet": "主表",
      "row_number": 31,
      "sequence": "1.3.2",
      "project_type": "重点项目",
      "project_name": "制定贷款促投放提升方案，有序压降票证规模",
      "record_type": "主体",
      "objective_or_requirement": "...",
      "owner_department": "某分行",
      "due_date": "2025-12-31",
      "progress_by_period": {
        "1222-1226": {
          "this_period_progress": "有序推进，10月投放X亿",
          "status": "进行中",
          "has_risk": false,
          "risk_note": ""
        }
      },
      "subitems": [
        {
          "subitem_id": "ledger-31-1",
          "content": "子事项内容",
          "status": "进行中"
        }
      ]
    }
  ]
}
```

**必要字段：**
| 字段路径 | 类型 | 说明 | 是否可缺 |
|---------|------|------|---------|
| records[].record_id | string | 台账记录唯一ID | ❌ 不可 |
| records[].project_name | string | 项目名称 | ❌ 不可 |
| records[].progress_by_period | object | 各期进展 | ❌ 不可 |
| records[].progress_by_period.{period}.status | string | 状态：进行中/黄灯/绿灯/结项 | ❌ 不可 |
| records[].subitems | array | 子事项列表 | ✅ 可（可为空） |

## 四、状态字段标准化（关键）

**样本台账的status值标准：**
- `进行中`（绿色，正常）
- `green`（绿色，正常）
- `yellow`（黄灯，有风险）
- `red`（红灯，严重问题）
- `结项`（项目结束）

**真实台账可能的状态值：**
| 值 | 是否需要转换 |
|------|------------|
| green/yellow/red | ✅ 直接兼容 |
| 进行中/进行 | 需映射为green |
| 已完成/已完成 | 需映射为green |
| 延期/有风险 | 需映射为yellow |
| 严重/阻断 | 需映射为red |
| 结项/关闭 | 需映射为green（结项逻辑单独判断） |

**若真实台账status值不标准：**
- 需要在normalizer中增加映射逻辑
- 这属于"轻微代码调整"，不违反冻结版约束（因为是数据兼容层，不是功能修改）
