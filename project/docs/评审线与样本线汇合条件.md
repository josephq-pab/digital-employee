| 线路 | 当前状态 | 距离汇合还差什么 |
|------|---------|----------------|

---

## 二、汇合条件

### 2.1 两条线完全汇合（评审通过 + 样本合格）

| 条件 | 说明 |
|------|------|
| 评审结论 | 分支A（通过）或分支B（先影子运行）|
| 样本状态 | 校验通过（全部6项）|
| 汇合后进入 | 正式影子运行 |

---

### 2.2 只有评审线完成

| 条件 | 说明 |
|------|------|
| 评审结论 | 分支A（通过）|
| 样本状态 | 未到位或未校验 |
| 下一步 | 继续等待样本到位，样本到位后独立执行校验和影子运行准备 |

---

### 2.3 只有样本线完成

| 条件 | 说明 |
|------|------|
| 评审结论 | 待评审 |
| 样本状态 | 校验通过 |
| 下一步 | 影子运行可执行，但结论待评审确认；可先做影子运行准备，结论确认后正式触发 |

---

## 三、汇合后的最小动作集

| 场景 | 最小动作 |
|------|---------|
| 两条线同时汇合 | 评审通过 + 样本合格 → 执行影子运行前最终检查 → 触发影子运行 |
| 评审先完成 | 继续等样本，样本到位后校验 → 校验通过 → 触发影子运行 |
| 样本先完成 | 可先执行校验，校验通过后通知评审方；评审通过后直接触发影子运行 |

---

## 四、两条线的独立性

| 维度 | 说明 |
|------|------|
| 互不阻塞 | 评审线和样本线独立执行，互不等待 |
| 并行处理 | 评审会议期间，样本可同步到位和校验 |
| 结果独立 | 评审结论与样本质量无关 |
