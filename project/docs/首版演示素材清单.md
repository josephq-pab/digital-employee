## 三、演示讲解口径

### 演示开场（1分钟）

```
"我们用真实的PMO周报数据，运行了数字员工原型。
输入：某总行PMO周报（3期）+ 项目台账（16条）
输出：领导摘要 + 事项台账 + 风险清单
全链路约3秒完成。"
```

### 演示核心（3分钟）

```
Step 1：看领导摘要
  → 11事项，7个申请结项，4个黄灯关注，0个需领导介入

Step 2：看黄灯事项的判断依据
  → 每条黄灯都附了判断规则名称和原始信号

Step 3：看事项台账的映射情况
  → 4项已映射到台账，8项未映射（行业维度差异）

Step 4：看风险清单
  → 按风险等级排序，可直接看到需要关注的事项

Step 5：强调"三层分离"
  → 事实（原始数据）→ 判断（规则名称+信号）→ 建议（动作）
  → 每层都可人工复核"
```

## 四、已知质量问题（演示时需说明）

| 问题 | 演示时如何处理 |
|------|--------------|
| 黄灯判断为"低风险" | 说明：这是engine.py未处理黄灯，需修复适配层（QUAL-01） |
| 结项申请出现在风险清单 | 说明：这是归因错误，修复后不出现（QUAL-02） |
| 责任人显示[待关联] | 说明：周报无owner字段，用台账替代（provisional） |
| 8项未映射 | 说明：行业vs项目维度差异，属正常结构差异 |

1. 新输出文件自动生成（带时间戳）

3. 建议保留最新一次运行的输出作为主演示素材
