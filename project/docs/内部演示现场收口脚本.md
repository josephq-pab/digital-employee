# 内部演示现场收口脚本

**版本**：v1.0  
**日期**：2026-04-09  
**阶段**：阶段4S-C《内部演示执行与会后决策收口》  
**用途**：演示主持人在最后5分钟必须问的3个问题和对应口径

---

## 一、收口时机

**开始时间**：演示第26分钟  
**结束时间**：第30分钟  
**执行人**：演示主持人  
**目的**：让参会人对3个问题有明确口头意向，并当场记录到《内部演示会后结论模板.md》

---

## 二、收口三问

### 问题1：是否认可当前输出结构

**主持人问题**：
> "以上是我们系统的完整输出——领导摘要、事项总台账、风险清单三件套。请各位判断：这套输出结构是否认可，可以用于辅助管理决策？"

**可选结论**：同意 / 有条件同意（需说明修改要求）/ 暂缓（需说明阻断问题）

**主持人追问（如参会人犹豫）**：
> "如果有修改建议，请直接说，我们记录下来。如果整体方向认可，可以选'有条件同意'，列出最关键的1-2项修改要求。"

**结论记录**：当场记录到《内部演示会后结论模板》结论1栏。

---

### 问题2：是否同意真实样本到位后进入影子运行

**主持人问题**：
> "如果FB-001或S-001真实样本到位，并且样本校验通过，是否同意直接启动影子运行——用真实数据跑完整链路？"

**可选结论**：同意 / 有条件同意（需说明前置条件）/ 暂缓（需说明阻断事项）

**主持人追问（如参会人犹豫）**：
> "如果同意，请说'同意'。如果有前置条件，请说'有条件同意'，并列出最关键的1项条件。"

**结论记录**：当场记录到《内部演示会后结论模板》结论2栏。

---

### 问题3：是否同意继续推进扩大评审准备

**主持人问题**：
> "在影子运行之外，是否同意同步推进扩大评审的准备工作——包括评审材料整理和评审会议安排？"

**可选结论**：同意 / 有条件同意（需说明并行条件）/ 暂缓（需说明原因）

**主持人追问（如参会人犹豫）**：
> "如果同意，请说'同意'。如果有并行条件，请说'有条件同意'。如果觉得还不是时机，请说'暂缓'并说明原因。"

**结论记录**：当场记录到《内部演示会后结论模板》结论3栏。

---

## 三、收口话术总结（主持人用）

```
好的，以上就是今天演示的全部内容。现在需要各位对3个问题做决策：

第1个问题：这套"领导摘要+事项台账+风险清单"的结构，各位是否认可？

第2个问题：如果真实样本到位，是否同意直接进入影子运行？

第3个问题：是否同意同步推进扩大评审准备？

每项请告诉我：同意 / 有条件同意（附带条件）/ 还是暂缓（说明原因）。

我现在记录下来，我们当场形成结论。
```

---

## 四、收口完成标准

```
□ 问题1结论已记录（同意/有条件同意/暂缓）
□ 问题2结论已记录（同意/有条件同意/暂缓）
□ 问题3结论已记录（同意/有条件同意/暂缓）
□ 管理层代表对3项结论有口头确认
□ 演示主持当场告知分支走向（根据3项结论）
□ 演示主持告知：详细反馈记录在《内部演示反馈登记表.md》
```

---

## 五、注意事项

| 注意事项 | 说明 |
|---------|------|
| 不诱导结论 | 不预设"您应该同意吧"等诱导性话术 |
| 不跳过问题 | 3个问题必须全部问完 |
| 不代替决策 | 如果决策人说"我需要再想想"，记录为"暂缓"，不强迫当场决策 |
| 有条件同意必须记条件 | 不能只记"有条件同意"，必须记具体条件 |

---

_本文档为演示主持人专用脚本，确保3个问题全部问到、记到、不诱导。_
