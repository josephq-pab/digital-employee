# 冒烟测试执行手册

**版本：** v1.0
**用途：** 指导Agent执行冒烟测试，包含执行顺序、记录方式、失败归因
**执行入口：** `bash scripts/run_smoke.sh`

---

## 一、执行前准备

### 1.1 环境检查

| 检查项 | 命令/方式 | 通过标准 |
|--------|---------|---------|
| Python版本 | `python3 --version` | ≥3.8 |
| 工作目录 | `pwd` | 在project根目录 |
| 样本目录 | `ls data/samples/s0/` | 至少存在一个样本 |
| 测试用例 | `ls data/test_cases/` | smoke_tests.json存在 |
| engine.py | `ls app/engine.py` | 存在 |

### 1.2 执行前确认清单

- [ ] T01-T05全部完成
- [ ] app/engine.py 可执行（无语法错误）
- [ ] S0样本在 data/samples/s0/
- [ ] smoke_tests.json 在 data/test_cases/
- [ ] 运行过一次 `python3 -c "import app.engine"` 无报错

---

## 二、执行顺序

### 2.1 执行主流程

```bash
bash scripts/run_smoke.sh
```

### 2.2 流程说明

```
执行run_smoke.sh
    │
    ▼
加载smoke_tests.json中的测试用例
    │
    ▼
逐条执行测试用例
    │
    ├── TC01-TC05（核心规则）
    │    └─ 任意失败 → 立即停止，输出失败信息
    │
    └── TC06-TC20（扩展用例，如执行）
         └─ 失败 → 记录到结果文件，继续
    │
    ▼
生成eval/smoke_test_results/{date}.json
    │
    ▼
显示通过率统计
```

### 2.3 测试用例加载顺序

| 顺序 | 用例ID | 分类 | 失败处理 |
|------|--------|------|---------|
| 1 | TC01 | 核心规则 | 立即停止 |
| 2 | TC02 | 核心规则 | 立即停止 |
| 3 | TC03 | 核心规则 | 立即停止 |
| 4 | TC04 | 核心规则 | 立即停止 |
| 5 | TC05 | 核心规则 | 立即停止 |
| 6 | TC06 | 扩展用例 | 记录，继续 |
| ... | ... | ... | ... |
| N | TC20 | 扩展用例 | 记录，继续 |

## 五、失败归因

### 5.1 归因树

```
测试失败
  │
  ├── TC01-TC05失败
  │    │
  │    ├── risk_level不一致
  │    │    └─ 规则逻辑错误 → 立即修复rule_engine.py
  │    │
  │    ├── escalation不一致
  │    │    └─ escalation逻辑错误 → 立即修复rule_engine.py
  │    │
  │    └── overall不一致
  │         └─ overall逻辑错误 → 立即修复rule_engine.py
  │
  └── TC06-TC20失败
       │
       ├── facts_extracted为空
       │    └─ FactExtractor错误 → 记录，延后修复
       │
       ├── 边界条件处理错误
       │    └─ 边界处理缺陷 → 记录，延后修复
       │
       └── 规则误判
            └─ 规则覆盖不足 → 记录，延后修复
```

### 5.2 归因决策表

| 失败模式 | 归因 | 动作 |
|---------|------|------|
| TC01-TC05 risk_level错误 | 核心逻辑错误 | **立即修复** |
| TC01-TC05 escalation错误 | 核心逻辑错误 | **立即修复** |
| TC01-TC05 overall错误 | 核心逻辑错误 | **立即修复** |
| TC06-TC20失败 | 边界处理缺陷 | 记录，延后修复 |
| facts为空 | FactExtractor错误 | 记录，延后修复 |
| 规则未触发 | patterns.py关键词缺失 | 记录，延后修复 |

## 六、回退策略

### 6.1 TC01-TC05失败回退

```
TC01-TC05失败
    │
    ▼
定位错误模块（InputParser/FactExtractor/RuleEngine）
    │
    ▼
修复代码
    │
    ▼
重新执行run_smoke.sh
    │
    ▼
直到TC01-TC05全部通过
    │
    ▼
方可继续执行TC06-TC20或进入下一阶段
```

### 6.2 TC06-TC20失败回退

```
TC06-TC20失败
    │
    ▼
记录失败原因到registry
    │
    ▼
标记为"待修复"
    │
    ▼
继续执行剩余用例
    │
    ▼
完成后输出报告
    │
    ▼
在阶段5前统一修复标记的用例
```

## 七、人工核验点

### 7.1 必须人工核验的节点

| 节点 | 核验内容 | 核验方式 |
|------|---------|---------|
| TC01-TC05结果 | risk_level/escalation/overall是否与预期一致 | 人工对比输入/输出 |
| 失败用例归因 | 归因结论是否正确 | 人工审核 |
| 最新结果 | eval/smoke_test_results/latest.json | Agent读取 |

### 7.2 人工核验流程

```
冒烟测试执行完成
    │
    ▼
Agent读取latest.json
    │
    ▼
输出总结报告（含通过率/失败列表/归因）
    │
    ▼
人工核验失败用例（如有）
    │
    ▼
确认归因结论
    │
    ▼
决定：立即修复 / 延后修复 / 继续下一阶段
```
