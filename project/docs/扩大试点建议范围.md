# 扩大试点建议范围

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4L《扩大评审准备与真实试运行待命》

---

## 一、建议扩大的范围

### P0（立即可扩，建议扩大评审时确认）

| 范围 | 说明 | 当前状态 |
|------|------|---------|
| 评审参与方扩大 | 纳入PMO其他成员、管理层代表 | 待确认 |
| 验证样本扩大 | 用更多期历史周报做对照验证（当前16期→24期+）| 技术上可执行 |
| 评审深度加深 | 核对系统输出与人工判断的一致性比例 | 待安排 |

### P1（评审通过后可扩，需真实样本）

| 范围 | 说明 | 前置条件 |
|------|------|---------|
| 真实样本影子运行 | 用3期真实周报做影子验证 | 业务方提供样本 |
| 适配器真实数据验证 | 验证适配器在真实周报上的表现 | 样本到位 |
| 输出格式确认 | 管理层对真实数据输出格式进行确认 | 影子运行后 |

### P2（条件满足后扩，需影子运行通过）

| 范围 | 说明 | 前置条件 |
|------|------|---------|
| PMO重点项目扩大 | 从当前试点项目扩展到更多重点项目 | P1完成 |
| 输出提交频率提升 | 从每周一次提升到按需触发 | P1完成 + 管理层确认 |

---

## 二、明确不建议扩大的范围

| 范围 | 原因 |
|------|------|
| 直接接入生产系统 | 一期试点边界明确，不超范围 |
| 替换人工判断 | 系统输出仍需人工确认，一期不变 |
| 扩大场景到其他业务线 | 一期仅限PMO重点项目 |
| 开放API给第三方调用 | 一期内不考虑 |
| 改变组织流程 | 一期仅辅助，不改变现有流程 |

## 四、一期试点边界声明

**明确：一期仅在 PMO 重点项目范围内做辅助生成，不改变现有流程，不接入生产系统，不替换人工判断。**

此边界在项目启动时已确定，本阶段重申并确认维持此边界不变。
