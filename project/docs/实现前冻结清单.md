# 实现前冻结清单

**版本：** v1.0
**用途：** 明确哪些东西在确认后必须冻结，哪些可以边做边调

---

## 一、冻结分类总览

| 分类 | 含义 | 变更方式 |
|------|------|---------|
| **必须冻结** | 确认后立即锁定，实现阶段不得改动 | 需提出正式变更申请，≥2人确认 |
| **可试点期微调** | 试点运行中可根据反馈小幅调整 | 需PMO书面确认，不走变更流程 |
| **暂不冻结** | 边做边定，不在实现前固化 | 无冻结要求 |

## 五、冻结管理流程

### 5.1 变更申请流程

```
提出变更申请
    │
    ▼
评估影响范围（影响哪些模块/文档）
    │
    ▼
冻结项 → 需≥2人确认（PMO+管理层）
微调项 → 需PMO书面确认
    │
    ▼
变更批准 → 更新对应文档 → 通知相关方
变更拒绝 → 维持原状
```

### 5.2 试点期微调流程

```
Agent提出微调建议
    │
    ▼
判断是否属于微调范围
    │
    ├── 是 → PMO确认 → 执行微调 → 记录到registry.json
    └── 否 → 走变更申请流程
```
