## 二、任务包详细说明

### T01：样本准备任务包

**目标：** 收集≥1份可用于开发和验证的真实样本，完成脱敏

**输入：** PMO提供的真实周报/月报/台账

**输出：**
- `data/samples/real_01_weekly.json`（脱敏后）
- 脱敏记录（记录哪些字段被修改）

**完成标准：**
- [ ] 至少1份样本到位
- [ ] 必填字段完整
- [ ] JSON格式合法
- [ ] 脱敏后逻辑判断点仍然存在
- [ ] Agent未接触过原始数据（由用户自行脱敏）

**前置依赖：** 无

**阻断风险：** 高（无样本则后续无法验证）

---

### T02：输入解析任务包

**目标：** 实现InputParser模块，支持格式A（周报/月报）

**输入：** `app/models.py`（数据模型定义）

**输出：**
- `app/input_parser.py`（格式A解析器）
- `tests/test_input_parser.py`（单元测试）

**完成标准：**
- [ ] JSON格式A可解析为InternalRecord
- [ ] 空字段按默认值处理
- [ ] 字段过长截断且标记truncated=true
- [ ] 字段缺失时报错
- [ ] 单元测试覆盖主要路径

**前置依赖：** T01（样本准备）

**可并行：** T03

---

### T03：事项归并任务包

**目标：** 实现FactExtractor模块，从InternalRecord提炼事实清单

**输入：** `app/input_parser.py`（已完成）

**输出：**
- `app/fact_extractor.py`（事实提取器）
- `tests/test_fact_extractor.py`（单元测试）

**完成标准：**
- [ ] promised_items原文透传
- [ ] current_progress原文透传
- [ ] risk_points非空时透传
- [ ] escalation=true时有明确标识
- [ ] 单元测试覆盖所有路径

**前置依赖：** T01（样本准备）

**可并行：** T02

---

### T04：风险识别任务包

**目标：** 实现RuleEngine模块，5条核心规则全部实现

**输入：** `app/fact_extractor.py` + 规则模式库

**输出：**
- `app/rule_engine.py`（规则引擎）
- `app/patterns.py`（关键词模式库）
- `tests/test_rule_engine.py`（单元测试）

**完成标准：**
- [ ] TC01-TC05全部规则实现
- [ ] 严重滞后先于一般滞后匹配（优先级正确）
- [ ] escalation=true直接触发高风险
- [ ] 每条规则记录rules_applied
- [ ] 原始关键词记录raw_signals
- [ ] 单元测试覆盖5条核心规则

**前置依赖：** T02 + T03

**串行说明：** 必须等T02和T03都完成才能开始

---

### T05：人工确认任务包

**目标：** 定义人工审阅流程（在原型中不实现，只定义）

**输入：** T04输出（SummaryGenerator前的中间结果）

**输出：**
- 人工审阅流程说明（写入《一期原型技术方案》或新增文档）
- 审阅要点清单（供用户使用）

**完成标准：**
- [ ] 明确审阅内容（事实/判断/建议/升级）
- [ ] 明确审阅时机（系统输出后，正式使用前）
- [ ] 明确审阅角色（管理层或PMO指定负责人）
- [ ] 审阅流程已文档化

**前置依赖：** T04

**说明：** 本阶段只定义流程，不实现自动化审阅环节

---

### T06：摘要生成任务包

**目标：** 实现SummaryGenerator模块，生成领导摘要

**输入：** `app/rule_engine.py`（已完成）

**输出：**
- `app/summary_generator.py`（摘要生成器）
- `tests/test_summary_generator.py`（单元测试）

**完成标准：**
- [ ] 生成UUID作为report_id
- [ ] generated_at为ISO8601格式
- [ ] progress_text截断至200字以内
- [ ] suggested_actions非空列表
- [ ] escalation_indicator正确区分
- [ ] 输出符合OUTPUT_SCHEMA

**前置依赖：** T04

---

### T07：冒烟测试任务包

**目标：** 完成TC01-TC05，run_smoke.sh通过

**输入：** T02+T03+T04+T06全部完成

**输出：**
- `data/test_cases/smoke_tests.json`（测试用例文件）
- `scripts/run_smoke.sh`（冒烟测试脚本，可更新）
- `tests/test_smoke.py`（冒烟测试执行脚本）
- `eval/smoke_test_results.json`（测试结果存档）

**完成标准：**
- [ ] TC01-TC05 100%通过
- [ ] 每个失败用例输出实际值vs预期值
- [ ] 测试结果写入eval目录（含时间戳）
- [ ] run_smoke.sh可独立运行

**前置依赖：** T02+T03+T04+T06全部完成

**可并行：** T08

---

### T08：演示素材任务包

**目标：** 准备可在评审时演示的材料

**输入：** T06（摘要生成已完成）

**输出：**
- `scripts/run_dev.sh`（更新为含演示流程的脚本）
- 演示用的脱敏样本数据（已准备）
- 演示流程说明（写入RUN_DEMO.md）

**完成标准：**
- [ ] bash run_dev.sh 可完整演示从输入到输出的闭环
- [ ] 演示时长≤30秒（本地运行）
- [ ] 演示结果可读、可解释
- [ ] 演示流程说明文档化

**前置依赖：** T06

**可并行：** T07

## 四、并行可行性分析

| 任务包组合 | 并行可行性 | 理由 |

## 五、关键路径（决定项目工期的路径）

```
T01(样本) → T02(InputParser) → T03(FactExtractor) → T04(RuleEngine) → T06(Summary) → T07(冒烟测试)
```

关键路径长度：6个任务包串行
任何关键路径上的任务延迟，都会直接延长整体工期
