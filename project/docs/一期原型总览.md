# 一期原型总览

**适用阶段：** 阶段1结束后，阶段2起，任何Agent接手的入口文档

---

## 一、项目是什么

**组织执行力数字员工（一期）**：给定一个重点项目的结构化周报/月报，输出标准化的执行状态摘要，供管理层审阅决策。

> 核心价值：用结构化替代自由文本，用规则推理替代人工汇总，用可复核的判断依据替代黑箱结论。

---

## 二、一期只解决一个问题

> **单项目追踪问题**：PMO或项目责任人提交周报/月报 → 系统生成标准化摘要 → 人工审阅后使用。

不解决：多项目横向对比、平台自动采集、自动升级推送、前端展示。

---

## 三、完整链路（五步）

```
步骤1：输入        步骤2：处理        步骤3：人工     步骤4：输出        步骤5：使用
               ┌──────────────────┐
人工填报       │  1. 字段解析      │                              管理层晨会审阅
周报/月报/台账 → 2. 事实提取       → 人工审阅 → 结构化摘要JSON → PMO存档追踪
（JSON格式）    │  3. 规则推理      │ （不可跳过）   +文本摘要    风险清单同步
               │  4. 风险判定      │
               │  5. 建议生成      │
               │  6. 依据标注      │
               └──────────────────┘
                        ↑
               纯规则推理（无LLM）
               处理过程无人工干预
```

---

## 四、一期的三个产出阶段

| 阶段 | 目标 | 形态 | 验证方式 |
|------|------|------|---------|
| 原型 | 验证逻辑闭环可跑通 | 本地脚本+JSON | 冒烟测试100%通过 |
| 演示版 | 验证业务价值可讲可看 | 真实样例+输出 | PMO/管理层审阅反馈 |
| MVP | 在PMO真实试点中可用 | API服务+真实数据 | 书面反馈接受/不接受 |

详见《原型_冒烟测试_演示版_MVP关系说明.md》

---

## 五、一期不做什么（边界）

| 类别 | 不做 |
|------|------|
| 数据源 | 不接OA/PMIS；仅支持人工填报 |
| 自动化 | 不做自动催报/通知/升级 |
| 展示 | 不做Dashboard/图表/前端 |
| 聚合 | 不做多项目横向对比/排名 |
| 决策 | 不替代人工做升级决定 |
| 生态 | 不做多Agent协作 |

---

## 六、核心约束（铁律）

1. **人工审阅不可跳过** — 系统输出必须经人工审阅方可用
2. **人工填报为唯一数据源** — 不做系统自动采集
3. **先有样本数据，后有开发** — 阶段4进入前必须有≥3份真实样本
4. **判断必须附依据** — 每条结论必须附规则名称和原始关键词

## 八、如何判断项目在正轨上

**以下任一情况发生，项目已偏离：**
- 开始讨论平台接入（OA/PMIS等）
- 开始做Dashboard或前端界面
- 没有真实样本数据就开始开发
- 判断结论不附依据
- 绕过人工审阅直接作为正式结论使用
