# 从并行运行到局部替代的判断条件

**版本**：v1.0  
**日期**：2026-04-09  
**阶段**：阶段5A《并行运行与价值验证》  
**用途**：明确什么条件下可以从"并行参考"进入"局部替代"

---

## 一、当前阶段定位

| 阶段 | 数字员工角色 | 人工状态 |
|------|------------|---------|
| **并行参考**（当前） | 输出供参考 | 人工做最终判断 |
| **局部替代**（目标） | 部分场景直接采用 | 人工做最终确认 |
| **规则固化**（远期） | 规则判断全自动 | 人工处理例外 |

**当前阶段**：并行参考（不替代人工）

---

## 二、进入"局部替代"的条件

当以下条件**全部满足**时，可以将该类事项从"并行参考"升级为"局部替代"：

### 条件1：连续稳定

| 要求 | 具体标准 |
|------|---------|
| 运行周期 | 连续4周以上稳定运行 |
| 偏差率 | 连续4周偏差率均低于10% |
| 稳定性 | 连续4周无系统性故障或输出错误 |

### 条件2：明显减负

| 要求 | 具体标准 |
|------|---------|
| 时间节省 | 该类事项处理时间减少50%以上 |
| 工作简化 | 该类事项不再需要人工逐条判断 |
| 频率 | 该类事项在日常工作中出现频率较高（每周多次） |

### 条件3：明显管理价值

| 要求 | 具体标准 |
|------|---------|
| 决策支持 | 管理层在周报/汇报中直接引用系统输出 |
| 风险识别 | 系统识别出的人工未注意到的风险项≥1个 |
| 一致性 | 不同参会人对同一事项的状态判断通过系统输出一致化 |

### 条件4：偏差可控

| 要求 | 具体标准 |
|------|---------|
| 偏差类型 | 偏差以"部分偏差"和"口径偏差"为主，无"完全偏差" |
| 偏差处理 | 所有偏差都有记录并在下一周期改进 |
| 偏差原因 | 偏差原因可解释，不是系统性误判 |

---

## 三、升级决策流程

当某类事项满足上述4个条件时，按以下流程决策：

```
某类事项连续4周满足条件
    ↓
指定人员提出升级申请（附4周价值验证记录）
    ↓
业务方+PMO+管理层三方确认
    ↓
决策通过 → 更新该事项判断规则 → 进入"局部替代"
决策不通过 → 继续并行运行1周期后再评估
```

---

## 四、局部替代的范围管理

| 范围 | 说明 |
|------|------|
| **单事项升级** | 满足条件后，每次只升级一个具体事项/场景 |
| **不全局放开** | 某类升级不等于所有事项都升级 |
| **保留复核** | 局部替代后，人工仍保留最终确认权 |

---

## 五、哪些事项优先考虑升级

| 优先级 | 事项类型 | 理由 |
|--------|---------|------|
| P1 | 量化指标类（偏离计算） | 规则明确，不易主观偏差 |
| P2 | 结项申请判断类 | 条件明确，逻辑清晰 |
| P3 | 黄灯/逾期识别类 | 有明确时间节点标准 |
| P4 | 风险分级类 | 需要经验积累，逐步验证 |

---

## 六、降级条件

当已升级为"局部替代"的事项出现以下情况，立即降级回"并行参考"：

| 条件 | 说明 |
|------|------|
| 偏差率回升 | 连续2周偏差率超过20% |
| 负反馈 | 业务方或管理层明确反馈不可用 |
| 数据源变化 | 输入数据格式或来源发生变化 |
