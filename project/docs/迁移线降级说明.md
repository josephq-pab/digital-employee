# 迁移线降级说明

**版本：** v1.0
**日期：** 2026-04-15
**阶段：** 阶段6H

---

## 一、降级结论

**迁移执行准备主线降级，不再作为当前阶段的主线。**

---

## 二、降级原因

| # | 原因 |
|---|------|
| 1 | 当前环境是主开发与主完善环境，拥有AI辅助工具，修改成本低 |
| 2 | 迁移后将缺少高效AI开发工具，修改成本显著上升 |
| 3 | 当前系统仍在需要持续打磨阶段，过早迁移会导致反复返工 |
| 4 | 正确的节奏：先在当前环境打磨到冻结版，再迁移 |

## 四、迁移主线恢复条件

**只有达到以下条件后，才恢复迁移主线：**

1. ✅ 功能闭环：F1~F8 全部通过
2. ✅ 页面体验：E1~E5 全部通过 + E6无阻断
3. ✅ 数据口径：D1~D5 全部通过
4. ✅ 连续运行：S1完成（连续3期无P0）+ S2~S4 全部通过
5. ✅ 问题等级：P0=0 + P1=0 + P2≤2
6. ✅ 业务可接受度：B1~B4 全部通过

**详见：**《投产前冻结条件清单.md》
