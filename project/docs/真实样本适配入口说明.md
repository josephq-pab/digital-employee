## 一、真实样本进入系统的唯一入口

**入口路径：** `openclaw_unified_sample_pack/openclaw_unified_sample_pack/`

| 文件类型 | 输入路径 | 命名规范 |
|---------|---------|---------|
| 周报 | `normalized/weekly_reports/by_period/` | `YYYY-MM-DD_YYYY-MM-DD.json` |
| 台账 | `normalized/project_ledger/by_period/` | `YYYY-MM-DD_YYYY-MM-DD.json` |
| 指标 | `normalized/quant_metrics/by_period/` | `YYYY-MM-DD_YYYY-MM-DD.json` |

---

## 二、字段映射原则

| 统一样本包字段 | 真实样本可能名称 | 映射层 |
|---------------|----------------|--------|
| request_lines | requests.delay_requests / 延期申请 / 事项申请 | november_adapter |
| alert_lines | key_alerts / 黄灯预警 / 重点提示 | november_adapter |
| raw_text | 全字段结构化 / raw_text | november_adapter |
| report_meta.report_period | report_period / period / 期间 | november_adapter |
| project_name | 项目名称 / 项目名称（全称）| normalizer |

**映射原则：** 语义等效优先，字段名称不同时在适配器层处理。

---

## 三、差异拦截标准

| 差异类型 | 处理方式 | 是否阻断影子运行 |
|---------|---------|----------------|
| period 缺失或模糊 | 拦截，要求补充 | ✅ 阻断 |
| 无任何 request_lines 和 alert_lines | 拦截，要求确认样本完整性 | ✅ 阻断 |
| 项目名称含未脱敏信息 | 拦截，要求重新脱敏 | ✅ 阻断 |
| 字段命名不同但语义可映射 | 通过，适配器处理 | ❌ 不阻断 |
| 字段嵌套层级不同 | 通过，适配器处理 | ❌ 不阻断 |
| 量化指标列名不同 | 通过，适配器处理 | ❌ 不阻断 |
