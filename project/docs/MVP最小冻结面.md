## 一、核心原则

**必须冻结：** 进入MVP试点前，核心判断逻辑和输出格式必须被官方确认，不能带着"provisional"的模糊状态进入MVP。

**可以微调：** MVP试点运行期间，部分展示细节和适配层处理可以继续优化，但不能破坏已有的判断逻辑。

## 三、MVP试跑期可以继续微调的内容

以下内容在MVP试点期间可以继续优化，但需记录变更：

| 内容 | 可微调范围 | 限制 |
|------|-----------|------|
| 展示格式（颜色/图标/顺序） | 在MGT-01确认的框架内调整 | 不改变结构 |
| 台账字段顺序 | 在确认的字段列表内调整 | 不新增/删除字段 |
| 摘要分类标签名称 | 在确认的语义内调整 | 不改变分类逻辑 |
| 适配层映射workaround | 等MAP确认后更新 | 不破坏已有映射 |
| 演示脚本和讲解词 | 随时可更新 | 需记录版本 |

---

## 四、绝对不能在MVP期间修改的内容

| 内容 | 原因 |
|------|------|
| engine.py的5条核心规则（JUDGE逻辑） | 破坏TC01-TC05通过状态 |
| 判断优先级顺序 | 改变判断结果 |
| 输出三层层分离结构 | 影响管理层使用习惯 |
| 人工确认节点位置 | 影响一期设计原则 |

如果以上内容需要修改，必须：
1. 记录当前状态
2. 说明修改原因
3. 重新执行TC01-TC05验证
4. PMO和用户双重确认
