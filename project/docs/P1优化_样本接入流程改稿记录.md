# P1优化_样本接入流程改稿记录

**版本：** v1.0
**日期：** 2026-04-12
**阶段：** 6D（第二周期运行与P1优化落地）
**优化项：** P1-002 样本接入流程优化

---

## 一、优化目标

| 优化维度 | 改前问题 | 改后目标 |
|---------|---------|---------|
| 齐套可见性 | 样本是否齐套不一眼可见 | 齐套状态一眼可见 |
| 缺失提示 | 缺失项不显式提示 | 缺失项显式提示，哪个文件缺一目了然 |
| 下一步动作 | 接入后下一步不清楚 | 接入后下一步动作清楚 |
| 职责明确 | 业务方不知道谁做什么 | 明确告知数据方/系统/PMO各自职责 |

---

## 二、改前/改后对比

### 2.1 齐套状态展示优化

#### 【改前】数据接入页展示：

```
| 样本ID | 周报 | 台账 | 指标 | 状态 |
|--------|------|------|------|------|
| REAL-HIST-PMO-2025-1117-1121 | ✅ 完整 | ✅ 完整 | ✅ 完整 | 已接入 |
```

#### 【改后】数据接入页展示：

```
┌─────────────────────────────────────────────────────────┐
│ 📦 第二周期样本状态：⚠️ 未齐套                          │
│                                                         │
│  周报(.docx)：❌ 缺失 → 请联系公司金融管理委员会秘书处   │
│  台账(.xlsx)：✅ 已到位（2026-04-14 10:23）             │
│  指标(.xlsx)：⏳ 等待中                                 │
│                                                         │
│  📋 齐套标准：三份文件全部到位方可触发运行               │
│  📞 咨询联系人：数据提供方                              │
└─────────────────────────────────────────────────────────┘
```

**为什么这样改：**
- 原来"已接入"状态不直观，需要看三个字段才判断是否齐套
- 改后顶部直接显示"未齐套"或"✅ 已齐套"，一眼可见
- 缺失项有红色❌标记，旁边直接写"请联系谁"
- 齐套标准明确告知，避免误解

---

### 2.2 缺失项显式提示优化

#### 【改前】缺失提示：

```
当前选中样本详情
- 样本ID：LIVE-2026-W16-REAL
- 数据完整性：✅ 周报+台账+指标均完整
```

#### 【改后】缺失提示：

```
┌─────────────────────────────────────────────────────────┐
│ ⚠️ 样本接入检查                                          │
├─────────────────────────────────────────────────────────┤
│ 周报（周报_20260414-0418.docx）                         │
│   状态：✅ 已到位                                        │
│   位置：data/raw/周报_20260414-0418.docx                │
│                                                         │
│ 台账（台账_20260414-0418.xlsx）                         │
│   状态：❌ 缺失                                          │
│   要求：data/raw/目录下需有台账_20260414-0418.xlsx      │
│   联系人：数据提供方 → 补充后重新上传                    │
│                                                         │
│ 指标（指标_20260414-0418.xlsx）                         │
│   状态：⏳ 待提供                                        │
│   联系人：数据提供方 → 补充后重新上传                    │
├─────────────────────────────────────────────────────────┤
│ ▶ 下一步：补齐缺失文件后，点击"重新检查"按钮            │
└─────────────────────────────────────────────────────────┘
```

**为什么这样改：**
- 原来缺失项和正常项混在一起，不突出
- 改后每项单独一行，状态用✅/❌/⏳明显区分
- 缺失项直接告诉要求什么文件、在什么路径、联系谁
- "下一步"按钮引导明确

### 2.3 接入后下一步动作优化

#### 【改前】接入后提示：

```
样本接入成功。请前往领导摘要页查看。
```

#### 【改后】接入后提示：

```
┌─────────────────────────────────────────────────────────┐
│ ✅ 样本接入完成                                         │
│                                                         │
│ 已接入：周报 + 台账 + 指标（三份齐套）                   │
│ 切片ID：06_live_2026W16                                │
│ 切片时间：2026-04-12 14:30                             │
│                                                         │
│ ▶ 系统将在2小时内完成运行                                │
│ ▶ 预计14:30前可在"领导摘要"页查看输出                   │
│                                                         │
│ 📋 业务方动作：                                          │
│    1. 查看领导摘要，确认整体判断是否准确                 │
│    2. 查看事项总台账，筛选重点关注事项                   │
│    3. 查看风险清单，确认黄灯/延期识别                   │
│    4. 24小时内填写人工复核结论                          │
│                                                         │
│ 📞 如有问题联系：项目总控Agent                          │
└─────────────────────────────────────────────────────────┘
```

**为什么这样改：**
- 原来接入后不知道下一步，系统跑完也不知道看哪里
- 改后明确告知：什么时候看、看什么、谁负责复核
- 业务方知道样本到位后自己该做什么（不是只等着）

### 2.4 职责明确化优化

#### 【改前】数据接入页职责说明（缺失）：

```
（无明确职责说明）
```

#### 【改后】数据接入页职责说明：

```
┌─────────────────────────────────────────────────────────┐
│ 📋 样本接入角色说明                                      │
├─────────────────────────────────────────────────────────┤
│ 数据提供方：                                             │
│   - 提供周报(.docx)、台账(.xlsx)、指标(.xlsx)          │
│   - 将文件放入 data/raw/ 目录                           │
│   - 确认文件名符合规范                                  │
│                                                         │
│ 系统/Agent：                                             │
│   - 检测到文件后自动切片                                │
│   - 4小时内完成系统运行                                 │
│   - 生成领导摘要/台账/风险清单                          │
│                                                         │
│ PMO协调人：                                             │
│   - 收到通知后24小时内复核                             │
│   - 填写人工判断与系统判断一致性                        │
│   - 做出ABC决策                                         │
│                                                         │
│ 业务方：                                                 │
│   - 收到输出后查看领导摘要和风险清单                    │
│   - 对照原始周报评估系统准确性                          │
│   - 反馈具体问题（不反馈抽象评价）                      │
└─────────────────────────────────────────────────────────┘
```

**为什么这样改：**
- 原来谁负责什么不明确，出现问题互相推
- 改后数据方/系统/PMO/业务方各自职责一目了然
- 每个角色知道自己该做什么、什么时候做

## 三、优化后样本接入流程（完整版）

```
第一步：数据提供方准备文件
   ↓
   周报(.docx) + 台账(.xlsx) + 指标(.xlsx)
   ↓
第二步：数据提供方放入标准路径
   ↓
   data/raw/目录
   ↓
第三步：系统检测 → 显式展示齐套状态
   ↓
   ┌────────────────────────────────────────┐
   │ 若齐套：✅ 三份完整 → 触发切片+运行    │
   │ 若不齐套：❌ 显示缺失项 → 告知数据方补齐│
   └────────────────────────────────────────┘
   ↓
第四步：切片完成后，系统通知业务方
   ↓
   "领导摘要已生成，请前往查看"
   ↓
第五步：业务方查看输出
   ↓
   查看领导摘要 → 查看事项台账 → 查看风险清单
   ↓
第六步：PMO协调人24小时内复核
   ↓
   填写人工判断 → 登记偏差（如有）→ 做出ABC决策
   ↓
第七步：周五16:00前收口
   ↓
   本周运行归档 → 价值判断 → 下周决策
```

## 四、本轮优化解决了什么问题

| 问题 | 是否解决 | 说明 |
|------|---------|------|
| 样本是否齐套不一眼可见 | ✅ 已解决 | 顶部状态栏直接显示"已齐套/未齐套" |
| 缺失项不显式提示 | ✅ 已解决 | 缺失项有❌标记，旁注联系人和路径 |
| 接入后下一步不清楚 | ✅ 已解决 | 接入后明确告知看什么、谁复核、什么时候 |
| 业务方不知道谁做什么 | ✅ 已解决 | 四种角色职责明确，各自知道自己该做什么 |
