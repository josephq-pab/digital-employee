## 二、第一优先级：推进阻断清零

**核心任务：** 把Q01-Q05从"未发出/未到位"推进到"已确认/已到位"

### 2.1 立即可执行的动作

| # | 动作 | 优先级 | 说明 |

| 1 | 检查5个确认包是否已发出 | P0 | 未发出的立即发出 |

| 3 | 更新门控卡（G1-G7） | P0 | 每次状态变化后更新 |

| 5 | 准备下周推进周报 | P1 | 每周五更新 |

### 2.2 不能做的事

| # | 禁止事项 | 原因 |

| 1 | 不得在Q01-Q05未清零时进入阶段4B开发 | 违反阶段推进原则 |

| 3 | 不得接入消息通道 | 一期不做 |

## 三、当前工作路径

### 3.1 确认包发出链路

```
确认包已准备好（已有12份确认包文档）
    │
    ▼
执行发出动作（登记到《确认包发出清单》）
    │
    ▼
登记到《确认包签收与反馈登记表》
    │
    ▼
等待反馈
    │  超期 → 催办
    ▼
收到反馈 → 按《确认反馈处理SOP》处理
    │
    ▼
通过 → 更新《决策记录与待确认事项.md》
    ▼
更新《阶段4B进入门控卡》
```

### 3.2 样本收集链路

```
FU准备样本（参考《样本提交流转包》）
    │
    ▼
PC执行5维验收（参考《样本验收登记与退回模板》）
    │
    ▼
通过 → 更新registry.json → 通知Agent
    │
    ▼
更新《样本收集跟踪表》
    │
    ▼
S1≥1份到位 → Q02解除
    ▼
S2≥3份到位 → Q05解除
```

## 七、常见问题处理

| 问题 | 解答 |
|------|------|
| 确认包已发出但无回应 | 按《催办话术》发起第1次提醒 |
| PMO说"需要讨论" | 记录为"暂缓确认"，设定跟进日期 |
| 样本被退回怎么办 | 按《样本验收登记与退回模板》通知FU补充 |
| 多个确认包并行推进 | 各自独立跟踪，统一更新门控卡 |
| Q03和Q04都未确认能否进4B | 绝对不可以 |
