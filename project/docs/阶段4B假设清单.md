# 阶段4B假设清单

**版本：** v1.0
**日期：** 2026-04-07
**性质：** provisional / conditional（外部确认返回前不作为正式规则）

---

## 假设编号规则

- ADPT-ASSUME-01~04：适配层假设
- RULE-ASSUME-01~02：规则层假设
- OUTPUT-ASSUME-01~01：输出层假设

---

## 一、适配层假设

### ADPT-ASSUME-01：owner缺失处理

| 字段 | 内容 |
|------|------|
| 假设内容 | 周报本身不包含owner字段，以台账的owner_department字段作为责任人字段 |
| 来源依据 | 样本中周报无owner字段，台账有owner_department字段 |
| 当前风险 | 周报事项与台账项目无法直接关联owner |
| 替换条件 | PMO确认ADPT-03方案后，以确认的替代方案为准 |

### ADPT-ASSUME-02：risk_points抽取逻辑

| 字段 | 内容 |
|------|------|
| 假设内容 | 从周报key_alerts[].raw字段和management_summary.delayed_items中抽取风险点 |
| 来源依据 | key_alerts.alert_level=="yellow" → 触发风险；management_summary.items_requiring_leadership_attention → 需领导关注 |
| 当前风险 | 可能包含噪音（如"计划11月启动"属于正常计划描述，非风险信号） |
| 替换条件 | PMO确认ADPT-02后，以确认的抽取逻辑为准 |

### ADPT-ASSUME-03：escalation_requested缺失处理

| 字段 | 内容 |
|------|------|
| 假设内容 | 以关键词"请求领导关注"、"需协调"、"请领导"出现在management_summary.items_requiring_leadership_attention时，推断escalation_requested=true |
| 来源依据 | 周报/台账均无escalation_requested字段，management_summary.items_requiring_leadership_attention中明确写了才触发 |
| 当前风险 | 可能漏识别（仅依赖关键词）或误识别（依赖文本匹配准确性） |
| 替换条件 | PMO确认ADPT-03替代方案后，以确认方案为准 |

### ADPT-ASSUME-04：行业→项目映射默认规则

| 字段 | 内容 |
|------|------|
| 假设内容 | 周报中"汽车行业"黄灯 → 台账中ledger-10"六策合一-汽车零部件"；周报中"学校行业"黄灯 → 台账中暂无精确匹配 |
| 来源依据 | 映射文件中M1项，汽车行业fuzzy匹配到ledger-10；M2学校行业在台账中无精确对应 |
| 当前风险 | 行业维度与项目维度存在结构性差异，约67%的事项无法精确联动 |
| 替换条件 | PMO确认ADPT-04和MAP-01~04后，以确认的映射规则为准 |

## 二、规则层假设

### RULE-ASSUME-01：黄灯即中风险

| 字段 | 内容 |
|------|------|
| 假设内容 | 周报key_alerts.alert_level=="yellow"的任何事项，直接触发"中风险"判断 |
| 来源依据 | key_alerts.alert_level=="yellow"表示需要关注，与"规则_风险自述"语义接近 |
| 当前风险 | 黄灯是周报维度的状态，不等于项目维度的风险等级；可能存在误判 |
| 替换条件 | PMO确认PMO-02规则后，以正式规则为准 |

### RULE-ASSUME-02：结项申请视为已完成

| 字段 | 内容 |
|------|------|
| 假设内容 | 周报key_requests.closure_requests中的事项，视为"已完成/申请结项"状态 |
| 来源依据 | closure_requests表示责任人申请结项，在确认前视为进展正常 |
| 当前风险 | 结项申请不等于实际结项；部分申请可能未获批准 |
| 替换条件 | PMO确认规则后，以正式规则为准 |

---

## 三、输出层假设

### OUTPUT-ASSUME-01：领导摘要采用模板A

| 字段 | 内容 |
|------|------|
| 假设内容 | 领导摘要采用三层结构（事实→判断→建议），模板A格式 |
| 来源依据 | 管理层确认外发版中模板A为推荐选项，Q03尚未确认 |
| 当前风险 | 管理层可能选择模板B（简版），则需调整输出结构 |
| 替换条件 | 管理层确认MGT-01后，以选择的模板为准 |

## 四、假设影响范围

| 假设 | 影响模块 | 影响程度 |
|------|---------|---------|
| ADPT-ASSUME-01 | 适配层 | 中（责任人字段缺失，影响可追溯性） |
| ADPT-ASSUME-02 | 适配层/规则层 | 高（risk_points是核心判断依据） |
| ADPT-ASSUME-03 | 规则层 | 高（escalation是最高优先级触发器） |
| ADPT-ASSUME-04 | 适配层 | 高（67%事项无法联动，影响覆盖率） |
| RULE-ASSUME-01 | 规则层 | 中（黄灯直接判中风险，可能过度敏感） |
| RULE-ASSUME-02 | 规则层 | 低（结项申请数量少） |
| OUTPUT-ASSUME-01 | 输出层 | 中（结构已三层分离，调整成本不高） |
