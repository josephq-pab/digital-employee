# 正式试跑期回退机制

**版本：** v1.0
**日期：** 2026-04-08
**适用范围：** 正式MVP试跑期间

---

## 一、回退触发条件

| # | 触发条件 | 回退级别 | 说明 |
|---|---------|---------|------|
| 1 | 发现P0阻断问题 | 回退到问题修复前 | 立即停止，修复后继续 |
| 2 | 规则被确认错误 | 回退到规则修复前 | 暂停试跑，重新走Q01确认 |
| 3 | 样本数据严重不准确 | 回退到样本补充前 | 暂停，等样本补充 |
| 4 | 管理层/PMO要求暂停 | 计划性回退 | 按暂停通知执行 |
| 5 | 连续3次试跑输出不稳定 | 条件性回退 | 观察后决定 |

---

## 二、回退级别定义

### 级别1：轻微回退

**触发条件：** 单次试跑中出现展示/数据补充问题

**动作：**
1. 停止当前试跑
2. 修复问题（展示优化/补充数据）
3. 下一期重新执行
4. 不影响整体试跑进度

**恢复条件：** 修复后下一期正常

---

### 级别2：暂停试跑

**触发条件：** P0阻断 / 规则被确认错误

**动作：**
1. 立即暂停试跑
2. 发出暂停通知（按角色通知模板）
3. 修复P0或重新确认规则
4. 验证修复后恢复试跑

**恢复条件：** P0修复 + 验证通过 或 规则重新确认

---

### 级别3：退回演示版/模拟试跑状态

**触发条件：** 核心规则被否定 / 试点项目根本性变更

**动作：**
1. 暂停试跑并发出退回通知
2. 评估影响范围
3. 确定是走演示版还是从头开始
4. 更新门控卡状态
5. 如需重新确认Q01/Q03/Q04，重新执行确认流程

**恢复条件：** 重新完成Q01/Q03/Q04确认 + 门控更新

---

## 三、回退执行流程

```
触发回退条件
    ↓
评估回退级别（1/2/3）
    ↓
发出回退通知（按角色通知模板）
    ↓
执行回退动作
    ↓
修复/重新确认
    ↓
验证修复结果
    ↓
更新门控卡（如有变化）
    ↓
发出恢复通知
    ↓
恢复试跑
```

---

## 四、回退通知模板

### 暂停通知

```
主题：【正式试跑暂停通知】- [日期]

正式试跑暂停原因：[原因描述]

暂停级别：[级别1/2/3]
预计恢复时间：[如已知]
当前试跑状态：[最后一期输出路径]

处理动作：[正在处理的事项]
负责人：[角色代称]

预计恢复时间：待定
```

### 恢复通知

```
主题：【正式试跑恢复通知】- [日期]

正式试跑恢复。

已修复问题：[问题编号]
验证结果：[验证说明]

试跑状态：[正常/观察中]
```

## 六、回退时的文档处理

| # | 文档 | 处理方式 |
|---|------|---------|
| 1 | MVP试跑问题登记表 | 新增回退相关问题记录 |
| 2 | 正式MVP试跑周报模板 | 在周报中记录暂停原因和恢复时间 |
| 3 | 门控卡 | 如有门控变化，同步更新 |
| 4 | 输出文件 | 已生成的输出文件保留归档，不得删除 |
| 5 | 决策记录与待确认事项 | 如回退涉及Q01/Q03/Q04，口径变更需更新 |

---

## 七、恢复到正式试跑的前置条件

| 级别 | 恢复前置条件 |
|------|------------|
| 级别1（轻微） | 问题修复 + 下一期输出验证通过 |
| 级别2（暂停） | P0修复 + 验证通过 或 规则重新确认 |
| 级别3（退回） | Q01/Q03/Q04重新确认 + 门控全🟢 + 新启动通知发出 |
