## 二、周报JSON兼容性判定

### 2.1 到位后第一步检查

```
步骤1: 检查文件是否存在且非空
  ls -la samples/realtime/weekly_reports/*.json

步骤2: 检查JSON是否合法
  python3 -c "import json; json.load(open('samples/realtime/weekly_reports/xxx.json'))"

步骤3: 检查字段存在性
  python3 -c "
    import json
    data = json.load(open('samples/realtime/weekly_reports/xxx.json'))
    assert isinstance(data, list), '周报应为数组'
    assert len(data) > 0, '周报为空数组'
    assert 'report_meta' in data[0], '缺少report_meta'
    assert 'project_overview' in data[0], '缺少project_overview'
    print('字段检查通过')
  "
```

### 2.2 阻断字段（缺失则不能接入）

| 字段路径 | 类型 | 说明 | 备注 |
|---------|------|------|------|
| `report_meta.report_period` | string | 期数标识 | ❌ 缺失则阻断 |
| `project_overview.total_items` | int | 事项总数 | ❌ 缺失则阻断 |

### 2.3 可缺字段（缺失不影响接入）

| 字段路径 | 类型 | 说明 | 备注 |
|---------|------|------|------|
| `report_meta.report_date_text` | string | 日期文本 | ✅ 有默认值 |
| `project_overview.status_light` | string | 状态灯 | ✅ 有默认值（green） |
| `key_alerts` | array | 黄灯预警 | ✅ 可空 |
| `management_summary` | object | 管理层摘要 | ✅ 系统可生成 |

### 2.4 阻断情况处理

**若阻断字段缺失：**
1. 记录缺失字段名称
2. 联系项目方补充数据
3. 阻断接入，直到补充完成

## 三、台账JSON兼容性判定

### 3.1 到位后第一步检查

```
步骤1: 检查文件是否存在且非空
  ls -la samples/realtime/project_ledger/*.json

步骤2: 检查JSON是否合法
  python3 -c "import json; json.load(open('samples/realtime/project_ledger/xxx.json'))"

步骤3: 检查字段存在性
  python3 -c "
    import json
    data = json.load(open('samples/realtime/project_ledger/xxx.json'))
    assert 'records' in data, '缺少records字段'
    assert isinstance(data['records'], list), 'records应为数组'
    assert len(data['records']) > 0, 'records为空数组'
    r = data['records'][0]
    assert 'record_id' in r, 'records[0]缺少record_id'
    assert 'project_name' in r, 'records[0]缺少project_name'
    print('字段检查通过')
  "
```

### 3.2 阻断字段（缺失则不能接入）

| 字段路径 | 类型 | 说明 | 备注 |
|---------|------|------|------|
| `records` | array | 记录数组 | ❌ 缺失则阻断 |
| `records[].record_id` | string | 记录唯一ID | ❌ 缺失则阻断 |
| `records[].project_name` | string | 项目名称 | ❌ 缺失则阻断 |

### 3.3 可缺字段（缺失不影响接入）

| 字段路径 | 类型 | 说明 | 备注 |
|---------|------|------|------|
| `records[].progress_by_period` | object | 各期进展 | ✅ 可为空对象 |
| `records[].owner_department` | string | 负责部门 | ✅ 可缺 |
| `records[].subitems` | array | 子事项 | ✅ 可空 |

## 四、兼容性判定结果分类

### 4.1 可直接接入（A级）

**满足全部A级条件：**

| # | 条件 | 说明 |
|---|------|------|
| 1 | 周报JSON格式合法 | JSON.parse无报错 |
| 2 | 周报阻断字段全部存在 | report_period + total_items |
| 3 | 台账JSON格式合法 | JSON.parse无报错 |
| 4 | 台账阻断字段全部存在 | records + record_id + project_name |
| 5 | 项目方确认与样本差距可接受 | Q21回答"否"或"低" |

**可直接执行文件替换：**
```bash
cp samples/realtime/weekly_reports/xxx.json samples/samples/weekly_reports/weekly_report_series_latest3.json
cp samples/realtime/project_ledger/xxx.json samples/samples/project_ledger/project_ledger_latest3.json
cd project && python3 scripts/run_prototype.py
```

### 4.2 需轻度适配（B级）

**满足全部B级条件：**

| # | 条件 | 说明 |
|---|------|------|
| 1 | JSON格式合法 | JSON.parse无报错 |
| 2 | 阻断字段全部存在 | 核心字段不缺 |
| 3 | 非核心字段名称不一致 | normalizer可做映射 |
| 4 | 项目方能提供字段映射表 | 开发方可配置 |

**适配步骤：**
1. 项目方提供字段映射表（周报字段名→样本字段名）
2. 开发方在normalizer中配置映射
3. 执行文件替换和冒烟测试
4. 通过7/7 PASS后进入验证

### 4.3 暂不可接入（C级）

**满足任一C级条件：**

| # | 条件 | 说明 |
|---|------|------|
| 1 | JSON格式不合法 | JSON.parse报错 |
| 2 | 阻断字段缺失 | report_period/total_items/records等缺失 |
| 3 | 数据为空数组 | 周报或台账为空 |
| 4 | 非JSON格式无法转换 | 导出为Excel/CSV，无法转换为JSON |

**处理方式：**
1. 记录问题类型
2. 联系项目方处理
3. 等问题解决后重新判定
