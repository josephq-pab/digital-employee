# 扩大试点风险与边界说明

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4L《扩大评审准备与真实试运行待命》

---

## 一、最可能出现的风险（按可能性排序）

### 风险1：真实数据适配性不足（高可能性）

**描述：** 真实周报与历史样本格式存在差异，导致适配器识别率下降（如：字段命名不同、结构不同、特殊情况未覆盖）。

**影响：** 系统输出不完整，部分事项被遗漏。

**应对原则：**
- 先做影子运行（不替代人工）
- 发现适配性问题 → 记录 → 修正适配器 → 再验证
- 适配器无法解决的结构性问题 → 暂停 → 回退到人工流程

**止损线：** 影子运行中若超过20%的事项识别错误，暂停扩大，先修正适配器。

---

### 风险2：人工确认负担超出预期（中等可能性）

**描述：** 系统输出量大，人工逐条确认耗时超出预期，导致确认流程流于形式。

**影响：** 人工确认变成走过场，失去质量控制意义。

**应对原则：**
- 设计分级确认机制（高风险事项优先确认，低风险事项抽样确认）
- 管理层确认接受一定的容错率
- 发现负担过重 → 反馈 → 调整输出粒度或阈值

**止损线：** 确认耗时超过人工处理时间的50%时，触发流程优化讨论。

---

### 风险3：输出误导管理层（低可能性但影响大）

**描述：** 系统对某事项的判断（正常/黄灯）与人工判断不一致，且未经确认直接提交。

**影响：** 管理层基于错误信息做决策。

**应对原则：**
- 强提醒：所有系统输出必须经过人工确认才能提交
- 标记机制：系统自动标记"待人工确认"状态
- 不替代人工：最终决策权在人，不在系统

**止损线：** 任何误判且未经确认直接提交的情况，立即暂停，触发问题调查。

---

## 二、明确边界（不可逾越）

| 边界 | 说明 | 违反后果 |
|------|------|---------|
| 不写回生产系统 | 任何情况下不修改生产数据 | 立即暂停+问题调查 |
| 不替换人工判断 | 系统输出仅供参考，最终决策在人 | 立即暂停+流程重审 |
| 不接入生产平台 | 仅做输入→输出比对，不改变任何系统 | 立即停止扩大试点 |
| 不超一期范围 | 不扩大到PMO重点项目以外的场景 | 需重新评审 |

---

## 三、止损原则

1. **最小止损单位：** 暂停扩大试点，本地修正，不影响任何生产流程
2. **恢复条件：** 问题修复后，经评审方确认方可继续
3. **完全回退：** 关闭系统即可，无任何生产侧遗留问题

---

## 四、观察触发条件

以下情况触发预警，需评审方介入：

| 触发条件 | 建议动作 |
|---------|---------|
| 影子运行识别准确率 < 80% | 暂停，扩大试点讨论 |
| 人工确认耗时 > 预期50% | 优化流程设计 |
| 发现重大适配漏洞 | 暂停，问题修复后重审 |
| 任意边界违反 | 立即停止，问题调查 |
