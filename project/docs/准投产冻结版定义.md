# 准投产冻结版定义

> 版本：v1.0（2026-04-16）
> 用途：明确"冻结版必须达到什么程度才能启动最终移交"

---

## 一、冻结版定义

**准投产冻结版** = 满足以下全部6个维度的版本，是启动最终移交的前提条件。

```
冻结版达成  →  最终移交启动  →  迁移后只剩少量配置/缺陷修
```

---

## 二、冻结版必须满足的6个维度

### 维度1：主流程完整

| 检查项 | 要求 |
|--------|------|
| 数据口径 | DATA-001 登记的口径问题全部关闭 |
| 归一化逻辑 | 多期数据归一化结果稳定一致 |
| 规则判断 | 规则逻辑无歧义，判断结果可解释 |
| 输出字段 | 输出格式和字段定义不再变更 |

### 维度2：页面体验收口

| 检查项 | 要求 |
|--------|------|
| MVP页面 | 页面功能完整，无阻断性体验问题 |
| 输出文件 | 领导摘要/台账/风险清单格式锁定 |
| 错误处理 | 异常输入有明确错误提示，不崩溃 |
| 反馈表单 | 表单逻辑锁定，提交后数据可追溯 |

### 维度3：数据口径稳定

| 检查项 | 要求 |
|--------|------|
| 口径登记 | DATA-001 所有指标口径已登记 |
| 口径一致性 | 台账与原始周报数据一致 |
| 样本完整性 | 样本覆盖主要业务场景 |
| 口径锁定 | 无未登记的口径漂移 |

### 维度4：连续运行稳定

| 检查项 | 要求 |
|--------|------|
| S1 连续3期 | 连续3期无P0阻断性问题 |
| P1 问题 | 无P1级阻断问题（或已全部关闭） |
| 影子运行 | 完成至少1次影子运行验证 |
| 错误日志 | 错误日志可定位、可复现 |

### 维度5：业务可接受

| 检查项 | 要求 |
|--------|------|
| B1 业务可接受 | 业务方确认原型逻辑符合业务预期 |
| B2 输出格式可接受 | 业务方确认输出格式可直接使用 |
| B3 反馈机制可接受 | 业务方确认反馈表单机制有效 |
| B4 整体流程可接受 | 业务方确认整体流程可推进 |

### 维度6：迁移后不再需要主线级开发

| 检查项 | 要求 |
|--------|------|
| 功能边界 | 功能边界明确，无模糊地带 |
| 代码稳定 | 代码结构稳定，迁移后不需大重构 |
| 配置锁定 | 配置项明确，迁移后仅少量配置调整 |
| 缺陷清单 | 剩余缺陷清单明确，均为P2及以下 |

---

## 三、冻结版达成后的移交范围

达到冻结版后，移交内容：

| 类别 | 内容 |
|------|------|
| 代码 | 冻结版完整代码包 |
| 配置 | 所有配置项及说明 |
| 样本 | 冻结版样本数据 |
| 文档 | 冻结版操作手册 + 生产SOP |
| 缺陷清单 | 剩余缺陷清单（P2及以下） |
| 数据口径 | DATA-001 完整登记册 |

---

## 四、迁移后可以继续做的事情

冻结版移交后，**允许**在生产环境继续做：

| 动作 | 范围 |
|------|------|
| 替换模型 | 用生产级模型替换当前原型模型 |
| 替换环境 | 部署到生产环境 |
| 接生产数据 | 接入真实业务数据 |
| 配置调整 | 少量配置参数调整 |
| 缺陷修复 | P2及以下缺陷修复 |

---

## 五、冻结版未达成之前禁止的事项

| 禁止事项 | 原因 |
|---------|------|
| 禁止启动最终迁移 | 迁移后无法继续主线完善 |
| 禁止宣称"最终交付完成" | 当前不是最终版，是冻结建设版 |
| 禁止修改数据口径后不登记 | DATA-001 是冻结版前置条件 |
| 禁止在未验证连续运行前移交 | 生产环境可能出现未预知问题 |

---

## 六、冻结版检查清单

**冻结版自检（达到以下条件才能启动最终移交）：**

```
□ 主流程：DATA-001 全部关闭
□ 主流程：归一化结果连续3期一致
□ 主流程：规则逻辑无歧义
□ 体验：页面功能完整
□ 体验：输出格式已锁定
□ 口径：DATA-001 完整登记
□ 口径：台账与原始周报一致
□ 运行：S1 连续3期无P0
□ 运行：无P1阻断性问题
□ 业务：B1~B4 至少3/4 确认
□ 边界：功能边界明确
□ 缺陷：剩余缺陷均为P2及以下
□ 文档：冻结版操作手册完成
□ 迁移：迁移检查清单通过
```

---

*本文档为准投产冻结版的权威定义，是启动最终移交的判定标准。*
