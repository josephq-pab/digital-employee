# 哪些问题需要决策，哪些问题不需要

**版本**：v1.0  
**日期**：2026-04-09  
**阶段**：阶段5A《并行运行与价值验证》  
**用途**：明确什么进入ABC决策流程，什么不进入

---

## 一、需要进入ABC决策的具体问题

### 1. 判断口径类

| 问题类型 | 示例 | 是否进入决策 |
|---------|------|------------|
| 风险等级判断标准 | "项目延期2周算黄灯还是逾期？" | ✅ 进入 |
| 结项判断标准 | "申请结项需要哪些条件同时满足？" | ✅ 进入 |
| 指标偏离阈值 | "偏离目标多少算严重偏离？" | ✅ 进入 |
| 分类名称定义 | "这种类型应该叫'指标偏离'还是'结构性压力'？" | ✅ 进入 |

### 2. 输出结构类

| 问题类型 | 示例 | 是否进入决策 |
|---------|------|------------|
| 摘要字段增减 | "领导摘要里要不要加'主责部门'字段？" | ✅ 进入 |
| 台账列顺序调整 | "系统判断列应该放在哪一列？" | ✅ 进入 |
| 风险清单展示方式 | "风险清单要不要加历史追踪时间线？" | ✅ 进入 |
| 报告格式变更 | "要不要加周期对比表格？" | ✅ 进入 |

### 3. 使用方式类

| 问题类型 | 示例 | 是否进入决策 |
|---------|------|------------|
| 输出分发范围 | "领导摘要应该发给谁？" | ✅ 进入 |
| 使用场景定义 | "这个输出适合在周会还是月会使用？" | ✅ 进入 |
| 与现有工具的配合 | "系统台账和原有台账怎么并行使用？" | ✅ 进入 |
| 复核流程确认 | "人工复核的标准流程是什么？" | ✅ 进入 |

### 4. 切换时机类

| 问题类型 | 示例 | 是否进入决策 |
|---------|------|------------|
| 扩大使用范围 | "某类事项可以扩大到直接用系统判断吗？" | ✅ 进入 |
| 减少人工复核 | "某类输出可以免复核直接用吗？" | ✅ 进入 |
| 规则库更新 | "某条新规则可以加入自动判断吗？" | ✅ 进入 |
| 暂停/恢复运行 | "某周期暂停运行需要谁批准？" | ✅ 进入 |

## 二、不需要进入ABC决策的问题

### 1. 抽象评价类

| 问题类型 | 示例 | 为什么不需要 |
|---------|------|------------|
| "这个系统好不好用？" | 泛泛评价，无具体案例 | 抽象，无法决策 |
| "我觉得不太对" | 无具体说明哪里不对 | 无具体场景，无法分析 |
| "感觉不够专业" | 无具体指出哪里不专业 | 抽象，无法改 |
| "领导会不会接受？" | 无具体场景 | 猜测，不是问题 |

### 2. 泛泛意见类

| 问题类型 | 示例 | 为什么不需要 |
|---------|------|------------|
| "应该更智能一点" | 未说明具体智能场景 | 无法执行 |
| "要多考虑实际情况" | 未说明哪些情况 | 无法改 |
| "最好能自动XXX" | 无具体需求说明 | 无法评估 |
| "要符合行业标准" | 未说明哪个标准 | 无法对接 |

### 3. 无具体案例的问题

| 问题类型 | 示例 | 为什么不需要 |
|---------|------|------------|
| "这种情况怎么处理？" | 无具体场景描述 | 无法分析 |
| "如果出现XXX怎么办？" | 假设性问题，无实际发生 | 等实际发生再决策 |
| "别人家怎么做的？" | 无直接参照价值 | 背景不同，判断不同 |
| "以后可能会XXX" | 未来假设，未发生 | 等发生再决策 |

### 4. 其他不进入决策的事项

| 事项 | 原因 |
|------|------|
| 整体方向讨论 | 本阶段不争论大方向 |
| 泛泛的"评审"要求 | 已有具体问题决策机制 |
| 人事/资源问题 | 不在技术决策范围内 |
| 数据安全问题 | 需单独走安全评估流程 |

## 三、判断标准速查

| 如果遇到这个问题 | 问自己 | 进入决策？ |
|----------------|--------|---------|
| "我有个问题想讨论" | 有具体案例或场景吗？ | 有→是 / 无→否 |
| "这种情况应该怎么判" | 有具体的数据或事项吗？ | 有→是 / 无→否 |
| "能不能加个功能" | 有明确的使用场景吗？ | 有→是 / 无→否 |
| "这个系统好像不太对" | 能说出具体哪里不对吗？ | 能→是 / 不能→否 |
| "领导说…" | 有具体指令或反馈吗？ | 有→是 / 猜测→否 |

---

## 四、问题进入决策的最小要求

进入ABC决策的问题必须满足：

1. **有具体场景**：至少有一个实际发生的案例
2. **有影响范围**：能说明影响的是哪类事项
3. **有选项**：至少能提出A/B两个可执行的选项
4. **有决策人**：能指定谁来做这个决策

不满足以上四条的问题，记录到《待确认事项》待后续补充。
