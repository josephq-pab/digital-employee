# MVP试跑运行SOP

**版本：** v1.0
**日期：** 2026-04-08
**试跑性质：** MVP模拟试跑（dry-run / conditional）

---

## 一、试跑前（Pre-run）

### 1.1 样本准备

**动作：** 确认样本文件存在且可读

**操作步骤：**
1. 检查以下文件是否存在：
   - `samples/samples/weekly_reports/weekly_report_series_latest3.json`
   - `samples/samples/project_ledger/project_ledger_latest3.json`
   - `samples/samples/linked/weekly_to_ledger_mapping_latest3.json`
2. 确认文件非空（>1KB）
3. 确认Python环境可用：`/tmp/py39env/bin/python --version`

**异常处理：**
- 如文件缺失 → 停止试跑，登记问题[P1-文件缺失]，转《试跑问题转任务规则》
- 如Python不可用 → 停止试跑，登记问题[P0-环境异常]

**输出：** 样本文件状态确认

---

### 1.2 脚本准备

**动作：** 确认执行脚本可用

**操作步骤：**
1. 确认 `project/scripts/run_prototype.py` 存在
2. 确认所有依赖模块存在（src/loaders, src/adapters, src/generators, app/engine）

**输出：** 脚本就绪状态

---

### 1.3 输出目录准备

**动作：** 确认输出目录存在

**操作步骤：**
1. 确认以下目录存在（或自动创建）：
   - `project/data/output/`（输出文件）
   - `project/data/tmp/`（中间文件）
2. 清理上次遗留文件（可选，保持归档）

**输出：** 目录就绪状态

---

## 二、试跑中（Run）

### 2.1 正式执行

**动作：** 运行原型执行脚本

**执行命令：**
```bash
cd /home/admin/.openclaw/workspace-digital-employee
/tmp/py39env/bin/python project/scripts/run_prototype.py
```

**观察要点：**
- Step 1/6：样本加载是否正常（周报3期，台账20条，映射12条）
- Step 2/6：归一化是否正常（期望11条事项）
- Step 3/6：规则判断是否正常（无报错）
- Step 4/6：输出生成是否正常（摘要、台账、清单）
- Step 5/6：文件保存是否正常（检查文件路径）
- Step 6/6：冒烟测试是否全部PASS（期望7/7）

**关键指标：**
| 指标 | 期望值 | 异常阈值 |
|------|--------|---------|
| 归一化事项数 | ≥10 | <8 |
| 映射覆盖率 | 36%+ | <30% |
| 风险项数 | ≥1 | =0 |
| 冒烟测试 | 7/7 PASS | <5/7 |

**异常处理：**
- 冒烟测试 <5/7 → 立即停止，登记[P0-阻断性问题]
- 脚本报错 → 截图错误信息，登记[P0-运行异常]

---

### 2.2 人工复核视角检查

**动作：** 对照样本原始数据，检查输出的可合理性

**操作步骤：**
1. 对比原始周报与领导摘要：事项是否有遗漏？
2. 对比风险信号与原始周报：风险判断是否合理？
3. 检查owner字段：是否正确显示[待关联]或实际值？
4. 检查期间覆盖：3期周报是否全部覆盖？

**检查标准（详见《MVP试跑评估标准》）：**
- P0：输出错误/缺失，无法进入正式试跑
- P1：输出有问题但可继续
- P2：观察项，不阻断试跑

**输出：** 人工复核结论（通过/不通过）

---

## 三、试跑后（Post-run）

### 3.1 输出归档

**动作：** 将本轮输出归档到指定路径

**操作步骤：**
1. 记录本轮输出的时间戳（如 `20260408_050528`）
2. 将以下文件标记为"已归档"：
   - `project/data/output/领导摘要_<ts>.txt`
   - `project/data/output/事项总台账_<ts>.txt`
   - `project/data/output/风险清单_<ts>.txt`
   - `project/data/tmp/normalized_<ts>.json`
3. 在《首轮MVP模拟试跑记录.md》中登记输出路径

**命名规范：**
- 输出文件自动带时间戳，无须重命名
- 问题登记以"P"开头，后跟日期+序号（如P1, P2）

---

### 3.2 问题登记

**动作：** 将所有发现的问题登记到《MVP试跑问题登记表》

**操作步骤：**
1. 对照《MVP试跑评估标准》对每条问题定级
2. 在《MVP试跑问题登记表》新增记录
3. 在《首轮MVP模拟试跑问题清单.md》中追加

**登记时机：**
- 运行中发现的问题 → 运行后立即登记
- 人工复核中发现的问题 → 复核后立即登记

---

### 3.3 结果判定

**动作：** 对照《MVP试跑结果判定规则》给出A/B/C结论

**判定输入：**
- 冒烟测试通过数
- 人工复核结论
- 问题登记表（P0/P1/P2数量）

**判定流程：**
1. 统计P0数量 → 有P0则直接C级
2. 统计P1数量 → P1≥3则B级，否则参考P2
3. 无P0且P1<3 → 参考《MVP试跑评估标准》最终判定

**输出：** 《首轮MVP模拟试跑结果结论.md》

---

### 3.4 版本记录

**动作：** 在《阶段4E修订记录.md》中记录本轮试跑结果

**记录内容：**
- 执行时间
- 输入版本
- 输出路径
- 问题数量（P0/P1/P2）
- 最终结论（A/B/C）
