## QUAL-01修复详情

### 问题
黄灯事项被engine.py判断为"低风险"，应判断为"中风险"。

### 根因
engine.py的LAG_PATTERNS检查`current_progress`字段，模式`滞后`不能匹配中文词"逾期"。adapter注入"[黄灯]存在逾期风险，进度滞后"到`risk_points`，但LAG_PATTERNS不检查risk_points。

### 修复方案
在`normalizer.py`的`build_normalized_item`中，当`is_yellow_alert=True`时，向`current_progress`字段注入"进度滞后"文本，确保LAG_PATTERNS能匹配。

### 修改文件
`project/src/adapters/normalizer.py`

### 修改内容

**修改前：**
```python
# 黄灯标记（等同于中风险信号）
if is_yellow_alert and '风险' not in risk_points and '滞后' not in risk_points.lower():
    risk_points = (risk_points + f'[黄灯]{project_name}有风险提示').strip('；')

# 责任人（provisional：周报无owner，用台账owner_department）
owner = ''
if ledger_record:
    owner = ledger_record.get('owner_department', '')
else:
    owner = '[待关联]'
```

**修改后：**
```python
# 黄灯 → 中风险注入（QUAL-01修复）
# 关键：engine.py的LAG_PATTERNS检查的是current_progress字段（不是risk_points）
# 因此注入"进度滞后"到current_progress，确保能被LAG_PATTERNS识别
if is_yellow_alert:
    if '进度滞后' not in risk_points and '滞后' not in risk_points:
        risk_points = (risk_points + '[黄灯]存在逾期风险，进度滞后').strip('；')
    # QUAL-01核心修复：注入"进度滞后"到current_progress（供LAG_PATTERNS匹配）
    if '进度滞后' not in current_progress:
        current_progress = current_progress + '存在逾期风险，进度滞后。'

# 责任人（provisional：周报无owner，用台账owner_department）
# QUAL-05修复：优先从ledger_record提取owner_department
owner = '[待关联]'
if ledger_record:
    owner = ledger_record.get('owner_department', '') or '[待关联]'
```

## QUAL-02修复详情

### 问题
7个结项申请（closure_requests）误入风险清单，标记为"其他风险"。

### 根因
`risk_list_generator.py`中`is_closure_request=True`被当作is_risk条件之一，但结项申请代表"风险已释放"，不应进入风险清单。

### 修复方案
在risk_list_generator.py中，对`is_closure_request=True`的事项直接跳过（continue），不进入风险清单。

### 修改文件
`project/src/generators/risk_list_generator.py`

### 修改内容

**修改前：**
```python
for item, result in zip(normalized_items, engine_results):
    risk_level = result.get('risk_assessment', {}).get('level', '未知')
    escalation = result.get('escalation_recommended', False)
    reasoning = result.get('reasoning', {})
    overall = result.get('status_summary', {}).get('overall', '未知')

    # 判断是否属于风险事项
    is_risk = (
        item.get('is_yellow_alert')
        or risk_level in ('中', '高')
        or escalation
        or '延期' in item['project_name']
        or item.get('is_closure_request')  # ← bug
    )

    if not is_risk:
        continue
```

**修改后：**
```python
for item, result in zip(normalized_items, engine_results):
    # QUAL-02修复：结项申请 = 风险已释放，不进入风险清单
    if item.get('is_closure_request'):
        continue

    risk_level = result.get('risk_assessment', {}).get('level', '未知')
    escalation = result.get('escalation_recommended', False)
    reasoning = result.get('reasoning', {})
    overall = result.get('status_summary', {}).get('overall', '未知')

    # 判断是否属于风险事项
    is_risk = (
        item.get('is_yellow_alert')
        or risk_level in ('中', '高')
        or escalation
        or '延期' in item['project_name']
    )

    if not is_risk:
        continue
```

### 修复后效果

## QUAL-05修复详情

### 问题
所有事项的"责任人"字段均显示"[待关联]"。

### 根因
`normalize_weekly_report`中处理closure_requests时，未传递`ledger_record`参数，导致即使映射文件中有对应ledger记录，也无法提取owner_department。

### 修复方案
为closure项也查找映射并传递ledger_record。

### 修改文件
`project/src/adapters/normalizer.py`

### 修改内容

**修改前（closure处理部分）：**
```python
for closure in wr_item.get('key_requests', {}).get('closure_requests', []):
    item = build_normalized_item(
        project_name=closure.get('project_name', '未知'),
        ...
        ledger_record=None,  # ← 未传递
        is_yellow_alert=False,
    )
```

**修改后：**
```python
for closure in wr_item.get('key_requests', {}).get('closure_requests', []):
    # QUAL-05修复：为closure项也查找台账以提取owner
    closure_ledger_rec = None
    for m in mapping:
        if m['period'] == period and m['source_type'] == 'closure_requests':
            if closure.get('project_name', '') and closure.get('project_name', '').startswith(m['source_project_name'][:8]):
                if m['ledger_record_id']:
                    closure_ledger_rec = ledger_index.get(m['ledger_record_id'])
                    break
    item = build_normalized_item(
        ...
        ledger_record=closure_ledger_rec,  # ✅ 传递
        is_yellow_alert=False,
    )
```
