## 校验步骤

### 【Step 1】样本齐全性检查

| 检查项 | 标准 | 结果 |
|--------|------|------|
| 周报文件存在 | `normalized/weekly_reports/by_period/YYYY-MM-DD_YYYY-MM-DD.json` | ☐ |
| 台账文件存在 | `normalized/project_ledger/by_period/YYYY-MM-DD_YYYY-MM-DD.json` | ☐ |
| 指标文件存在（如有）| `normalized/quant_metrics/by_period/YYYY-MM-DD_YYYY-MM-DD.json` | ☐ |
| 文件可读取 | json.loads()成功 | ☐ |

**Step 1 结论：** ☐ 全部通过 → 继续Step 2　　☐ 任一不通过 → **补充后再校验**

---

### 【Step 2】脱敏检查

| 检查项 | 标准 | 结果 |
|--------|------|------|
| 项目名称不包含真实客户名 | 无企业/人名敏感信息 | ☐ |
| 无个人身份信息 | 无身份证/手机号/邮箱 | ☐ |

**Step 2 结论：** ☐ 通过 → 继续Step 3　　☐ 不通过 → **退回重新脱敏**

---

### 【Step 3】期间标识检查

| 检查项 | 标准 | 结果 |
|--------|------|------|
| period 字段存在 | report_period 或 period 字段存在 | ☐ |
| period 可解析 | 能确定起止日期 | ☐ |

**Step 3 结论：** ☐ 通过 → 继续Step 4　　☐ 不通过 → **补充后再校验**

---

### 【Step 4】周报/台账/指标对应检查

| 检查项 | 标准 | 结果 |
|--------|------|------|
| 周报期间与台账期间一致 | period 匹配 | ☐ |
| 周报期间与指标期间一致（如适用）| period 匹配 | ☐ |
| 周报中项目能在台账中找到 | 至少部分项目有映射 | ☐ |

**Step 4 结论：** ☐ 通过 → 继续Step 5　　☐ 不通过 → **标注差异，继续**

---

### 【Step 5】统一结构映射检查

| 检查项 | 标准 | 结果 |
|--------|------|------|
| 能映射到 request_lines | 有申请事项字段 | ☐ |
| 能映射到 alert_lines | 有预警/黄灯字段 | ☐ |
| 能提取 project_name | 有项目名称字段 | ☐ |
| 有事项内容 | request_lines 或 alert_lines 非空 | ☐ |

**Step 5 结论：** ☐ 通过 → 进入影子运行　　☐ 不通过 → **标注差异，继续**

---

### 【Step 6】阻断性缺口检查

| 检查项 | 缺口类型 | 结果 |
|--------|---------|------|
| period 完全缺失 | 阻断 | ☐ 有 / ☐ 无 |
| 无 request + alert | 阻断 | ☐ 有 / ☐ 无 |
| 台账完全缺失 | 阻断 | ☐ 有 / ☐ 无 |
| 完全无法映射 | 阻断 | ☐ 有 / ☐ 无 |

**Step 6 结论：** ☐ 无阻断缺口 → **通过，可进入影子运行**　　☐ 有阻断缺口 → **暂停，补充后重验**
