# 原型模块拆解与目录设计

**版本：** v1.0
**用途：** 阶段4实现的模块划分和目录结构设计，供后续Agent按模块接力开发

---

## 一、模块划分原则

- 每个模块有明确的输入、输出、依赖
- 模块之间无循环依赖
- 可独立测试
- 可按优先级串行或并行开发

---

## 二、模块清单

### 模块1：输入解析器（InputParser）

**功能：** 将格式A（周报/月报）或格式B（台账）JSON解析为标准内部记录

**输入：** 原始JSON（字符串或dict）

**输出：** InternalProjectRecord（标准化内部对象）

**InternalProjectRecord结构：**
```python
{
    "project_name": str,
    "owner": str,
    "report_type": str,          # "weekly" | "monthly"
    "report_date": str,           # ISO date
    "promised_items": str,
    "current_progress": str,
    "risk_points": str,           # "" if empty
    "escalation_requested": bool,
    "items": list,                # 格式B专用，格式A为空列表
    "format": str,               # "weekly" | "ledger"
    "truncated": bool            # 是否因过长被截断
}
```

**前置依赖：** 无

**是否必需：** ✅ 是（原型必须实现）

**完成标准：**
- [ ] 格式A JSON可正确解析
- [ ] 格式B JSON可正确解析（阶段5）
- [ ] 空字段按默认值处理
- [ ] 字段过长截断且标记truncated=true
- [ ] 字段缺失时报错，不静默通过

---

### 模块2：事实提取器（FactExtractor）

**功能：** 从InternalProjectRecord中提炼不可争辩的事实清单

**输入：** InternalProjectRecord

**输出：** Facts（列表）

**Facts结构：**
```python
[
    "承诺事项：{promised_items原文}",
    "当前进展：{current_progress原文}",
    "风险自述：{risk_points原文}",
    # 如escalation=true，追加：
    "责任人主动请求领导介入"
]
```

**前置依赖：** InputParser

**是否必需：** ✅ 是（原型必须实现）

**完成标准：**
- [ ] promised_items原文完整透传（截断记录）
- [ ] current_progress原文完整透传
- [ ] risk_points为空时不输出（不写"无风险"）
- [ ] escalation=true时有明确标识

---

### 模块3：规则引擎（RuleEngine）

**功能：** 按关键词模式库执行判断，返回JudgementResult

**输入：** InternalProjectRecord

**输出：** JudgementResult

**JudgementResult结构：**
```python
{
    "risk_level": str,           # "低" | "中" | "高"
    "escalation_recommended": bool,
    "overall": str,              # "正常" | "关注" | "危险"
    "triggered_signals": list,
    "rules_applied": list,
    "raw_signals": str
}
```

**前置依赖：** FactExtractor（获取原始文本用于正则匹配）

**是否必需：** ✅ 是（原型必须实现）

**完成标准：**
- [ ] 5条核心规则全部实现（见《判断规则与状态机说明》）
- [ ] 优先级顺序正确（严重滞后先于一般滞后）
- [ ] escalation=true直接触发高风险
- [ ] 每条规则记录rules_applied
- [ ] 原始关键词记录raw_signals

---

### 模块4：摘要生成器（SummaryGenerator）

**功能：** 将JudgementResult生成为可读文本（领导摘要）

**输入：** JudgementResult + InternalProjectRecord + Facts

**输出：** LeaderSummary（结构化dict）

**LeaderSummary结构：**
```json
{
  "report_id": "uuid-v4",
  "generated_at": "ISO8601",
  "project_name": "string",
  "owner": "string",
  "status_summary": { ... },
  "risk_assessment": { ... },
  "suggested_actions": ["string"],
  "escalation_recommended": bool,
  "escalation_indicator": "system_suggested | user_requested",
  "reasoning": { ... }
}
```

**前置依赖：** RuleEngine

**是否必需：** ✅ 是（原型必须实现）

**完成标准：**
- [ ] 生成UUID作为report_id
- [ ] generated_at为ISO8601格式
- [ ] status_summary.progress_text截断至200字以内
- [ ] suggested_actions非空列表
- [ ] escalation_indicator正确区分system_suggested和user_requested

---

### 模块5：台账生成器（LedgerGenerator）

**功能：** 将事项列表生成为事项总台账（阶段5扩展输出）

**输入：** InternalProjectRecord（包含items数组）+ JudgementResult

**输出：** ItemLedger（结构化列表）

**前置依赖：** InputParser（格式B）+ RuleEngine

**是否必需：** ❌ 否（阶段5要求，阶段4不实现）

---

### 模块6：风险清单生成器（RiskListGenerator）

**功能：** 将风险自述生成为风险清单（阶段5扩展输出）

**输入：** InternalProjectRecord + JudgementResult

**输出：** RiskList（结构化列表）

**前置依赖：** RuleEngine

**是否必需：** ❌ 否（阶段5要求，阶段4不实现）

## 四、模块依赖关系图

```
InputParser
    │  (输出InternalRecord)
    ▼
FactExtractor
    │  (输出Facts + 透传InternalRecord)
    ▼
RuleEngine ────────────────────▶ SummaryGenerator
    │                                  │ (输出LeaderSummary)
    │                                  ▼
    │                            LeaderSummaryJSON
    │
    ▼（阶段5）
LedgerGenerator ──────────────▶ ItemLedgerJSON
    │
 ▼（阶段5）
RiskListGenerator ────────────▶ RiskListJSON
```

## 五、模块开发优先级

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P1 | InputParser | 第一个实现，后续模块依赖它 |
| P1 | FactExtractor | 第二个实现，RuleEngine依赖 |
| P1 | RuleEngine | 第三个实现，核心判断逻辑 |
| P1 | SummaryGenerator | 第四个实现，输出最终结果 |
| P2 | LedgerGenerator | 阶段5 |
| P2 | RiskListGenerator | 阶段5 |

**P1可全部完成后，再进入P2。**
