# 推演样本标识优化记录

**日期**：2026-04-15
**优化编号**：MVP-Patch-002
**执行动作**：P1-推演样本"推演"水印标签

---

## 一、为什么需要这个标识

MVP 页面同时承载两类样本：

| 类型 | 示例 | 性质 |
|------|------|------|
| 真实样本 | 6BLIVE（真实周报数据） | 生产参考 |
| 推演样本 | SIMW16~SIMW20 | 演示/验证用 |

在业务方试用或汇报场景中，如果两类样本混用但不加以区分，容易产生误解：**"你们说这周有缺口，是不是真的？你们系统出了什么问题？"**

推演样本标识的作用是：**在不影响阅读体验的前提下，明确告知当前内容是推演数据**。

---

## 二、样式和位置

**位置**：首页结论卡顶部，周期标题右侧，与标题同行排列。

**外观**：
```
2026-04-14 / 04-18 本周期结论  [推演样本]
```

- 字号 11px，比标题小两级，不抢视觉焦点
- 背景半透明白色（`rgba(255,255,255,0.2)`），与深蓝背景自然融合
- 白色描边（`rgba(255,255,255,0.35)`），水印感，不喧宾夺主
- 圆角药丸形状（`border-radius:20px`），轻量感
- `letter-spacing:0.05em`，稍微拉开间距，显得精致

**不采用警告框的原因**：
- 警告框（红/黄底）会让用户第一时间认为是"错误"，而非"样本类型"
- 水印标签是"状态说明"，不是"问题提示"
- 推演样本本身不是负面信息，不需要强调

---

## 三、为什么这样处理不会削弱专业感

1. **位置**：嵌在标题旁，不是显著的大标签，用户主动看到时才注意到
2. **颜色**：白色系，与背景对比度低，不抢主标题
3. **形状**：药丸形，与互联网产品中的"标签"语义一致，专业感强
4. **文字**："推演样本"而非"推演数据"——"样本"暗示这是测试用的数据，语义准确
5. **功能可见性**：仅在 SIMW16~SIMW20 五个推演按钮切换时出现；切换回 6BLIVE 或历史样本时自动消失，避免对真实数据造成干扰

---

## 四、实现说明

**HTML**（静态写入）：
```html
<span id="home-simulation-badge" style="display:none; ...">推演样本</span>
```

**JS**（`renderHomepage` 驱动）：
```javascript
var isSimulation = (currentSample === 'SIMW16' || ...);
document.getElementById('home-simulation-badge').style.display =
    isSimulation ? 'inline-block' : 'none';
```

**切换行为**：
- 点击 W16→W20 任意按钮：出现
- 点击 6BLIVE / 历史1 / 历史2：消失
- 不影响结论卡其他内容

---

## 五、与业内常见实践的对比

| 实践 | 优点 | 缺点 |
|------|------|------|
| 本方案：标题旁药丸标签 | 不破坏阅读，自然提示 | 不够醒目 |
| 大字警告横幅 | 醒目 | 干扰判断，破坏专业感 |
| 角标（右下角） | 不干扰主内容 | 容易被忽略，含义不清 |
| 弹窗提示 | 强制注意 | 过度打断，体验差 |

本方案处于"适度可见"区间，是 MVP 阶段最合理的选择。
