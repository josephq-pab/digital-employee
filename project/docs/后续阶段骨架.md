# 后续阶段骨架

> 以下为后续阶段框架，仅列目标、进入条件、退出条件，不展开实现细节。
> 不得在阶段1未关闭前进入阶段2的实际执行。

---

## 阶段2：原型方案设计

**阶段目标：**
完成原型详细设计方案，包括输入表单设计、输出格式设计、处理流程设计、数据格式规范。

**进入条件：**
- 阶段1《一期术语与判断口径》已获业务方（PMO负责人）书面确认
- 至少1份真实样本数据已到位并完成口径校验（注：阶段4进入需要≥3份）
- 试点项目范围已确认（不少于1个）

**退出条件：**
- 输入字段规范已确定（并写入 INPUT_SCHEMA.md）
- 输出格式已确定（并写入 OUTPUT_SCHEMA.md）
- 原型开发方案已确定（不涉及代码实现）
- 阶段2验收报告已完成

---

## 阶段3：试点运行机制设计

**阶段目标：**
设计一期试点期间的运行机制，包括数据提交流程、审阅流程、反馈收集机制、闭环验证方式。

**进入条件：**
- 阶段2输出文档已确认
- PMO确认试点开始时间
- 确认管理层对接人

**退出条件：**
- 数据提交流程已书面化（谁填、什么时候填、交给谁）
- 审阅流程已书面化（输出给谁、什么时候审、怎么反馈）
- 反馈收集机制已建立
- 阶段3验收报告已完成

---

## 阶段4：原型实现与冒烟测试

**阶段目标：**
完成核心逻辑实现，通过冒烟测试，具备本地可运行状态。

**进入条件：**
- 阶段3运行机制已就绪
- 阶段2方案无未决事项

**退出条件：**
- run_dev.sh 可正常运行演示案例
- run_smoke.sh 全部测试用例通过
- 阶段4验收报告已完成

---

## 阶段5：演示版打磨与MVP收敛

**阶段目标：**
使用真实样本数据完成演示，在PMO试点中验证业务价值，收敛MVP范围。

**进入条件：**
- 阶段4冒烟测试100%通过
- PMO确认试点项目数据已可用
- 管理层确认愿意审阅试点输出

**退出条件：**
- 演示版可处理真实样本数据
- PMO给出书面反馈（接受/不接受/需调整）
- MVP最终范围已确定
- 阶段5验收报告已完成

---

## 各阶段依赖关系

```
阶段1 ──→ 阶段2 ──→ 阶段3 ──→ 阶段4 ──→ 阶段5
  │          │          │          │          │
  └─必须     └─必须     └─必须     └─必须     └─最终
  完成才      完成才      完成才      完成才      验证
  进入下       进入下       进入下       进入下       目标
  一阶段       一阶段       一阶段       一阶段
```

**强制规则：阶段不可跳过，不可并行，必须顺序推进。**
