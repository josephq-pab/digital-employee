# 冻结建设版后续推进规则

> 版本：v1.0（2026-04-16）
> 用途：冻结版建设期间，每轮新增工作必须满足的判定规则

---

## 一、推进规则（三问判定法）

**每轮新增工作，必须能同时回答"是"：**

| # | 问题 | 判断标准 |
|---|------|---------|
| Q1 | 这个工作是否让系统更接近"准投产冻结版"？ | 能让6个冻结维度之一更接近达成 |
| Q2 | 这个工作是否能减少迁移后的改动量？ | 现在做比迁移后做代价更小 |
| Q3 | 这个工作是否在当前环境更值得先做？ | 当前环境可控性高于迁移后环境 |

**三个问题都回答"是"** → 当前可以做  
**任何一个回答"否"** → 默认不做，特殊情况需项目负责人确认

---

## 二、新工作的分类处理

### 当前可做的工作

| 类型 | 示例 |
|------|------|
| 主流程完善 | DATA-001口径修正、归一化逻辑稳定 |
| 体验收口 | MVP页面优化、输出格式稳定 |
| 连续运行验证 | S1第二/三期验证、影子运行 |
| 口径登记 | 补充DATA-001登记、消除判断歧义 |
| 冻结条件冲刺 | 推进B1~B4业务确认 |

### 迁移后做更合理的工作

| 类型 | 示例 | 原因 |
|------|------|------|
| 替换模型 | 换生产级LLM | 生产环境有更好基础设施 |
| 替换环境 | 部署到生产K8s | 生产环境有专业运维 |
| 接生产数据 | 接入真实业务数据库 | 生产环境有真实数据源 |
| 大规模重构 | 拆分微服务 | 生产环境有更大规模需求验证 |

## 三、冻结版建设期间的阶段优先级

### 当前阶段（冻结建设版起点，c1e85bc）

**主攻：A（主流程补齐）+ D（数据口径修正）**

```
高优先：DATA-001口径登记补充、归一化逻辑稳定
中优先：规则判断歧义消除
低优先：其他
```

### 中段阶段（主流程基本冻结后）

**主攻：B（体验收口）+ C（连续运行验证）**

```
高优先：S1第二/三期验证
中优先：MVP页面收口、输出格式锁定
低优先：反馈表单逻辑稳定
```

### 冲刺阶段（主流程+体验基本就绪后）

**主攻：E（冻结条件达成）+ B1~B4业务确认**

```
高优先：业务方确认反馈收集
中优先：冻结版自检、剩余问题关闭
低优先：文档收口
```

## 四、每轮推进的执行标准

**每轮新增工作，必须输出：**

```
工作名称：
所属方向：（A/B/C/D/E）
Q1答案：（是/否 + 理由）
Q2答案：（是/否 + 理由）
Q3答案：（是/否 + 理由）
结论：（当前可做 / 迁移后做 / 不做）
```

**不满足三问判定的工作，禁止直接执行。**

---

## 五、与冻结版主线的关系

**冻结版主线是唯一主线。**

所有推进工作必须属于《冻结建设版任务框架》中的5个方向：

```
方向A：主流程补齐
方向B：体验收口
方向C：连续运行验证
方向D：数据口径修正
方向E：冻结条件达成
```

**不属于5个方向的工作，默认不是当前优先。**

---

## 六、特殊情况处理

**如果遇到以下情况，可以突破三问判定，但需要明确记录：**

| 情况 | 处理方式 |
|------|---------|
| P0/P1阻断性问题 | 立即处理，优先于所有其他工作 |
| 业务方紧急需求 | 评估影响后决定是当前做还是迁移后做 |
| 技术重大风险 | 评估后决定是否当前解决 |

**突破三问判定时，必须记录：**
- 突破原因
- 评估结论
- 项目负责人确认
