# 并行运行机制说明

**版本**：v1.1（更新6C内容，2026-04-11）  
**日期**：2026-04-09（初始）/ 2026-04-11（6C更新）  
**阶段**：阶段6C《第二周期并行运行与体验流程优化》  
**用途**：说明数字员工当前与现有业务机制并行的运行方式

---

## 一、核心原则

**不替代人工**，而是与现有业务机制并行运行。  
**不争论大方向**，只对具体问题做ABC决策。  
**按周运行**，每期产出后由人工复核，有价值的部分逐步扩大使用范围。

---

## 二、当前不替代人工

| 人工负责事项 | 当前数字员工状态 |
|-------------|----------------|
| 最终管理决策 | 仅提供摘要参考，人工决策 |
| 与业务方沟通协调 | 仅提供台账数据，人工沟通 |
| 催报和跟进 | 尚无此功能 |
| 风险升级和处置 | 仅提供风险清单，升级处置由人工负责 |
| 跨部门协调 | 仅提供信息记录，人工协调 |

**当前数字员工定位**：工具助手，不是替代者。

---

## 三、当前与现有业务机制并行

| 现有业务机制 | 并行中的数字员工 |
|-------------|----------------|
| PMO周报处理 | 接收同一份PMO周报，生成并行摘要供对照 |
| 项目台账维护 | 生成独立台账视图，人工保留原台账 |
| 风险点识别 | 提供独立风险清单，人工保留原判断方式 |
| 领导汇报材料 | 生成摘要初稿，人工修改后使用 |

**并行意义**：让业务方在同一数据源上看到两套输出，通过对照发现数字员工的价值和偏差。

---

## 四、数字员工每期产出什么

每运行周期（按周）产出：

1. **领导摘要**（1208-1212期格式）
   - 本周期新增/变更事项
   - 风险清单更新
   - 结项申请状态

2. **事项总台账**
   - 全量事项列表
   - 系统判断列（正常/黄灯/逾期/结项）

3. **风险清单**
   - 本期新增风险项
   - 风险判断依据
   - 历史风险追踪

4. **价值验证记录**
   - 本期与人工判断一致率
   - 偏差案例记录
   - 减负效果说明

---

## 五、人工仍负责什么

| 事项 | 人工动作 |
|------|---------|
| 接收PMO周报 | 人工从原渠道接收 |
| 数据录入 | 人工录入或确认数据来源 |
| 输出复核 | 人工对照现有机制复核数字员工输出 |
| 管理决策 | 人工在数字员工输出基础上做决策 |
| 沟通协调 | 人工负责跨部门沟通 |
| 问题处置 | 人工负责风险升级和处置 |

---

## 六、运行前提

**开始并行运行的条件**：
- PMO周报数据已接入（JSON格式）
- 系统输出格式已与现有机制对齐
- 人工复核角色已指定

**暂停/终止条件**：
- 连续3期偏差率超过20%
- 业务方反馈无法使用输出
- 管理层明确停止

- **独立关注标签**：系统新增"关注（中）"独立状态（橙色badge），与"黄灯"区分
- **指标快照**：数据接入页直接展示指标（M-001/002/003）
- **切片归档**：每周期切片归档至 openclaw_unified_sample_pack/slices/
- **框架文档**：接入记录/运行记录/复核记录/问题登记/价值判断

### 6C流程优化方向

1. **样本接入**：建立更顺畅的上传-识别-路由机制
2. **PMO复核**：轻量化核查+结构化记录
3. **ABC登记**：在线决策登记替代纯线下
4. **周五收口**：标准收口时间线和检查清单
