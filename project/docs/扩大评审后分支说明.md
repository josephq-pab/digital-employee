# 扩大评审后分支说明

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4L《扩大评审准备与真实试运行待命》

---

## 一、三条分支路径

### 分支A：评审通过，进入扩大试点准备

**触发条件：** 评审方认可四周试跑结论，同意扩大验证范围

**最小动作集：**
1. 确认扩大评审参与方名单
2. 确认真实样本提供计划（时间/数量）
3. 确认人工确认流程
4. 启动"真实周报样本接入"准备工作

**预计周期：** 评审通过后 1-2 周内可启动影子运行

---

### 分支B：评审通过，但要求先做真实影子运行

**触发条件：** 评审方认可技术能力，但希望先用真实数据验证，再扩大范围

**最小动作集：**
1. 业务方提供 ≥ 3期 真实周报样本
2. 执行影子运行（按《真实生产数据影子运行SOP》）
3. 评估结果（按《真实试运行结果登记表》）
4. 差异率 ≤ 20% → 分支A继续
5. 差异率 > 20% → 修正 → 再验证

**预计周期：** 样本到位后 1-2 周内完成影子运行评估

---

### 分支C：评审暂缓，维持当前范围

**触发条件：** 评审方认为当前验证仍不充分，或有其他顾虑

**最小动作集：**
1. 记录评审反馈（具体顾虑点）
2. 针对顾虑点制定补充验证计划
3. 补充验证完成后，重新申请评审
4. 继续用现有历史样本做更多验证（不等待真实样本）

**预计周期：** 根据评审反馈内容确定

---

## 二、分支决策矩阵

| 分支 | 核心差异 | 关键决策点 |
|------|---------|-----------|
| A | 直接扩 | 评审方对历史样本验证结果充分信任 |
| B | 先影子运行 | 评审方要求真实数据验证 |
| C | 继续补充验证 | 评审方认为仍有缺口 |

---

## 三、各分支风险

| 分支 | 主要风险 | 应对 |
|------|---------|------|
| A | 真实数据适配性未知 | 密切监控影子运行，发现问题及时修正 |
| B | 样本到位时间不确定 | 业务方尽快提供样本，推进验证 |
| C | 验证周期拉长 | 尽可能用现有历史样本做更多对照，缩短补充验证时间 |
