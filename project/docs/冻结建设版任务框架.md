# 冻结建设版任务框架

> 版本：v1.0（2026-04-16）
> 用途：冻结版建设期间的工作分类框架，不发散功能池

---

## 一、框架说明

**冻结建设版期间，所有工作必须属于以下5个方向之一。不属于这5个方向的工作，默认不做。**

---

## 二、5个冻结建设方向

### 方向A：主流程补齐

**目标：数据口径稳定，规则逻辑稳定，输出格式稳定。**

| 任务 | 当前状态 | 目标 |
|------|---------|------|
| DATA-001 口径登记修正 | 部分登记 | 全部登记并关闭 |
| 归一化逻辑稳定 | 多期结果有差异 | 连续3期结果一致 |
| 规则判断逻辑 | 有歧义项 | 无歧义，可解释 |
| 输出字段定义 | 部分定义 | 全部锁定 |

### 方向B：体验收口

**目标：MVP页面体验达到生产可用，输出文件格式锁定。**

| 任务 | 当前状态 | 目标 |
|------|---------|------|
| MVP页面体验 | 可用但未收口 | 无阻断性体验问题 |
| 领导摘要格式 | 格式存在 | 格式稳定，内容完整 |
| 事项台账格式 | 格式存在 | 格式稳定，字段完整 |
| 风险清单格式 | 格式存在 | 格式稳定，判断准确 |
| 反馈表单 | 可提交 | 逻辑锁定，数据可追溯 |

### 方向C：连续运行验证

**目标：S1 连续3期无P0，影子运行验证完成。**

| 任务 | 当前状态 | 目标 |
|------|---------|------|
| S1 第一期验证 | 已完成 | 继续推进 |
| S1 第二期验证 | 未开始 | 启动 |
| S1 第三期验证 | 未开始 | 启动 |
| 影子运行 | 未开始 | 完成至少1次 |

### 方向D：数据口径修正

**目标：所有口径问题有明确结论，不带模糊问题迁移。**

| 任务 | 当前状态 | 目标 |
|------|---------|------|
| DATA-001 登记补充 | 部分登记 | 全部登记 |
| 战略客户标注范围 | 已标注v2 | 确认冻结 |
| 台账与周报一致性 | 基本一致 | 完全一致或有明确记录 |
| 判断口径歧义 | 存在 | 全部消除或有明确口径记录 |

### 方向E：冻结条件达成

**目标：6个维度冻结条件全部达成（见《准投产冻结版定义》）。**

| 维度 | 当前完成度 | 目标 |
|------|-----------|------|
| 主流程冻结 | 部分 | 全部 |
| 体验收口 | 部分 | 全部 |
| 口径稳定 | 部分 | 全部 |
| 连续运行 | 部分 | 全部 |
| 业务确认 | 0/4 | 3/4+ |
| 边界锁定 | 部分 | 全部 |

---

## 三、工作分类规则

**每轮新增工作，必须能回答：**

```
1. 这个工作属于哪个方向？（A/B/C/D/E）
2. 这个工作让冻结版更接近达成吗？
3. 迁移后还需要做这个工作吗？
```

**如果回答"迁移后还需要做"，→ 属于冻结建设版任务，当前做。**  
**如果回答"迁移后不需要做"，→ 属于生产环境任务，不当前做。**

---

## 四、禁止发散的功能池

**冻结建设版期间，以下方向默认不做：**

| 禁止方向 | 原因 |
|---------|------|
| 新增独立功能模块 | 属于生产级开发，不是冻结版前置 |
| 全新页面设计 | 属于生产级体验，不当前做 |
| 替换模型/算法 | 属于迁移后动作 |
| 接入真实生产数据 | 属于迁移后动作 |
| 大规模代码重构 | 如非阻断性问题，不当前做 |

## 五、框架与阶段的对应关系

| 当前阶段 | 主攻方向 | 说明 |
|---------|---------|------|
| 冻结建设版起点 | A + D | 主流程和口径是最高优先 |
| 冻结建设中段 | B + C | 体验和连续运行 |
| 冻结版冲刺 | E | 冻结条件全部达成 |
| 冻结版达成 | — | 启动最终移交 |
