## 一、规则引擎说明

### 1.1 核心文件

- 路径：`project/app/engine.py`
- 类：`ProjectTracker`
- 方法：`evaluate(project_input: dict) → dict`

### 1.2 规则列表（已实现）

| 规则名称 | 触发条件 | 风险等级 | 升级 |
|---------|---------|---------|------|
| rule_escalation_requested | escalation_requested=True | 高 | 是 |
| rule_严重滞后 | LAG_PATTERNS匹配"严重滞后" | 高 | 是 |
| rule_进度滞后 | LAG_PATTERNS匹配"滞后/延后/behind" | 中 | 否 |
| rule_risk_self_reported | RISK_SELF_REPORTED_PATTERNS匹配 | 中/低 | 否 |
| rule_normal_progress | NORMAL_PATTERNS匹配 | 低 | 否 |

### 1.3 风险等级判断逻辑

```
escalation_requested=True → 高风险 + 升级
严重滞后 → 高风险 + 升级
进度滞后 + 风险自述 → 中风险
进度滞后 → 中风险
风险自述 → 低风险
无信号 → 低风险
```

### 1.4 规则优先级（避免短路）

```
优先级1：escalation_requested（最高）
优先级2：严重滞后（LAG_PATTERNS[0]）
优先级3：进度滞后/停滞/未完成（LAG_PATTERNS[1:4]）
优先级4：风险自述（RISK_SELF_REPORTED_PATTERNS）
优先级5：正常进展（NORMAL_PATTERNS）
```

## 二、判断追溯机制

### 2.1 输出中的追溯字段

每条判断结果包含：

```python
{
    'reasoning': {
        'facts_extracted': [...],      # 提取的事实
        'rules_applied': [...],       # 触发的规则名称
        'raw_signal': str,             # 原始触发词
    },
    'risk_assessment': {
        'level': str,                 # 风险等级
        'triggered_by': [...],        # 触发原因
    }
}
```

### 2.2 追溯示例

```
输入："汽车行业：计划11月启动中试。项目逾期，黄灯继续提示。"

判断结果：
  rules_applied: ['rule_risk_self_reported']
  raw_signal: "责任人自述风险"
  level: 低
  风险等级为什么是低？→ 因为"项目逾期"未被LAG_PATTERNS匹配，
  只触发了risk_self_reported规则（关键词"风险"出现在文本中）
```
