## 二、领导摘要（summary_generator.py）

### 2.1 输出格式（模板A）

```
【领导摘要】
期间：1208-1212, 1215-1219, 1222-1226
总事项数：11
其中：申请结项7 | 需领导介入0 | 黄灯4 | 延期/中风险0 | 正常0

【已完成/已解决】
  ✓ 制定贷款促投放提升方案
  ✓ 外币低息存款上量
  ...

【延期/黄灯事项】
  🟡 汽车行业 | 状态:正常 | 风险:低
     判断依据：责任人自述风险

【需领导介入】
  （空）

【注意】本摘要为provisional版本，待管理层确认格式后生效
```

### 2.2 结构（三层分离）

| 层级 | 内容 | 说明 |
|------|------|------|
| 事实 | 项目名称/承诺/进展 | 原始数据，不可篡改 |
| 判断 | 触发规则/原始信号/风险等级 | 可追溯到engine.py输出 |
| 建议 | 建议动作（最多3条） | engine.py的suggested_actions |

### 2.3 provisional说明

领导摘要当前采用模板A（完整版），基于OUTPUT-ASSUME-01：
- 管理层确认MGT-01后，如选择模板B，需调整summary_generator.py
- 三层结构不变，只是展示格式调整

## 三、事项总台账（ledger_generator.py）

### 3.1 用途

面向PMO/Agent的完整事项清单，用于：
- 核对AI判断是否准确
- 追溯每个事项的来源和映射情况
- 检查哪些事项未匹配到台账

### 3.2 输出字段

```python
{
    '事项名称': str,
    '来源': str,            # weekly_report / ledger
    '期间': str,
    '责任人': str,          # provisional
    '整体状态': str,        # 正常/关注/危险
    '风险等级': str,        # 低/中/高
    '升级建议': str,        # 是/否
    '映射状态': str,        # 已映射(fuzzy)/未映射(unmatched)
    '判断规则': str,        # rule_xxx, rule_yyy
    '原始信号': str,        # 引擎返回的raw_signal
    '进展描述': str,
}
```

## 四、风险清单（risk_list_generator.py）

### 4.1 用途

聚焦高优先级风险，用于：
- 快速识别需要关注的事项
- 内部校验AI判断是否捕捉到真实风险

### 4.2 分类

| 分类 | 条件 |
|------|------|
| 需领导介入 | escalation=True 或 risk_level=高 |
| 黄灯关注 | is_yellow_alert=True |
| 延期/中风险 | risk_level=中 |

### 4.3 QUAL-02修复

当前风险清单包含结项申请（不应在风险清单中）。QUAL-02修复后需重新生成。
