## 二、5维验收结果

| 维度 | 检查项 | 检查方式 | 结果 |
|------|--------|---------|------|
| 字段完整性 | 必填字段是否存在 | 对照INPUT_SCHEMA | □ 通过 □ 不通过 |
| 时间信息 | promised_items含时间词 | 搜索时间关键词 | □ 通过 □ 不通过 |
| 状态可判定 | current_progress含判断关键词 | 搜索滞后/完成词 | □ 通过 □ 不通过 |
| 责任人 | owner字段有内容 | 目视 | □ 通过 □ 不通过 |
| 文字可解析 | 文字>10字符，非纯符号 | 目视 | □ 通过 □ 不通过 |

## 三、验收结果

| 结果 | 条件 | 处置方式 |
|------|------|---------|
| **通过** | 5维全部通过 | 直接进入样本库，更新registry |
| **部分通过** | 3-4维通过 | 降级使用，注明缺口，不进入正式测试 |
| **退回补充** | <3维通过 | 退回FU补充，注明缺失项 |
| **仅可局部使用** | 有维度不通过但可接受 | 进入样本库，注明仅可用于某类测试 |

## 四、不同结果的后续动作

### 结果A：通过

```
样本验收通过
    │
    ▼
更新registry.json（accepted=true, usable=true）
    │
    ▼
通知Agent：S1/S2样本新增一份，路径如下
    │
    ▼
样本文件存档到对应目录（s1/ 或 s2/）
    │
    ▼
更新《样本收集跟踪表》
```

### 结果B：部分通过（降级使用）

```
样本部分通过
    │
    ▼
更新registry.json（accepted=true, usable=false, gap_notes说明）
    │
    ▼
通知Agent：样本降级，不可用于正式测试，仅可用于XXX
    │
    ▼
记录缺口（哪些维度不通过）
    │
    ▼
询问FU是否可补充，无法补充则降级存档
```

### 结果C：退回补充

```
样本退回
    │
    ▼
填写退回说明（见下方模板）
    │
    ▼
通知FU：样本不符合验收标准，请补充后重新提交
    │
    ▼
退回说明：
    - 缺失维度：_______________
    - 具体缺口：_______________
    - 补充要求：_______________
    │
    ▼
FU补充后，重新提交，重新验收
```

### 结果D：仅可局部使用

```
样本局部可用
    │
    ▼
更新registry.json（accepted=true, usable=false, gap_notes说明）
    │
    ▼
通知Agent：样本可用于XXX测试，不可用于YYY测试
    │
    ▼
登记缺口，更新跟踪表
```
