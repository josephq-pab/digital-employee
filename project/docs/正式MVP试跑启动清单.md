## 三、试跑首周要做什么

| 日期 | 核心任务 |
|------|---------|
| Day 1 | 启动正式试跑 → 收集样本 → 执行 → 人工复核 → 发出首份试跑周报 |
| Day 2-3 | 继续收集新样本 → 执行 → 观察输出稳定性 |
| Day 4-5 | 收集反馈 → 更新问题登记 → 判断是否继续 |
| Day 7 | 发出首周结论 → 判断是否建议继续试跑 |

**首周结论判断：**
- 继续：输出稳定，问题≤P2，可继续
- 暂停：发现P0阻断或规则逻辑根本性错误，暂停并回退
- 调整：部分问题需修复，调整后继续
