## 一、判断规则总览

### 1.1 规则优先级（匹配顺序至关重要）

```
优先级1：escalation_requested = true
  → 立即触发【高风险 + 建议升级】
  → 跳过后续所有检查

优先级2：进度严重滞后
  → 触发【高风险 + 建议升级】
  → 跳过后续所有检查

优先级3：进度滞后 + 风险自述同时存在
  → 触发【中风险 + 不建议升级】

优先级4：仅有进度滞后
  → 触发【中风险 + 不建议升级】

优先级5：仅有风险自述（关键词命中）
  → 触发【低风险 + 不建议升级】

优先级6：以上均不满足
  → 触发【低风险 + 不建议升级】
```

## 二、关键词模式库

### 2.1 进度正常类关键词（满足则记录为正常信号）

| 模式 | 正则 | 含义 |
|------|------|------|
| 完成 | `完成\|已交付\|已上线\|已发布\|done` | 事项已完成 |
| 正常推进 | `正常\|按计划\|顺利\|on track\|on schedule` | 进展正常 |

### 2.2 进度异常类关键词（优先级从高到低）

| 优先级 | 模式 | 正则 | 触发结果 |
|--------|------|------|---------|
| P1（最高） | 严重滞后 | `严重滞后\|大幅落后\|critical delay` | 高风险+建议升级 |
| P2 | 停滞 | `停滞\|卡住\|blocked\|stuck` | 中风险 |
| P3 | 一般滞后 | `滞后\|延后\|behind` | 中风险 |
| P4 | 未完成 | `未完成\|未到位\|incomplete` | 中风险 |

> **重要：** 优先级高的模式必须先匹配，否则"严重滞后"会被"滞后"先匹配，导致判断降级。

### 2.3 风险自述关键词

| 模式 | 正则 | 触发结果 |
|------|------|---------|
| 风险词 | `风险\|不确定\|可能影响\|障碍\|困难` | 低风险 |

> 系统只记录风险自述，不对风险内容做真伪判断，内容原样透传。

### 2.4 无明确进展判定

满足任一即为"无明确进展"：
- current_progress 连续两期文字重合度 > 70%（跨周期对比，阶段5实现）
- current_progress 包含"待、等、暂无、尚未"且无后续计划
- current_progress 为空字符串

> 一期原型只实现"当前周期"的判断，跨周期对比在阶段5实现。

## 三、状态机定义

### 3.1 状态定义

| 状态名 | 含义 | 对应输出 |
|--------|------|---------|
| NORMAL | 无风险信号，进展正常 | overall=正常, risk_level=低 |
| ATTENTION | 存在需要关注的情况 | overall=关注, risk_level=中 |
| DANGER | 存在高风险，需尽快处理 | overall=危险, risk_level=高 |

### 3.2 状态转换图

```
                    escalation=true
    ┌─────────────────────────────────┐
    │                                 ▼
    │                          ┌───────────┐
    │  进度严重滞后             │  DANGER   │
    │────────────────────────▶ │ risk_level │
    │                          │  = 高     │
    │                          └───────────┘
    │
    │  进度滞后              ┌───────────┐
    │───────────────────────▶ │ ATTENTION │
    │  risk_points命中       │ risk_level│
    │───────────────────────▶ │  = 中     │
    │                          └───────────┘
    │
    │  有risk_points关键词     ┌───────────┐
    │───────────────────────▶ │  NORMAL   │
    │                         │ risk_level │
    │                         │  = 低     │
    └───────────────────────▶ └───────────┘
                    无任何信号
```

### 3.3 升级建议逻辑

| 状态 | 升级建议 | 说明 |
|------|---------|------|
| DANGER | escalation_recommended = true | 系统强建议升级 |
| ATTENTION | escalation_recommended = false | 系统建议不升级，人工决定 |
| NORMAL | escalation_recommended = false | 无需升级 |

## 四、单周期判断执行流

```
输入 current_progress + risk_points
         │
         ▼
① 检查 escalation_requested
  ├─ true  → [高风险, 建议升级, overall=危险] → 结束
  └─ false → 继续
         │
         ▼
② 进度严重滞后检查（模式库P1优先）
  ├─ 命中  → [高风险, 建议升级, overall=危险] → 结束
  └─ 未命中 → 继续
         │
         ▼
③ 进度滞后检查（P3）
  ├─ 命中  → 检查 risk  │_points
         ├─ 风险词命中 → [中风险, 不升级, overall=关注]
  │         └─ 无风险    → [中风险, 不升级, overall=关注]
  └─ 未命中 → 继续
         │
         ▼
④ 风险自述关键词检查
  ├─ 命中  → [低风险, 不升级, overall=正常]
  └─ 未命中 → [低风险, 不升级, overall=正常]
```

## 五、判断规则可执行版本（可直接转为代码）

```python
# 伪代码，可直接转化为 Python 实现
def evaluate(project_record):
    signals = []
    rules_fired = []

    # ① escalation_requested
    if project_record.escalation_requested:
        return make_result("高", True, "危险", ["请求领导介入"], ["rule_escalation"])

    # ② 进度检查（按优先级）
    progress = project_record.current_progress
    for pattern, label, level, escalation in [
        (r"严重滞后|大幅落后|critical delay", "严重滞后", "高", True),
        (r"停滞|卡住|blocked|stuck", "停滞", "中", False),
        (r"滞后|延后|behind", "滞后", "中", False),
        (r"未完成|未到位|incomplete", "未完成", "中", False),
    ]:
        if re.search(pattern, progress, re.I):
            signals.append(label)
            rules_fired.append(f"rule_{label}")
            risk_level = level
            esc = escalation
            break

    # ③ 风险自述检查
    risk = project_record.risk_points or ""
    if re.search(r"风险|不确定|可能影响|障碍|困难", risk, re.I):
        signals.append("责任人自述风险")
        rules_fired.append("rule_risk_self_reported")

    # ④ 决定最终结果
    if risk_level == "高":
        overall = "危险"
    elif signals:  # 有任何信号
        overall = "关注"
    else:
        overall = "正常"

    return make_result(risk_level, esc, overall, signals, rules_fired)
```

## 六、必人工确认的判断（规则不能覆盖）

以下情况，系统给出建议，最终由人工判断：

| 情况 | 系统给出 | 人工决定 |
|------|---------|---------|
| 延期可接受 | 风险等级建议 | 是否实际升级 |
| 风险真伪 | 记录+透传 | 是否采纳 |
| 事项是否真正完成 | 关键词判定 | 人工最终确认 |
| 是否真正需要领导介入 | escalation_recommended | 人工最终决定 |

## 七、规则更新与扩展规范

后续新增/修改规则必须：
1. 在本文件中新增条款，注明版本号和日期
2. 在 EVAL_RULES.md 中同步新增/修改对应用例
3. 规则变更必须记录"变更原因"和"变更前后对比"
4. 由业务方（PMO负责人）书面确认后方可生效
