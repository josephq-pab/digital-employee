## 三、适配处理（normalizer.py）

### 3.1 周报归一化（normalize_weekly_report）

将单期周报归一化为标准事项列表，处理三类事项：

| 来源 | 处理方式 | 对应规则 |
|------|---------|---------|
| key_alerts（黄灯） | alert_item → risk_points + is_yellow_alert | 黄灯触发中风险 |
| closure_requests（结项申请） | closure_item → is_closure_request=True | 结项申请视为已完成 |
| items_requiring_leadership_attention | summary推断 → escalation_requested=True | 推断升级需求 |

### 3.2 缺失字段处理

| 缺失字段 | 处理方式 | 依据 |
|---------|---------|------|
| 周报无owner字段 | 以台账owner_department替代（provisional） | ADPT-ASSUME-01 |
| 无escalation_requested字段 | 以management_summary推断（provisional） | ADPT-ASSUME-03 |
| risk_points无标准字段 | 从key_alerts.raw+summary推断（provisional） | ADPT-ASSUME-02 |

### 3.3 台账归一化（normalize_ledger_record）

按周期提取台账主记录的进展文本，返回标准化格式。

### 3.4 标准输出格式

每条归一化事项包含：

```python
{
    'project_name': str,         # 事项名称
    'period': str,               # 期间
    'source': str,               # 来源（weekly_report/ledger）
    'owner': str,                # 责任人（provisional）
    'promised_items': str,       # 承诺事项
    'current_progress': str,     # 当前进展
    'risk_points': str,          # 风险描述（provisional）
    'escalation_requested': bool,# 是否请求升级（provisional）
    'is_closure_request': bool,  # 是否结项申请
    'is_yellow_alert': bool,     # 是否黄灯
    'ledger_record_id': str,     # 台账ID（如已映射）
}
```
