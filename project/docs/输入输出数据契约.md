# 输入输出数据契约

**版本：** v1.0
**状态：** 已定稿，待PMO确认
**覆盖范围：** 周报/月报格式、项目台账格式、领导摘要、事项台账、风险清单

---

## 一、输入契约

### 1.1 格式A：周报/月报格式（标准JSON）

**触发条件：** 用户以周报或月报形式提交，字段为自由文本

| 字段名 | 类型 | 必填 | 来源 | 说明 |
|--------|------|------|------|------|
| `project_name` | string | 是 | 用户填写 | 项目名称 |
| `owner` | string | 是 | 用户填写 | 责任人姓名 |
| `report_type` | string | 否 | 自动/用户填 | weekly/monthly，默认weekly |
| `report_date` | string | 否 | 自动生成 | ISO日期，YYYY-MM-DD |
| `promised_items` | string | 是 | 用户填写 | 本周期承诺完成的工作，自由文本 |
| `current_progress` | string | 是 | 用户填写 | 本周期实际进展，自由文本 |
| `risk_points` | string | 否 | 用户填写 | 风险自述，可空字符串 |
| `escalation_requested` | boolean | 否 | 用户填写 | 是否主动请求领导介入，默认false |

**JSON Schema（参考实现用）：**

```json
{
  "type": "object",
  "required": ["project_name", "owner", "promised_items", "current_progress"],
  "properties": {
    "project_name":   {"type": "string", "minLength": 1},
    "owner":         {"type": "string", "minLength": 1},
    "report_type":   {"type": "string", "enum": ["weekly", "monthly"], "default": "weekly"},
    "report_date":   {"type": "string", "format": "date"},
    "promised_items":{"type": "string", "minLength": 1},
    "current_progress": {"type": "string", "minLength": 1},
    "risk_points":   {"type": "string", "default": ""},
    "escalation_requested": {"type": "boolean", "default": false}
  }
}
```

### 1.2 格式B：项目台账格式（标准JSON）

**触发条件：** 用户以项目台账形式提交，每个事项有独立进度

| 字段名 | 类型 | 必填 | 来源 | 说明 |
|--------|------|------|------|------|
| `project_name` | string | 是 | 用户填写 | 项目名称 |
| `owner` | string | 是 | 用户填写 | 责任人姓名 |
| `last_updated` | string | 否 | 自动生成 | ISO日期，YYYY-MM-DD |
| `items` | array | 是 | 用户填写 | 事项列表，结构见下方 |
| `overall_risk_points` | string | 否 | 用户填写 | 总体风险自述，可空 |
| `escalation_requested` | boolean | 否 | 用户填写 | 是否主动请求介入，默认false |

**items数组中每个事项的结构：**

| 子字段 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `item_id` | string | 是 | 事项唯一标识符 |
| `promised` | string | 是 | 承诺的里程碑或交付物 |
| `current_progress` | string | 是 | 当前进展描述 |
| `status` | string | 否 | 状态标签：in_progress/completed/blocked |

### 1.3 容错与降级规则

| 场景 | 处理规则 |
|------|---------|
| risk_points为空 | 视为无风险自述 |
| escalation_requested缺失 | 默认为false |
| report_type/report_date缺失 | 自动填充默认值 |
| items数组缺失 | 降级为格式A处理 |
| items数组存在但为空 | 报错，要求至少一个事项 |
| promised_items过长（>5000字） | 截断至5000字符，并标记truncated=true |

## 二、输出契约

### 2.1 领导摘要输出（必须）

**用途：** 晨会/周报审阅、投影/打印

```json
{
  "report_id": "uuid-v4",
  "generated_at": "ISO8601时间戳",
  "project_name": "string",
  "owner": "string",
  "status_summary": {
    "overall": "正常 | 关注 | 危险",
    "risk_level": "低 | 中 | 高",
    "progress_text": "string（≤200字）",
    "completion_estimate": "string（≤100字）",
    "no_clear_progress": false
  },
  "risk_assessment": {
    "level": "低 | 中 | 高",
    "triggered_by": ["string"],
    "summary": "string",
    "risk_points_present": true
  },
  "suggested_actions": ["string"],
  "escalation_recommended": true,
  "escalation_indicator": "system_suggested | user_requested",
  "reasoning": {
    "facts_extracted": ["string"],
    "rules_applied": ["string"],
    "raw_signal": "string"
  }
}
```

### 2.2 事项总台账输出（扩展，阶段5要求）

**用途：** PMO存档、跨周期对比追踪

```json
{
  "project_name": "string",
  "items": [
    {
      "item_id": "string",
      "promised": "string",
      "current_progress": "string",
      "status": "completed | in_progress | unclear | blocked",
      "progress_signal": "完成类关键词 | 滞后类关键词 | 停滞类关键词 | 空",
      "judgment": "string（判定依据）"
    }
  ]
}
```

### 2.3 风险清单输出（扩展，阶段5要求）

**用途：** 风险专项汇报

```json
{
  "project_name": "string",
  "risks": [
    {
      "risk_id": "R-001",
      "source": "risk_points字段 | progress描述",
      "description": "string（原文透传）",
      "severity": "高 | 中 | 低",
      "response_required": true,
      "escalation_recommended": true
    }
  ]
}
```

## 三、字段说明

| 字段路径 | 说明 |
|---------|------|
| status_summary.no_clear_progress | 标识是否存在"无明确进展"情况 |
| status_summary.overall | 综合状态：正常/关注/危险 |
| status_summary.risk_level | 风险等级：低/中/高 |
| risk_assessment.triggered_by | 触发当前风险等级的原始条件列表 |
| escalation_recommended | 系统是否建议升级（最终由人工确认） |
| escalation_indicator | 升级来源：system_suggested（系统建议）或user_requested（用户主动请求） |
| reasoning.raw_signal | 用于人工复核的原始关键词，便于追溯判断链路 |

---

## 四、输入输出映射关系

```
周报/月报JSON → 输入解析层 → 内部记录(InternalRecord)
                                    ↓
                               规则引擎判断
                                    ↓
领导摘要JSON ←── 输出生成层 ←── 评判结果(JudgementResult)
事项台账JSON ←── 输出生成层 ←── 评判结果 + items数据（阶段5）
风险清单JSON ←── 输出生成层 ←── 评判结果 + risk_points（阶段5）
```
