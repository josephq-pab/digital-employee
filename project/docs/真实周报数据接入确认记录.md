## 一、样本JSON格式基准（供参考对比）

**文件路径：** `samples/samples/weekly_reports/weekly_report_series_latest3.json`

**JSON结构：**
```json
[
  {
    "sample_type": "pmo_weekly_report",
    "sample_version": "v1",
    "is_desensitized": true,
    "report_meta": {
      "report_id": "PMO-WR-2025-1222-1226",
      "report_period": "1222-1226",
      "report_date_text": "二零二五年十二月二十六日",
      "reporting_unit": "某总行条线PMO",
      "report_scope": "重点项目推进周报"
    },
    "project_overview": {
      "total_items": 60,
      "category_1_plus_3_items": [...],
      "other_key_items": [...],
      "closed_or_paused_or_terminated": [...],
      "in_progress_items": [...],
      "status_light": "green"
    },
    "key_requests": {
      "closure_requests": [...],
      "delay_requests": [...],
      "termination_requests": [...]
    },
    "key_alerts": [...],
    "new_items": [...],
    "management_summary": {
      "overall_judgment": "...",
      "completed_or_resolved": [...],
      "delayed_items": [...],
      "no_clear_progress_items": [...],
      "items_requiring_leadership_attention": [...]
    },
    "source_note": { "source_form": "...", "removed_content": "..." }
  }
]
```

**必要字段（系统运行所依赖）：**

| 字段路径 | 类型 | 说明 | 备注 |
|---------|------|------|------|
| `report_meta.report_period` | string | 期数标识，格式如"1222-1226" | ❌ 不可缺 |
| `report_meta.report_date_text` | string | 日期文本 | ✅ 可缺（有默认值） |
| `project_overview.total_items` | int | 事项总数 | ❌ 不可缺 |
| `project_overview.status_light` | string | 状态灯：green/yellow/red | ✅ 可缺 |
| `key_requests.closure_requests` | array | 结项申请列表 | ✅ 可空 |
| `key_requests.delay_requests` | array | 延期申请列表 | ✅ 可空 |
| `key_alerts` | array | 黄灯预警列表 | ✅ 可空 |
| `management_summary.overall_judgment` | string | 总体判断 | ✅ 可缺（系统可生成） |

## 三、格式差距处理方案

**若JSON格式不完全兼容，处理方式如下：**

| 差距类型 | 系统处理方式 | 需要的步骤 |
|---------|------------|---------|
| 字段名称不同 | normalizer中做字段映射 | 开发方配置 |
| 缺少可选字段 | 使用默认值 | 无需处理 |
| 缺少必要字段 | 报错，无法运行 | 联系项目方补充 |
| 格式完全不同 | 需转换脚本 | 开发方编写 |
