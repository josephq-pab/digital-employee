## 二、证据链详情

### E1：开发样本验证

**验证目标：** 基础功能（适配→归一化→规则判断→输出生成）端到端可跑通

**样本：** slices/01_dev_core_december.json（December 3期）
**执行时间：** 2026-04-08 08:49
**结果：** 11事项（结项7 + 黄灯4），7/7 PASS

**证明了什么：**
- 适配器可将横杠前缀 alert_lines 正确解析
- 核心引擎（归一化/规则/生成器）端到端无结构性报错
- 输出一致性：领导摘要/台账/风险清单全部生成

---

### E2：连续运行验证

**验证目标：** 同一样本重复运行，输出结果稳定一致

**样本：** 同 E1，重复执行 MVP-DRYRUN-01 和 MVP-DRYRUN-02
**执行时间：** 2026-04-07 / 2026-04-08 05:17
**结果：** 两次运行输出结构完全一致

**证明了什么：**
- 系统无随机性因素
- 输出结果可复现
- 满足"可审计、可追溯"要求

---

### E3：留出样本验证

**验证目标：** 适配器可处理 November 格式（alert_lines 含"申请结项"混入 + request_lines 仅标题）

**样本：** slices/02_holdout_validation_november.json（November 4期：1103-1128）
**执行时间：** 2026-04-08 16:26
**关键修复：** W3-P1（结项误合并）和 W3-P2（延期漏识别）
**结果：** 17事项（黄灯15 + 结项1 + 延期1），7/7 PASS

**证明了什么：**
- 修复 W3-P1 后："搭建战略客户样板"从误混入黄灯 → 正确归入结项
- 修复 W3-P2 后："数字经营"延期申请从漏识别 → 正确识别
- 适配器支持"标题+raw_text配对"格式

---

### E4：压力回归验证

**验证目标：** 适配器可处理 Aug/Sep 格式（alert_lines 数字编号 + request_lines 空数组 + closure 在 raw_text 独立段落）

**样本：** slices/03_stress_regression_aug_sep.json（Aug/Sep 9期：0728-0926）
**执行时间：** 2026-04-08 18:02
**结果：** 38事项（黄灯33 + 结项4 + 延期1），全部稳定

**关键证据：**
- 0728-0801：5条黄灯 + 2条结项申请 → 正确识别"投行派生存款增存"和"地产行业策略专项汇报"的终止申请
- 0908-0912：识别出"海南分行FT账户联动方案"延期申请
- 适配器扩展：数字编号正则 + raw_text 申请段落解析，无核心引擎改动

**证明了什么：**
- 适配器可处理数字编号 alert_lines（1./2./3. 前缀）
- 适配器可处理 request_lines 空数组场景（从 raw_text 提取 closure）
- 高黄灯比例（86.8%）样本下系统仍然稳定
- 核心引擎未因适配层扩展而引入回归
