# 试点范围确认外发版

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 发给PMO协调员+负责人，确认试点项目范围
**页数：** 建议1页

---

## 一、背景

数字员工一期原型已完成工程开发，即将进入真实数据验证阶段。需要确认具体的试点项目范围，以便：
1. 收集试点项目的历史数据（周报+台账）
2. 确定原型演示时的展示项目
3. 定义试点期的成功标准

---

## 二、我们需要确认的事项

### 2.1 试点项目数量

**最低要求：** 不少于1个项目

请确认以下选项之一：
- [ ] **1个项目**（最小闭环，快速验证）
- [ ] **2-3个项目**（扩大验证范围）
- [ ] **更多项目**（请注明数量）

---

### 2.2 试点项目选择标准

我们建议试点项目符合以下条件（请选择）：

| 标准 | 说明 | 是否接受 |
|------|------|---------|
| 重要性 | 属于PMO重点跟踪项目 |  |
| 数据可用性 | 有完整的历史周报+台账数据 |  |
| 代表性 | 能代表一类项目的执行特征 |  |
| 时效性 | 当前处于执行中阶段（非已结项） |  |

---

### 2.3 试点期的运作方式

| 项目 | 内容 |
|------|------|
| 数据提供 | PMO提供试点项目的历史周报+台账（已脱敏） |
| 验证方式 | Agent用试点数据运行原型，输出状态摘要 |
| 人工复核 | PMO逐条核对AI判断是否准确 |
| 调整机制 | 有问题则在试点期内调整规则，不影响原型的可演示性 |

---

### 2.4 试点期的时长

建议：2-4周

期间我们会：
- 每周运行一次原型
- 产出每周状态摘要
- 记录判断准确率
- 试点结束后汇总试点报告

---

## 三、确认后对各方的影响

| 对象 | 影响 |
|------|------|
| PMO | 需要提供1-3个项目的历史数据（已脱敏） |
| 管理层 | 会定期收到试点项目的AI状态摘要 |
| 项目责任人 | 会收到自己项目的AI状态摘要（可选择是否回复确认） |
