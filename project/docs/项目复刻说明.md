# 项目复刻说明

**版本：** v1.0
**适用场景：** 将本项目复制到另一个工作区、另一个Agent、或另一台机器继续执行

---

## 一、复刻前提条件

复刻前请确认目标环境：
- Python 3.8+ 已安装
- 有写入权限的工作目录
- 无需任何外部网络依赖（一期原型完全离线）

---

## 二、最小复刻步骤

### 2.1 步骤一：复制项目目录

```bash
# 在目标机器/工作区执行
mkdir -p new-workspace
cd new-workspace

# 复制整个project目录
cp -r original-workspace/project ./project

# 复制根目录的workspace配置文件
cp original-workspace/AGENTS.md ./
cp original-workspace/SOUL.md ./
cp original-workspace/USER.md ./
cp original-workspace/IDENTITY.md ./
```

### 2.2 步骤二：验证目录结构

```bash
find ./project -name "*.md" | grep -v __pycache__ | sort
# 应看到 docs/ 目录下 ≥23 份文档
```

### 2.3 步骤三：验证脚本可执行

```bash
cd project
bash ../scripts/run_smoke.sh
# 应显示 冒烟测试 执行结果
```

### 2.4 步骤四：确认无硬编码路径

检查以下文件中是否存在绝对路径（不应存在）：
- `scripts/run_dev.sh`
- `scripts/run_smoke.sh`
- `scripts/run_report.sh`

如有硬编码路径，替换为相对路径或环境变量引用。

## 三、复刻后的初始化检查清单

| # | 检查项 | 验证方式 | 如有问题 |
|---|------|---------|---------|
| 1 | 文档数量 | `find docs -name "*.md" \| wc -l` 应≥23 | 检查缺失文件 |

## 四、更换试点项目的方法

如需将试点从"项目A"更换为"项目B"，且规则和口径不变：

### 4.1 需要更新的内容

| 内容 | 更新方式 |
|------|---------|
| 样本数据 | 用新项目的脱敏样本替换 data/samples/ 中的样本 |
| 试点范围定义 | 更新《决策记录与待确认事项》中的Q04 |
| 试点运行SOP | 更新《试点运行机制》中的FU（填报人）信息 |

### 4.2 不需要更新的内容

- 术语口径
- 判断规则
- 输入输出格式
- 模块设计
- 目录结构

---

## 五、更换试点项目需要重做的内容

| 内容 | 原因 |
|------|------|
| S1/S2样本数据 | 原始数据不同 |
| 试点SOP中的填报人/责任人 | 人员不同 |
| 试点运行SOP中的对接人信息 | 人员不同 |
| 演示素材（M1/M2/M3） | 样本不同 |

## 六、规则迁移方法

如需将本项目的规则迁移到另一个相似项目：

### 6.1 规则迁移步骤

1. 复制 `project/docs/判断规则与状态机说明.md` 到新项目
2. 复制 `project/app/engine.py`（参考实现）到新项目
3. 根据新项目调整关键词模式库（在 patterns.py 中）
4. 用新项目的样本数据跑冒烟测试
5. 根据测试结果调整规则

### 6.2 规则迁移注意事项

- 规则优先级顺序不得改变（严重滞后必须先于一般滞后）
- escalation=true 的强制升级逻辑不得改变
- 判断依据（reasoning字段）必须保留
- 术语定义如需调整，必须走变更流程

## 八、常见复刻问题处理

| 问题 | 原因 | 处理方式 |
|------|------|---------|
| run_smoke.sh 报错"找不到engine" | Python路径问题 | 检查 `sys.path.insert(0, 'project/app')` |
| JSON解析报错 | Python版本问题 | 确认≥3.8 |
| 文档显示乱码 | 文件编码问题 | 确认UTF-8编码 |
| 脚本无执行权限 | 权限问题 | `chmod +x scripts/*.sh` |
