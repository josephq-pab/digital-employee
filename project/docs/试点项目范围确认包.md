# 试点项目范围确认包

**版本：** v1.0
**发出方：** 数字员工项目组
**接收方：** PMO协调员 / 项目负责人
**用途：** 请确认一期试点的具体项目范围

---

## 一、确认包概述

本包用于确认"一期试点具体是哪个或哪类项目"。
确认后，试点的项目边界将被正式锁定。

---

## 二、一期试点项目选择标准

以下条件须**同时满足**：

| # | 标准 | 说明 |
|---|------|------|
| S1 | PMO重点跟踪的项目 | 有明确的PMO负责人和定期汇报要求 |
| S2 | 有明确的责任人 | 项目负责人可定期提交周报/月报 |
| S3 | 有结构化数据 | 责任人可提供符合格式A或格式B的数据 |
| S4 | 已有或计划有进展汇报 | 不是僵尸项目，有实际推进 |
| S5 | 管理层关注 | 管理层希望看到这个项目的状态摘要 |

## 四、"至少1个试点项目"的定义

**有效试点项目须同时满足：**
1. 有明确的项目名称
2. 有可联系的责任人（FU）
3. PMO确认该项目属于"重点跟踪"范畴
4. 责任人承诺按周/月提交数据

**无效试点项目（不可接受）：**
- 只有项目代号，无实际责任人
- 责任人无法承诺定期提交
- 项目处于暂停/僵尸状态
- 只有历史数据，无持续更新计划
