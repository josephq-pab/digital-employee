## 四、演示后决策收口机制结论

### 4.1 三项决策机制

| 决策项 | 决策问题 | 选项 |
|--------|---------|------|
| A | 是否认可当前输出结构 | 同意 / 有条件同意 / 暂缓 |
| B | 是否同意真实样本到位后进入影子运行 | 同意 / 有条件同意 / 暂缓 |
| C | 是否先推进扩大评审 | 同意 / 有条件同意 / 暂缓 |

### 4.2 三分支走向

```
决策记录填写完成
    ↓
A同意 且 B同意/有条件 → 分支A：演示认可 → 真实样本影子运行
A/B有条件同意 → 分支B：先修小问题 → 再进入影子运行
C同意（评审优先）→ 分支C：先推进扩大评审 → 再进入影子运行
```

### 4.3 收口机制特点

- **有记录**：每项决策有模板有签字栏
- **有分类**：4类反馈有登记表，不遗漏
- **有去向**：3分支有明确进入条件和最小动作
- **有约束**：明确"不能宣称"的事项，不越界
