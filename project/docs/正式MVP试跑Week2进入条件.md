# 正式MVP试跑Week2进入条件

**版本：** v1.0
**日期：** 2026-04-08
**阶段：** 阶段4G《正式MVP试跑启动与首周运行》
**适用版本：** MVP-RUN-01（Week 2运行）

---

## 一、Week 2进入门槛

以下条件**必须全部满足**方可进入Week 2运行：

| # | 门槛条件 | Week 1结果 | 是否满足 |
|---|---------|-----------|---------|
| 1 | P0问题数量 = 0 | 0 | ✅ |
| 2 | P1问题数量 = 0 | 0 | ✅ |
| 3 | 冒烟测试通过率 ≥ 5/7 | 7/7 PASS | ✅ |
| 4 | 三件套（领导摘要+台账+风险清单）全部生成 | 全部生成 | ✅ |
| 5 | 阶段判断结论为"继续"或"调整后继续" | 继续 | ✅ |
| 6 | 上周观察项未升级为P0/P1 | 未升级 | ✅ |

**结论：** ✅ **满足全部门槛条件，可进入Week 2**

---

## 二、Week 2继续运行条件

Week 2运行期间，如发生以下情况应**暂停**并启动评估：

| # | 暂停条件 | 说明 |
|---|---------|------|
| 1 | 发现P0问题 | 立即暂停，回退或修复 |
| 2 | P1问题数量 ≥ 3 | 暂停评估，判断是否继续 |
| 3 | 冒烟测试连续2次 < 5/7 | 暂停，检查原因 |
| 4 | 样本严重不准确导致误判 | 暂停，补充/更换样本 |
| 5 | 管理层或PMO要求暂停 | 立即暂停 |

---

## 三、Week 2回退条件

如发生以下情况，应**回退到演示版/待命包**：

| # | 回退条件 | 说明 |
|---|---------|------|
| 1 | 规则被确认根本性错误 | 暂停，回退到Q01确认前版本 |
| 2 | 核心判断逻辑被否定 | 暂停，重新走Q01确认流程 |
| 3 | 试点项目根本性变更 | 暂停，更新Q04，重新确认 |
| 4 | 连续3次运行均发现P0 | 回退，修复后重新进入 |
| 5 | 管理层/PMO明确要求回退 | 立即回退 |
