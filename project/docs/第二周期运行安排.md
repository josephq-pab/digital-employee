## 三、系统运行节奏

### 3.1 运行流程

```
样本到位 → 标准化切片 → 系统内运行 → 生成领导摘要/台账/风险清单/运行日志 → 进入业务复核 → 周五收口
```

### 3.2 各环节时效要求

| 环节 | 时效要求 | 责任方 |
|------|---------|-------|
| 样本补齐通知 | 样本到位后1小时内 | 数据提供方 |
| 标准化切片 | 样本到位后2小时内 | 系统/Agent |
| 系统内运行 | 切片完成后4小时内 | 系统 |
| 领导摘要生成 | 运行完成后1小时内 | 系统 |
| 业务复核 | 摘要生成后24小时内 | PMO协调人 |
| ABC决策登记 | 复核完成后当日 | PMO协调人 |
| 周五收口 | 每周五16:00前 | PMO协调人+项目组 |

---

## 四、业务复核要求

### 4.1 复核内容

| 复核项 | 复核内容 | 填写方 |
|--------|---------|-------|
| 判断一致性 | 对照原始周报，评估系统判断与人工判断是否一致 | PMO协调人 |
| 偏差记录 | 若存在偏差，具体记录偏差事项和偏差原因 | PMO协调人 |
| ABC决策 | 基于复核结果选择A/B/C并填写理由 | PMO协调人 |
| 问题登记 | 发现的具体问题记录到《第二周期问题登记表》 | PMO协调人 |

### 4.2 ABC决策说明

| 决策 | 含义 | 使用场景 |
|------|------|---------|
| A（接受） | 系统输出可直接使用，无需修改 | 判断一致率≥90% |
| B（修改后接受） | 系统输出需要人工修改部分内容后使用 | 判断一致率60%-90% |
| C（不接受） | 系统输出存在重大偏差，不适合作为参考 | 判断一致率<60%或存在重大误判 |

## 五、周五收口安排

### 5.1 收口时间

每周五16:00前完成本周第二周期运行收口。

### 5.2 收口内容

| 收口项 | 内容 | 完成标准 |
|--------|------|---------|
| 运行记录 | 本周样本接入、系统运行、输出结果归档 | 文档完整 |
| 价值判断 | 评估系统本周输出对管理的参考价值 | 有明确结论 |
| 问题登记 | 本周发现的具体问题全部登记 | 问题分级完成 |
| 下一步决策 | 明确下周是否继续运行、是否需要调整 | 有明确决策 |
