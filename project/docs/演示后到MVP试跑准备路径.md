# 演示后到MVP试跑准备路径

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 明确从"演示完成"到"MVP试跑准备"的最短路径

---

## 一、路径总览

```
演示完成
    ↓
反馈归并（1天）
    ↓
门控更新（等外部确认）
    ↓
Q01/Q03/Q04全部确认
    ↓
MVP冻结面确认（1天）
    ↓
试跑样本与试点项目确认（1-3天）
    ↓
试跑SOP准备（1天）
    ↓
MVP试跑开始
```

---

## 二、分步说明

### 步骤1：反馈归并

**动作：** 演示后1天内完成
**输入：** 演示版验收记录表.md
**处理：**
1. 按演示反馈归并规则.md分类所有反馈
2. 按演示反馈收口机制.md指定处理文档
3. 按演示反馈转MVP任务规则.md判断是否进入P0/P1/P2

**输出：**
- 各类反馈已归入对应文档
- P0/P1任务进入MVP任务池
- 行动项清单已更新

**退出条件：** 反馈全部归类，无遗漏

---

### 步骤2：门控更新

**动作：** 基于外部确认更新门控
**输入：** 确认反馈
**处理：**
- Q01确认 → G1 🟢
- Q03确认 → G2 🟢
- Q04确认 → G3 🟢

**重要：** 门控更新必须基于正式签字确认，不基于演示口头反馈

**输出：** 阶段4B进入门控卡.md 更新

**退出条件：** G1+G2+G3 全部🟢

---

### 步骤3：Q01/Q03/Q04全部确认

**动作：** 等PMO/管理层正式确认
**最短路径：**
```
发出确认包（用户负责） →
等待签字（取决于外部） →
收到确认 →
Agent更新门控卡 →
发出"可进入MVP"通知
```

**门控状态目标：**
| G | 条件 | 目标状态 |
|---|------|---------|
| G1 | Q01 PMO术语口径 | 🟢已确认 |
| G2 | Q03 管理层格式 | 🟢已确认 |
| G3 | Q04 试点范围 | 🟢已确认 |
| G4 | S1样本到位 | 🟢已确认 |
| G5 | S2样本到位 | 🟢已确认 |
| G6 | 文档冻结 | 🟢已冻结 |
| G7 | 冒烟测试 | 🟢已就绪 |
**总体：🟢**

**退出条件：** G1+G2+G3+G4+G5 全部🟢

---

### 步骤4：MVP冻结面确认

**动作：** Q01/Q03/Q04确认后1天内
**输入：** MVP最小冻结面.md
**处理：**
1. 对照检查清单逐项确认
2. 所有provisional规则→正式版本
3. 更新engine.py注释（去除provisional标记）

**输出：** MVP最小冻结面.md 更新状态

**退出条件：** 所有P0检查项全部确认

---

### 步骤5：试跑样本与试点项目确认

**动作：** 1-3天
**输入：** 试点项目范围确认外发版.md
**处理：**
1. PMO确定具体试点项目（不少于1个）
2. 准备试点项目的周报/台账样本
3. 确认样本来源（谁提供、何时提供）

**输出：** 试点项目确认单（含项目名称、数据负责人、样本到位时间）

**退出条件：** 试点项目确定 + 样本准备计划确定

---

### 步骤6：试跑SOP准备

**动作：** 1天
**输入：** 试点SOP.md（如已存在）
**处理：**
1. 确认每期周报→输出的时间要求（建议24小时内）
2. 确认人工审阅节点的位置和动作
3. 确认反馈收集机制

**输出：** 试跑SOP（含时间要求、审阅节点、反馈机制）

**退出条件：** SOP已文档化并获得PMO认可

---

## 三、每步进入/退出条件

| 步骤 | 进入条件 | 退出条件 | 负责方 |
|------|---------|---------|--------|
| 1.反馈归并 | 演示完成 | 反馈全部归类 | Agent |
| 2.门控更新 | 有外部确认 | 门控状态更新 | Agent |
| 3.Q01/Q03/Q04确认 | 确认包已发出 | G1+G2+G3全部🟢 | PMO/管理层 |
| 4.MVP冻结面确认 | 步骤3完成 | 冻结面清单全部确认 | Agent |
| 5.试点项目确认 | 步骤3完成 | 试点项目确定 | PMO |
| 6.试跑SOP准备 | 步骤5完成 | SOP文档化 | Agent+PMO |

---

## 四、最短路径时间估算

| 步骤 | 预计时间 | 前提 |
|------|---------|------|
| 1.反馈归并 | 1天 | 演示完成 |
| 2.门控更新 | 即时 | 有确认 |
| 3.Q01/Q03/Q04确认 | 3-7天 | 确认包已发出 |
| 4.MVP冻结面确认 | 1天 | 步骤3完成 |
| 5.试点项目确认 | 1-3天 | 步骤3完成 |
| 6.试跑SOP准备 | 1天 | 步骤5完成 |

**最短路径：7-14天（假设Q01/Q03/Q04在1周内确认）**

---

## 五、路径中的风险点

| 风险点 | 影响 | 缓解措施 |
|--------|------|---------|
| Q01/Q03/Q04迟迟不确认 | 路径无限延长 | 按阻断项催办话术推进 |
| 演示反馈要求修改核心规则 | 可能阻断MVP | 按P0处理，等确认 |
| 试点项目无法确定 | 无法进入试跑 | 先用现有样本跑一期 |
| PMO无法提供试点样本 | 试跑无法开始 | 用现有S2样本先跑 |
