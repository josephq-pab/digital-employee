# 试跑问题转任务规则

**版本：** v1.0
**日期：** 2026-04-08

---

## 一、转换原则

| 原则 | 说明 |
|------|------|
| 不得降级 | 不得将P0降为P1/P2，不得将P1降为P2 |
| 不得删除 | 不得删除问题记录，只能标记"已处理" |
| 不得跳过 | 不得跳过外部确认流程自行假定确认结果 |
| 不得遗漏 | 每条问题必须明确处理路径 |

---

## 二、问题→任务转换规则

### 2.1 阻断型（P0）

| 判断条件 | 转换结果 |
|---------|---------|
| 任何P0问题 | 立即生成P0修复任务，优先级最高，阻断所有其他工作 |

**转换产物：**
- 任务名称：P0-修复-[问题编号]
- 优先级：P0
- 进入：《试跑后修订机制》- 立即修
- 门控联动：修复前不得进入正式试跑准备

---

### 2.2 优化型（P1）

**判断条件：** P1问题，且不涉及provisional规则/格式变更

| 问题类型 | 转换结果 |
|---------|---------|
| 映射问题 | 进入映射优化任务 |
| 展示问题 | 进入展示优化任务 |
| 数据问题 | 进入数据质量任务 |

**转换产物：**
- 任务名称：P1-优化-[问题编号]
- 优先级：P1
- 进入：《试跑后修订机制》- 立即修（如不依赖外部确认）或等确认后修（如依赖）
- 门控联动：修复完成后更新门控卡

---

### 2.3 口径确认型（P1）

**判断条件：** P1问题，且依赖Q01/Q03/Q04外部确认

| 依赖项 | 转换结果 |
|--------|---------|
| Q01（PMO术语口径） | 关联Q01确认任务，确认完成后解锁P1处理 |
| Q03（管理层格式） | 关联Q03确认任务，确认完成后解锁P1处理 |
| Q04（试点范围） | 关联Q04确认任务，确认完成后解锁P1处理 |

**转换产物：**
- 任务名称：P1-待确认-[Q编号]-[问题编号]
- 优先级：P1（但blocked by 外部确认）
- 进入：《试跑后修订机制》- 等确认后修
- 门控联动：外部确认完成前任务blocked

---

### 2.4 展示型（P1）

**判断条件：** P1问题，仅影响展示格式，不影响判断逻辑

| 处理方式 |
|---------|
| 直接进入展示优化任务，MVP期间处理 |

**转换产物：**
- 任务名称：P1-展示优化-[问题编号]
- 优先级：P1
- 进入：《试跑后修订机制》- 立即修

---

### 2.5 观察型（P2）

**判断条件：** P2问题

| 处理方式 |
|---------|
| 进入观察列表，MVP试跑期间跟踪，不作为必须修复任务 |

**转换产物：**
- 任务名称：P2-观察-[问题编号]
- 优先级：P2
- 进入：《试跑后修订机制》- MVP试跑中观察
- 门控联动：无（只观察不强制修复）

---

## 三、首轮试跑问题转换结果

| 问题编号 | 类型 | 转换任务名称 | 进入路径 |
|---------|------|------------|---------|
| P1 | 口径确认型 | P1-待确认-Q01-P1 | 等确认后修 |
| P2 | 口径确认型 | P1-待确认-Q04-P2 | 等确认后修 |
| P3 | 展示型 | P1-展示优化-P3 | 立即修（MVP期间） |
| P4 | 口径确认型 | P1-待确认-Q01-P4 | 等确认后修 |
| P5 | 口径确认型 | P1-待确认-Q01-P5 | 等确认后修 |
| P6 | 观察型 | P2-观察-P6 | MVP试跑中观察 |
| P7 | 口径确认型 | P1-待确认-Q01-P7 | 等确认后修 |

---

## 四、任务状态流转

```
发现问题
    ↓
判断P0/P1/P2
    ↓
P0 → P0-修复任务（立即修）→ 修复完成 → 更新门控
P1 → 判断类型
    ├─ 优化型 → P1-优化任务（立即修）→ 修复完成 → 更新门控
    ├─ 口径确认型 → P1-待确认任务（blocked by Q01/Q03/Q04）→ 确认完成 → 解锁处理
    └─ 展示型 → P1-展示优化任务（立即修）→ 修复完成 → 更新门控
P2 → P2-观察任务（观察）→ MVP期间评估是否修复
```

---

## 五、禁止事项

| 禁止 | 说明 |
|------|------|
| 不得将P0降级 | P0问题必须立即处理，不得降低严重度 |
| 不得将P1自行确认 | Q01/Q03/Q04必须由外部正式确认，不得由Agent自行判断 |
| 不得删除问题记录 | 问题登记后不得删除，只能标记"已处理"或"已观察" |
| 不得跳过等确认流程 | 依赖外部确认的问题，在确认完成前不得擅自处理 |
