## 三、推荐试用路径（5分钟完整路径）

> 业务方打开页面后，按此顺序走一遍，即可在5分钟内完整理解系统能力

### 第1步：看首页结论卡（1分钟）
```
动作：打开页面，默认显示首页结论卡（6BLIVE真实样本）
观察：
  - 周期标题（2026-04-14/04-18）
  - 整体判断文字（3项需重点跟踪）
  - 四个数字：总事项数/需重点跟踪/M-001缺口/已关闭
  - 风险等级（颜色+文字）
  - 下一步动作建议
重点反馈：数字和结论与原始周报是否一致？
```

### 第2步：体验推演样本切换（1分钟）
```
动作：点击「推演W16」按钮
观察：
  - 右上角出现"推演样本"标签（淡白色药丸形）
  - 结论卡内容变化（风险从中风险变为关注描述）
  - M-001五期趋势图出现在结论卡下方
  - 趋势条：W16红色（缺口8.2亿）→ W17橙色 → W18黄色 → W19/W20绿色
  - 依次点击W17/W18/W19/W20，观察颜色和gap文字变化
重点反馈：
  - 趋势图是否能一眼看懂趋势？
  - "推演样本"标签是否会让你产生困惑？
```

### 第3步：看领导摘要（1分钟）
```
动作：点击「📋 查看领导摘要」
观察：
  - 黄灯/关注类事项列表
  - 延期申请和结项申请是否正确分流
  - 判断依据（是否写了规则名称）
重点反馈：
  - 摘要文字是否像"人写的"？
  - 判断依据是否有助于你快速核实？
```

### 第4步：看风险清单（1分钟）
```
动作：点击「⚠️ 风险清单」
观察：
  - 黄灯/关注类事项列表（不应出现结项申请）
  - 每项的信号来源和判断规则
  - 负责人/部门信息
重点反馈：
  - 清单中的事项是否都是你实际认为需要跟踪的？
  - 有没有遗漏？有没有误入？
```

### 第5步：看复核与决策（1分钟）
```
动作：点击「✅ 复核与决策」
观察：
  - 一致/偏差/无法判断三栏
  - 每项后面的判断选项
  - 填写入口是否清晰
重点反馈：
  - 填写动作是否足够轻量？
  - 偏差选项是否有足够的语义区分度？
```

## 四、反馈收集规则（固化版）

### 反馈类型定义

| 类型 | 定义 | 举例 | 处理方向 |
|------|------|------|---------|
| **页面体验问题** | 操作/交互/可读性 | "切换样本后页面布局乱了" | 优化清单P2 |
| **表达问题** | 文字语义/措辞/完整性 | "风险描述里没写目标值，看不懂" | 优化清单P1/P2 |
| **流程问题** | 样本接入/复核/收口 | "上传文件步骤不清晰" | 流程优化 |
| **非阻断建议** | 建设性意见，无明确问题 | "建议增加图表" | 观察项 |
| **阻断问题** | 导致无法正常使用 | 页面打不开 | 立即修复P0 |

### 反馈记录格式（实际使用）

每条反馈必须记录以下字段，不可留空：

```
编号：[自动编号，如 FB-001]
时间：[填写时间]
反馈人：[姓名/角色]
分类：[页面体验 / 表达问题 / 流程问题 / 非阻断建议 / 阻断问题]
反馈内容：[一句话描述]
具体描述：[完整描述，包含操作路径]
是否可复现：[是 / 否]
D类触发：[是 / 否，触发哪项]
```

### 禁止记录的内容

- ❌ "看起来不错"（抽象评价）
- ❌ "感觉不太好用"（无具体描述）
- ❌ "能不能加个XXX功能"（无问题描述，只有功能请求）

## 五、D-01/D-02/D-03 暂不触发条件

以下条件**同时满足**时，D-01/D-02/D-03 **不主动触发**：

1. 试用反馈中没有出现具体的管理决策分歧案例
2. 反馈问题可归入"页面体验"或"表达问题"
3. 没有出现"样本判断与人工判断系统性不一致"的情况

**触发条件**：如果试用中出现以下情况之一，立即触发 D-01/D-02/D-03：

| 触发信号 | 对应决策 |
|---------|---------|
| 多个反馈提到"不知道系统结论是怎么来的" | D-01（判断依据透明度） |
| 多个反馈提到"结论与人工判断不符，不知道听谁的" | D-02（判断权威性认定） |
| 反馈中出现"系统建议A，但人工决定B，该怎么处理" | D-03（系统建议与人工决策关系） |

## 六、迭代输入管理

所有试用反馈在收集后，由 Agent 整理为：

**文件1：反馈汇总表**（追加到 `MVP试用反馈记录.md` 的反馈记录表）

**文件2：下一轮优化清单**（新增 `project/docs/迭代优化清单_YYYYMMDD.md`）
- 格式：问题描述 / 分类 / 优先级 / 是否触发D类 / 状态
- 优先级定义：
  - P0：阻断问题，立即修复
  - P1：影响试用主路径，04/18前修复
  - P2：非阻断，04/18后处理
