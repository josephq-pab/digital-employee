# 页面与输出映射说明

**版本**：v1.0  
**日期**：2026-04-10  
**阶段**：阶段6A《页面化MVP与并行运行系统建设》  
**用途**：明确每个页面对应哪些已有输出，页面不是新造口径，是承接既有输出结构  

---

## 一、核心原则

> 页面是已有输出的"容器"，不是新造口径。
> 每个页面展示的内容，都对应已有的、结构化的输出文件。
> 后续真实样本到位后，只需要替换数据，不需要改页面结构。

---

## 三、模板A格式说明

> 所有页面中的领导摘要，严格遵循模板A格式，不新造样式。

**模板A格式标准**：

| 节 | 内容要求 |
|----|---------|
| 第一节 | 周期总览统计（5项数字）+ 整体判断（2-3句） |
| 第二节 | 重点事项（分类列出，延期/黄灯/新增/结项） |
| 第三节 | 下周期重点跟踪（3-5项） |
| 第四节 | 周期数字核对（原文vs系统输出，一致性标注） |

## 四、后续真实样本到位后的替换逻辑

| 替换项 | 替换方式 |
|--------|---------|
| 领导摘要内容 | 替换领导摘要.md文件内容，页面自动读取最新版本 |
| 事项总台账内容 | 替换台账.md文件内容，页面自动读取 |
| 风险清单内容 | 替换风险清单.md文件内容，页面自动读取 |
| 样本元数据 | 新增样本接入记录文件，页面读取新样本列表 |
| 历史运行记录 | 更新首周期并行运行记录.md，页面自动追加 |

**原则**：页面结构不变，只替换数据文件。
