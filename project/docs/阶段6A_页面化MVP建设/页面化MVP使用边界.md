## 一、现在能做什么（Can Do）

| 功能 | 说明 | 入口 |
|------|------|------|

## 三、为什么现在先做并行运行系统，而不是生产系统

### 生产系统需要什么

| 维度 | 生产系统要求 | 当前状态 |
|------|------------|---------|
| 数据来源 | 自动从OA/邮件抓取 | ❌ 无接口 |
| 数据质量 | 经过校验的标准化数据 | ⚠️ 待验证 |
| 权限体系 | 多角色多部门权限 | ❌ 未建立 |
| 通知机制 | 自动通知/提醒 | ❌ 未建设 |
| 稳定性 | 7*24可用 | ❌ MVP阶段 |
| 响应速度 | 秒级响应 | ❌ MVP阶段 |
| 合规审计 | 操作日志完整可查 | ⚠️ 基础留痕 |

### 为什么先做并行运行MVP

| 维度 | MVP可以做到 | 生产系统做不到 |
|------|------------|--------------|
| 快速验证价值 | ✅ 1-2周可以内部试运行 | ❌ 生产系统建设周期长 |
| 收集真实反馈 | ✅ 业务方可以直接试用 | ❌ 需求讨论无法验证 |
| 验证系统能力 | ✅ 历史样本已验证100%一致率 | ❌ 需要真实数据验证 |
| 发现偏差和不足 | ✅ 早期低成本发现 | ❌ 生产环境发现代价大 |
| 逐步扩大范围 | ✅ 从单部门开始逐步扩展 | ❌ 大规模推广风险高 |

### 并行运行MVP的定位

> 并行运行MVP = 内部参照工具，不是生产系统
> 目的：让业务方在同一份数据上看到两套输出（现有机制+数字员工），通过对照发现价值
> 约束：不能替代现有PMO机制，不能直接做管理决策

## 四、MVP与生产系统的区别

| 维度 | MVP（本期） | 生产系统（未来） |
|------|-----------|---------------|
| 数据接入 | 人工选择样本包 | 自动从OA/邮件抓取 |
| 数据校验 | 人工确认 | 自动校验+异常告警 |
| 用户范围 | 单团队内部使用 | 多团队多部门 |
| 权限管理 | 无（内部工具） | 多角色分级权限 |
| 可用性 | 工作时间可用 | 7*24可用 |
| 通知方式 | 人工传递 | 自动通知 |
| 输出形式 | HTML页面查看 | 多格式导出/打印 |
| 错误处理 | 人工发现+记录 | 自动告警+补救 |
| 审计追溯 | 基础留痕 | 完整操作日志 |

## 五、什么时候可以从MVP升级到生产系统

> 当前不承诺生产系统上线时间。当前MVP稳定运行一段时间后（建议3期以上），根据业务方反馈再评估。

**升级前提条件**（按优先级）：

| 优先级 | 条件 | 当前状态 |
|--------|------|---------|
| P0 | 真实样本接入验证通过 | ⬜ 待04/14 |
| P1 | MVP内部试运行3期以上 | ⬜ 待04/14 |
| P2 | 业务方明确提出生产系统需求 | ⬜ 待验证 |
| P3 | 数据自动接入方案确认 | ⬜ 待评估 |
| P4 | 权限体系需求明确 | ⬜ 待评估 |
| P5 | 通知机制需求明确 | ⬜ 待评估 |

## 六、使用场景声明

| 场景 | 是否适合用MVP | 说明 |
|------|-------------|------|
| PMO内部周报快速查看 | ✅ 适合 | 直接打开系统，选择样本即可 |
| 向领导汇报参考材料 | ⚠️ 可用但需注明 | 标注"系统参照输出，人工确认后使用" |
| 管理决策依据 | ❌ 不适合 | MVP不能替代人工决策 |
| 跨部门协调沟通 | ❌ 不适合 | MVP没有通知和协作功能 |
| 给外部客户展示 | ❌ 不适合 | MVP是内部工具，不是产品 |
