# 页面化MVP最小操作流

**版本**：v1.0  
**日期**：2026-04-10  
**阶段**：阶段6A《页面化MVP与并行运行系统建设》  
**用途**：用业务语言写清楚从输入到输出到复核的最小闭环  

---

## 一、业务语言描述最小操作流

> 数字员工并行运行工具的使用，遵循以下8步闭环：

```
第1步：选择样本
第2步：触发运行
第3步：查看领导摘要
第4步：查看事项台账
第5步：查看风险清单
第6步：记录复核意见
第7步：登记ABC决策
第8步：确认留痕
```

---

## 二、每步详细说明

### 第1步：选择样本

| 字段 | 内容 |
|------|------|
| 做什么 | 在数据接入页选择要查看的样本周期 |
| 谁做 | 业务方（PMO协调人/业务负责人） |
| 什么时候做 | 每次查看系统前先确认当前选中的样本 |
| 入口 | 首页 → 数据接入页，或直接使用样本切换器 |
| 结果 | 确认当前样本周期，系统加载该样本数据 |

**业务语言**："我先要告诉系统，我要看哪一周的数据。"

---

### 第2步：触发运行

| 字段 | 内容 |
|------|------|
| 做什么 | 选择样本后，点击"运行"或"刷新" |
| 谁做 | 业务方 |
| 什么时候做 | 样本切换后，或数据更新后 |
| 入口 | 数据接入页 |
| 结果 | 系统重新解析样本数据，生成最新输出 |

**业务语言**："选了数据之后，我点一下运行，系统就自动整理出来了。"

---

### 第3步：查看领导摘要

| 字段 | 内容 |
|------|------|
| 做什么 | 查看系统生成的领导摘要 |
| 谁看 | PMO协调人/业务负责人/管理层 |
| 什么时候做 | 运行完成后 |
| 入口 | 首页"查看本周摘要"按钮，或侧边栏"领导摘要" |
| 看什么 | 整体判断/重点事项/统计数字/下周期建议 |
| 结果 | 对本期整体状态有快速了解 |

**业务语言**："点开摘要，我30秒就知道这周整体情况怎么样，哪些事要重点跟。"

---

### 第4步：查看事项总台账

| 字段 | 内容 |
|------|------|
| 做什么 | 查看全量事项列表，筛选重点关注事项 |
| 谁看 | PMO协调人/业务负责人 |
| 什么时候做 | 看摘要之后，想了解具体事项时 |
| 入口 | 领导摘要页的"查看事项总台账"链接，或侧边栏 |
| 看什么 | 所有事项名称/状态/风险/责任人/节点 |
| 筛选建议 | 优先看"重点关注"（黄灯/延期/结项/新增） |
| 结果 | 对所有事项有完整视图，知道哪些事项需要跟进 |

**业务语言**："我想看所有事项的列表，特别是那些黄灯和延期的事，看看都是谁在负责。"

---

### 第5步：查看风险清单

| 字段 | 内容 |
|------|------|
| 做什么 | 查看单独的风险清单页面 |
| 谁看 | PMO协调人/业务负责人/管理层 |
| 什么时候做 | 想快速知道"哪些事进了风险清单"时 |
| 入口 | 首页"查看风险清单"按钮，或侧边栏 |
| 看什么 | 黄灯/延期/指标偏离/新增项目，每项附判断依据 |
| 结果 | 知道为什么这些事项进了风险清单，以及建议动作 |

**业务语言**："我想知道哪些事需要重点关注，为什么进了一个专门的清单。"

---

### 第6步：记录复核意见

| 字段 | 内容 |
|------|------|
| 做什么 | 在人工复核页记录"一致"还是"有偏差" |
| 谁做 | PMO试点协调人（业务方） |
| 什么时候做 | 查看完摘要、台账、风险清单后 |
| 入口 | 侧边栏"人工复核" |
| 填什么 | 一致/偏差 + 具体偏差事项 + 偏差说明 |
| 结果 | 系统留痕，人工判断与系统判断的对比记录 |

**业务语言**："我看完了系统给的这些东西，跟我了解的情况对一下，有不一样的地方我记录下来。"

---

### 第7步：登记ABC决策

| 字段 | 内容 |
|------|------|
| 做什么 | 在ABC决策区选择A/B/C并填写理由 |
| 谁做 | PMO试点协调人（业务方） |
| 什么时候做 | 复核意见记录完成后 |
| 入口 | 人工复核页的ABC决策区 |
| 填什么 | A（接受）/ B（修改后接受）/ C（不接受）+ 理由 |
| 结果 | 系统留痕本次决策结果 |

**业务语言**："基于系统给的这些东西，我决定是接受、修改后接受还是不接受，记在系统里。"

---

### 第8步：确认留痕

| 字段 | 内容 |
|------|------|
| 做什么 | 在历史运行记录页确认本次运行已留痕 |
| 谁做 | 系统（自动）/ 业务方（确认） |
| 什么时候做 | 完成第7步后 |
| 入口 | 历史运行记录页 |
| 看什么 | 本期周期的"输出状态"和"决策状态"是否为"已完成" |
| 结果 | 本期运行完整闭环，历史可追溯 |

**业务语言**："系统自动把这个周期的运行结果和我的决策都记下来了，下次来看的时候能查到。"

## 四、谁在什么时候做什么

| 角色 | 动作 | 时间点 |
|------|------|-------|
| PMO协调人 | 选择样本，触发运行 | 每周收到周报后 |
| PMO协调人/管理层 | 查看领导摘要和风险清单 | 运行完成后 |
| PMO协调人 | 记录复核意见 | 查看输出后 |
| PMO协调人 | 登记ABC决策 | 复核完成后 |
| 系统 | 自动留痕到历史记录 | 每步完成后 |

---

## 五、这个流解决什么问题

| 问题 | 解决方式 |
|------|---------|
| "不知道怎么开始用这个系统" | 明确的8步操作流，按顺序走完即可 |
| "不知道填在哪里" | 每步有明确的入口和位置 |
| "不知道为什么要这样做" | 每步有业务语言说明 |
| "不知道填完去哪里确认" | 第8步专门留痕确认 |
| "历史记录在哪里看" | 历史运行记录页统一追溯 |
