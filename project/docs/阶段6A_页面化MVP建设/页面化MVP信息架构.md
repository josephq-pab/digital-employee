# 页面化MVP信息架构

**版本**：v1.0  
**日期**：2026-04-10  
**阶段**：阶段6A《页面化MVP与并行运行系统建设》  
**用途**：明确系统一级页面、每个页面解决什么问题、用户主要操作路径  

---

## 一、系统定位一句话说明

> 数字员工并行运行内部工具：为PMO和业务负责人提供"周报数据整理→结构化输出→人工复核→ABC决策记录"的完整闭环视图，辅助而不替代人工决策。

---

## 二、一级页面架构（7页）

```
┌─────────────────────────────────────────────────────┐
│                    首页/总览                          │
│  （系统入口，一眼说明"这是什么+现在什么状态"）            │
└─────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────┐
│                   数据接入页                          │
│  （选择用哪个样本/周期，确认数据完整性）                  │
└─────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────┐
│              领导摘要页 / 事项台账页 / 风险清单页        │
│  （查看三类核心输出）                                  │
└─────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────┐
│              人工复核与ABC决策页                       │
│  （记录一致/偏差，登记ABC决策）                        │
└─────────────────────────────────────────────────────┘
          │
          ▼
┌─────────────────────────────────────────────────────┐
│               历史运行记录页                           │
│  （查看已跑过的所有周期）                              │
└─────────────────────────────────────────────────────┘
```

## 三、每个页面详细说明

### 页面1：首页/总览页

| 字段 | 内容 |
|------|------|
| 页面名称 | 首页/总览 |
| 页面文件 | `pages/index.html` |
| 用户进入系统后第一眼看到的 | 是 |
| 解决什么问题 | "这是什么系统？现在什么状态？最近跑了什么？" |
| 一句话说明 | 系统入口页，展示项目定位、当前状态、最近周期关键数字 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 系统定位说明 | "数字员工并行运行内部工具，不是替代人工，是参照系统" |
| 当前阶段状态 | 展示现在处于哪个阶段（6A页面化MVP建设） |
| 最近运行周期 | 最近一次跑的周期ID + 日期 |
| 关键数字 | 总事项数/黄灯数/延期数/结项数（如有） |
| 本周最新运行结果入口 | 按钮跳转领导摘要页 |
| 样本切换器 | 可以切换查看两个历史样本 |

**用户操作路径**：打开系统 → 首页 → 点击"查看本周摘要" → 进入领导摘要页

---

### 页面2：数据接入页

| 字段 | 内容 |
|------|------|
| 页面名称 | 数据接入页 |
| 页面文件 | `pages/data.html` |
| 解决什么问题 | "现在用的哪个样本？数据齐不齐全？" |
| 一句话说明 | 展示可用样本列表，选择当前要查看的样本，确认数据完整性 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 可用样本列表 | 两个历史样本（1117-1121 / 1208-1212）+ 04/14真实样本（待接入） |
| 样本完整性状态 | 周报/台账/指标 是否完整 |
| 样本基本信息 | 周期/样本ID/场景类型 |
| 当前选中样本高亮 | 让用户知道现在在看哪个 |

**用户操作路径**：打开系统 → 选择"数据接入" → 选择样本周期 → 确认数据完整性 → 查看输出

---

### 页面3：领导摘要页

| 字段 | 内容 |
|------|------|
| 页面名称 | 领导摘要页 |
| 页面文件 | `pages/summary.html` |
| 解决什么问题 | "本周整体什么情况？哪些需要重点关注？" |
| 一句话说明 | 按模板A风格输出管理层材料，一眼看懂总体状态和风险 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 周期总览统计 | 总事项数/推进中/已关闭/延期/黄灯/结项（如有） |
| 整体判断 | 系统对本期整体状态的描述 |
| 本周期重点事项 | 延期/黄灯/新增/结项列表 |
| 下周期重点跟踪 | 系统给出的建议跟踪事项 |
| 周期数字核对 | 与原文统计数字是否一致 |
| 样本来源标注 | "本输出基于XX样本，非当前生产数据" |

**用户操作路径**：首页点击"查看领导摘要" → 进入领导摘要页 → 阅读理解 → 记录复核意见

---

### 页面4：事项总台账页

| 字段 | 内容 |
|------|------|
| 页面名称 | 事项总台账页 |
| 页面文件 | `pages/tracker.html` |
| 解决什么问题 | "所有事项都在这里，谁负责，什么状态？" |
| 一句话说明 | 全量事项列表，可筛选/可排序，不是平铺所有事项 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 事项总数统计 | 总数/推进中/已关闭 |
| 重点关注筛选器 | 只看"重点关注"（黄灯/延期/新增/结项） |
| 全量事项列表 | 事项名称/状态/风险等级/主责部门/计划节点/系统判断 |
| 状态标签 | 正常/黄灯/延期/结项/已关闭 颜色区分 |
| 判断依据列 | 每项事项的系统判断理由 |

**用户操作路径**：首页或领导摘要页 → 点击"查看事项总台账" → 进入台账页 → 筛选重点关注事项

---

### 页面5：风险清单页

| 字段 | 内容 |
|------|------|
| 页面名称 | 风险清单页 |
| 页面文件 | `pages/risks.html` |
| 解决什么问题 | "哪些事项进了风险清单？为什么？" |
| 一句话说明 | 单独展示黄灯/延期/指标偏离/重点关注，每项附判断依据 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 风险统计 | 黄灯数/延期数/指标偏离数/重点关注数 |
| 分层展示 | 黄灯（持续）/ 黄灯（提示）/ 延期 / 指标偏离 / 新增 |
| 每项风险详情 | 事项名称 + 风险类型 + 当前状态 + 判断依据 + 建议动作 |
| 不在风险清单的说明 | 明确"结项申请不进风险清单"原则 |

**用户操作路径**：首页 → 点击"查看风险清单" → 进入风险清单页 → 了解每项风险原因

---

### 页面6：人工复核与ABC决策页

| 字段 | 内容 |
|------|------|
| 页面名称 | 人工复核与ABC决策页 |
| 页面文件 | `pages/review.html` |
| 解决什么问题 | "在哪里记录人工复核意见？在哪里做ABC决策？" |
| 一句话说明 | 业务方复核系统输出的位置，登记一致/偏差和ABC决策结果 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|
| 本期运行概要 | 周期ID + 运行日期 + 样本来源 |
| 一致率汇总 | 人工与系统判断一致率 |
| 偏差记录区 | 记录系统判断与人工判断不符的具体事项 |
| ABC决策区 | A（接受）/ B（修改后接受）/ C（不接受）选项及理由 |
| 历史决策记录 | 本周期决策结果（收口时填写） |
| 复核状态 | 待复核 / 已复核 |

**用户操作路径**：查看完摘要/台账/风险清单后 → 进入复核页 → 填写复核意见 → 登记ABC决策

---

| 字段 | 内容 |
|------|------|
| 页面名称 | 历史运行记录页 |
| 页面文件 | `pages/history.html` |
| 解决什么问题 | "之前跑过哪些周期？每期输出了什么？有没有做决策？" |
| 一句话说明 | 已跑周期的历史记录，支持追溯和复盘 |

**页面内容**：

| 内容模块 | 说明 |
|---------|------|

| 统计概览 | 总周期数 / 已收口数 / 触发ABC决策数 |

**用户操作路径**：首页 → 点击"历史运行记录" → 查看所有已跑周期 → 点击具体周期查看详情

## 四、用户主要操作路径

### 路径1：首次查看（5分钟快速了解）

```
打开系统（首页）
→ 看系统定位说明
→ 看最近运行状态
→ 点击"查看领导摘要"
→ 阅读摘要
→ 点击"查看风险清单"
→ 了解重点风险
→ 返回首页
```

### 路径2：完整复核流程（30分钟）

```
打开系统（首页）
→ 进入数据接入页，选择样本
→ 查看领导摘要
→ 查看事项总台账（筛选重点关注）
→ 查看风险清单
→ 进入人工复核页，填写复核意见
→ 登记ABC决策
→ 查看历史运行记录确认本次已留痕
```

### 路径3：样本对比查看（10分钟）

```
打开系统（首页）
→ 选择样本1（1117-1121）
→ 查看领导摘要
→ 切换样本2（1208-1212）
→ 查看领导摘要
→ 对比两期待征
```
