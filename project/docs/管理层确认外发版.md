# 管理层确认外发版

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 发给管理层审阅人，一页内可读完
**页数：** 建议1页

---

## 一、我们正在做什么

数字员工一期正在建设"重点项目执行跟踪"原型，目标是将PMO重点项目（如智慧园区、对公改革等）的执行状态，从人工汇总改为AI辅助判断输出，让管理层每周看到标准化、结构化的状态摘要。

**当前阶段：** 即将完成原型开发，准备用真实试点项目验证效果。

---

## 二、我们需要您确认的事

**只有一件事：领导摘要的输出格式**

我们为管理层设计了两套领导摘要模板，请选择您更倾向哪一套：

### 模板A（完整版）- 推荐

```
项目：智慧园区一期
责任人：张明
状态评级：🔴 危险

风险等级：高
升级建议：是（请求领导关注）

【事实】
- 承诺6月30日交付数据看板
- 当前进展：数据看板滞后2周，接口联调未完成

【判断】
- 风险来源：rule_严重滞后
- 依据：关键词"严重滞后"出现在进展描述中

【建议】
- 建议与张明确认实际可交付时间
- 建议评估接口依赖方的影响范围
```

### 模板B（简版）

```
项目：智慧园区一期 | 责任人：张明
状态：🔴 危险 | 风险：高 | 升级：是
摘要：承诺6月30日交付，数据看板滞后2周，已触发严重滞后规则。
建议：关注。
```

---

## 三、两套模板的核心区别

| 维度 | 模板A（完整版） | 模板B（简版） |
|------|---------------|-------------|
| 信息量 | 事实/判断/建议三层分离 | 一行概览 |
| 适合场景 | 深度复盘、风险分析 | 快速浏览 |
| 长度 | 较长 | 1-2行 |
| 结构性 | ✅ 强 | ⚠️ 弱 |

---

## 四、如果您选择模板A

系统将输出包含"事实 → 判断 → 建议"三层结构的领导摘要，每层清晰分离，便于人工复核判断是否准确。

**我们推荐模板A**，因为它符合一期"可复核"的核心原则。
