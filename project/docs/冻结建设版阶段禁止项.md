# 冻结建设版阶段禁止项

> 版本：v1.0（2026-04-16）
> 用途：冻结版达成前，必须遵守的禁止规则

---

## 一、禁止启动最终移交

**禁止理由：** 最终移交后主线级开发无法继续，在冻结版达成前移交会把开发工作带到新环境里继续打磨，增加风险。

| 禁止项 | 替代动作 |
|--------|---------|
| 启动最终迁移 | 等冻结版达成后再启动 |
| 把当前当作"最终版"移交 | 当前是"冻结建设版起点"，不是最终版 |
| 宣称"项目已完成" | 项目仍在进行，目标"准投产冻结版" |

---

## 二、禁止启动正式迁移

**禁止理由：** 迁移后代码和环境都固定，继续完善只能在新环境里做，而新环境不如当前环境可控。

| 禁止项 | 替代动作 |
|--------|---------|
| 启动正式迁移准备 | 继续在当前环境推进冻结版 |
| 把生产环境部署当作首要目标 | 优先完成冻结版建设 |
| 替换生产级模型/组件 | 等冻结后迁移时做 |

---

## 三、禁止误宣称"已完成最终交付"

**禁止理由：** 当前是开发基线移交，不是最终版交付。混淆口径会导致新接手方期望偏差。

| 禁止表述 | 正确表述 |
|---------|---------|
| "已完成正式交付" | "已完成开发基线移交（c1e85bc）" |
| "当前是最终版" | "当前是冻结建设版起点" |
| "项目已结束" | "项目仍在冻结建设版主线" |
| "当前版本可以直接投产" | "当前版本需要先建设到冻结版" |

---

## 四、禁止无边界功能发散

**禁止理由：** 发散会拖慢冻结版达成时间，增加迁移后改动量。

| 禁止方向 | 原因 |
|---------|------|
| 新增大功能模块 | 属于生产级开发，不当前做 |
| 全新页面/交互设计 | 属于体验工作，当前优先级低于冻结主流程 |
| 替换底层算法/模型 | 属于迁移后动作 |
| 大规模代码重构 | 除非有明确阻断性问题，否则不做 |

---

## 五、禁止把问题带到新环境继续打磨

**禁止理由：** 冻结版之前的问题在当前环境解决更可控；带到新环境会增加不确定性。

| 禁止项 | 替代动作 |
|--------|---------|
| "这个问题迁移后再说" | 在当前环境解决后再迁移 |
| "当前不解决，迁移时一起修" | 当前优先解决，不带问题迁移 |
| "先迁移再优化" | 先冻结，再迁移 |

---

## 六、口径统一要求

**禁止在任意文档/对话中出现以下混淆口径：**

| 禁止表述 | 要求 |
|---------|------|
| "最终交付版" | 统一改为"冻结建设版起点（c1e85bc）" |
| "已达到交付标准" | 统一改为"已达到开发基线可演示/可交接标准" |
| "可以直接移交给业务方" | 统一改为"可以在冻结版达成后移交给新开发环境" |
| "项目已完成" | 统一改为"冻结版主线进行中" |

---

## 七、违反禁止项的处理

**如果发现违反禁止项：**

```
Step 1：立即停止该动作
Step 2：记录到 memory/当日日期.md
Step 3：评估影响范围
Step 4：回退到禁止项前的状态（如可能）
Step 5：汇报给项目负责人确认
```

---

*本文档为冻结建设版阶段的硬性禁止规则，违反即触发复盘。*
