# 演示后分支说明

**版本**：v1.0  
**日期**：2026-04-09  
**阶段**：阶段4S-A《内部演示执行与决策收口》  
**用途**：演示完成后，根据决策记录选择分支路径，明确各分支的进入条件和最小动作集

---

## 一、分支决策规则

```
演示完成
    ↓
填写《内部演示决策记录模板》（3项决策）
    ↓
判断分支
    ↓
├── 决策A=同意 且 决策B=同意/有条件同意
│   → 分支A：演示认可 → 真实样本影子运行
│
├── 决策A=有条件同意（展示层/小口径调整）
│   → 分支B：演示认可，但先修小问题 → 再影子运行
│
└── 决策C=同意（评审优先）
    → 分支C：先推进扩大评审
```

---

## 二、分支A：演示认可 → 真实样本影子运行

**进入条件（同时满足）**：
1. 决策项A（输出结构）= 同意 或 有条件同意（有条件同意时小问题已在处理中）
2. 决策项B（影子运行授权）= 同意 或 有条件同意（条件已满足）
3. 无P1级阻断反馈（管理视角无"这个判断我无法接受"类反馈）

**最小动作集**：

| # | 动作 | 负责方 | 产出 |
|---|------|--------|------|
| A1 | 记录反馈（30分钟） | 项目总控Agent | 填写《内部演示反馈登记表》 |
| A2 | 确认小问题修复状态（如决策A=有条件同意）| 项目总控Agent | 确认修改完成时间 |
| A3 | 通知相关方分支走向 | 项目总控Agent | 发出分支A确认通知 |
| A4 | 推进FB-001/S-001真实样本到位 | 业务方/PMO | 样本提交 |
| A5 | 样本到位后执行《真实样本校验执行卡》 | 项目总控Agent | 校验结果记录 |
| A6 | 校验通过后执行《真实影子运行启动卡》 | 项目总控Agent | 影子运行产出 |

**进入阶段**：阶段4S（影子运行）

---

## 三、分支B：演示认可，但先修小问题 → 再影子运行

**进入条件（满足任一）**：
1. 决策项A（输出结构）= 有条件同意，有展示层/小口径修改要求
2. 决策项B（影子运行授权）= 有条件同意，有前置条件未满足
3. 反馈登记中有P2级展示层或业务口径问题（不阻断核心逻辑）

**最小动作集**：

| # | 动作 | 负责方 | 产出 |
|---|------|--------|------|
| B1 | 整理修改清单（30分钟） | 项目总控Agent | 明确展示修改范围 |
| B2 | 执行展示层/口径小修改 | 项目总控Agent | 修改产出HTML（1天） |
| B3 | 回归验证（1小时） | 项目总控Agent | 确认修改不影响判断逻辑 |
| B4 | 向演示对象反馈修改结果 | 项目总控Agent | 发出修改确认通知 |
| B5 | 收集二次确认 | 业务方/PMO | 口头/书面确认修改已接受 |
| B6 | 二次确认后进入分支A（影子运行） | 项目总控Agent | 推进A4-A6 |

**进入阶段**：完成B1-B4后进入分支A；进入分支A后进入阶段4S

---

## 四、分支C：先推进扩大评审

**进入条件（满足任一）**：
1. 决策项C（扩大评审）= 同意
2. 决策项A或B暂缓，需要先建立更多信心后再推进影子运行
3. 管理层要求评审路径优先

**最小动作集**：

| # | 动作 | 负责方 | 产出 |
|---|------|--------|------|
| C1 | 整理扩大评审材料包 | 项目总控Agent | 完整评审材料（PDF/文档集） |
| C2 | 发出评审邀请 | 项目总控Agent | 评审时间/议程确认 |
| C3 | 扩大评审会议 | 评审方 | 评审结论记录 |
| C4 | 评审结果处理 | 项目总控Agent | 根据评审结论分支： |
| | → 评审通过 | | 进入分支A（影子运行） |
| | → 评审有小问题 | | 进入分支B（小问题修复） |
| | → 评审有重大问题 | | 暂停，进入问题修订循环 |

**评审材料包内容**：
- 《阶段4R进展报告》
- 《当前阶段管理结论》
- 《当前阶段业务复核结论》
- 《当前阶段技术结论》
- 《项目整体路线图_现状到落地》
- 《演示总览页_可跳转.html》（附件）

---

## 五、分支汇总对照表

| 分支 | 进入条件 | 最短路径时间 | 核心阻塞项 | 下一步 |
|------|---------|------------|-----------|--------|
| 分支A | A同意+B同意/有条件 | 样本到位即可 | 真实样本未到位 | 阶段4S影子运行 |
| 分支B | A/B有条件同意 | 1-3天（修小问题） | 展示/口径小问题 | 修完后→分支A |
| 分支C | C同意/评审优先 | 评审周期 | 评审方时间 | 评审通过→分支A |

---

## 六、各分支约束声明

| 约束 | 分支A | 分支B | 分支C |
|------|-------|-------|-------|
| 不自行宣称影子运行已启动 | ✅ | ✅ | ✅ |
| 不虚构演示反馈结果 | ✅ | ✅ | ✅ |
| 不跳过样本校验直接进入运行 | ✅ | ✅ | ✅ |
| 不绕过人工审阅使用系统结论 | ✅ | ✅ | ✅ |

---

## 七、演示后立即检查（演示结束后执行）

```
□ 演示反馈已记录（《内部演示反馈登记表》）
□ 决策项A/B/C已填写（《内部演示决策记录模板》）
□ 根据决策结果确认分支（A/B/C）
□ 相关方已收到分支走向通知
□ 项目路线图已反映最新分支状态
```
