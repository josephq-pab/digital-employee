## 三、决策收口机制结论

**3项决策 + 3个分支**：

| 决策 | 选项 | 触发分支 |
|------|------|---------|
| A：输出结构认可 | 同意/有条件同意/暂缓 | 决定是否先修小问题 |
| B：影子运行授权 | 同意/有条件同意/暂缓 | 决定影子运行时间 |
| C：扩大评审推进 | 同意/有条件同意/暂缓 | 决定评审 vs 影子运行优先级 |

| 分支 | 触发条件 | 下一步 |
|------|---------|--------|
| 分支A | A同意+B同意/有条件 | 直接进入阶段4S（影子运行） |
| 分支B | A/B有条件，小问题可修复 | 先修展示/口径问题，再进入分支A |
| 分支C | C同意，评审优先 | 先做扩大评审，再进入分支A |
