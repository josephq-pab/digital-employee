## 五、样本与INPUT_SCHEMA的适配性分析

### 5.1 字段差异

| INPUT_SCHEMA字段 | 周报/台账对应字段 | 差异说明 |
|----------------|-----------------|---------|
| project_name | 台账：project_name | ✅ 直接对应 |
| owner | 台账：owner_department | ⚠️ 周报无owner，台账有部门 |
| promised_items | 台账：objective_or_requirement | ✅ 对应（台账objective字段） |
| current_progress | 台账：progress_by_period | ✅ 对应（含3期进展） |
| risk_points | 周报：key_alerts/management_summary | ⚠️ 需从多个字段提取 |
| escalation_requested | 无直接对应 | ❌ 需人工判断 |
| report_type | 周报：report_meta.report_scope | ✅ 隐含为周报 |
| period | 周报：report_period | ✅ 直接对应 |

### 5.2 适配结论

| 项 | 结论 |
|----|------|
| 直接可用字段 | project_name, promised_items, current_progress |
| 需提取字段 | risk_points（从key_alerts/management_summary提取） |
| 缺失字段 | escalation_requested（周报/台账均无此字段） |
| 需格式转换 | 周报JSON → INPUT_SCHEMA JSON（阶段4B需实现适配层） |

**适配性结论：基本可适配，但risk_points提取逻辑和escalation_requested判断需人工确认**
