## 二、改后入口与状态

### 2.1 主次入口设计

**改后数据接入页顶部结构（6E优化版）：**

```
┌────────────────────────────────────────────────────────────────┐
│  📦 主入口：上传本周样本包                      [主按钮]     │
│    将周报+台账+指标打包zip上传，系统自动切片运行              │
│    [📤 上传样本包(zip)]   [📋 查看已有结果]                 │
├────────────────────────────────────────────────────────────────┤
│  📝 次入口：补传缺失文件                      [次按钮]       │
│    单独补传缺失项，系统自动识别哪些已存在哪些需补传          │
│    [📤 补传缺失文件]                                          │
└────────────────────────────────────────────────────────────────┘
```

**为什么这样改：**
- 主入口突出，用户一上来就知道"上传样本包"
- 次入口处理"补充"场景，不抢占主路径
- 每个入口有清晰描述，用户不会困惑

---

### 2.2 样本完整性状态卡

**改后（6E优化版）：**

```
┌─────────────────────────────────────────────────┐
│  📊 本周样本完整性状态                           │
│                                                 │
│  [📄周报]     [📋台账]     [📈指标]           │
│  ✅ 已到位   ✅ 已到位   ✅ 已到位              │
│                                                 │
│  ╔═══════════════════════════════════════╗      │
│  ║  ✅ 三份齐套 — 可进入系统运行         ║      │
│  ╚═══════════════════════════════════════╝      │
└─────────────────────────────────────────────────┘
```

**为什么这样改：**
- 三项并列，一眼可见哪项到位、哪项缺失
- 总体状态结论行在最显眼位置
- 缺失时红色❌+联系人和路径直接显示

---

### 2.3 接入后下一步提示

**改后（6E优化版）：**

```
▶ 接入后下一步

  ① 接入完成
     系统检测到三份文件齐套，确认接入
  
  ② 系统运行（约2-4小时）
     自动切片 → 生成领导摘要/台账/风险清单
  
  ③ 业务复核（24小时内）
     PMO协调人对照原始周报评估准确性
  
  ④ 周五16:00前收口
     填写ABC决策，完成本周运行收口
```

**为什么这样改：**
- 四步时间线，用户知道什么时候该看什么
- 每步明确"谁做什么"
- 接入成功后不再让用户困惑

---

### 2.4 角色说明折叠化

**改后（6E优化版）：**

```
📋 查看各角色职责说明 ▾   ← 默认折叠，点击展开
```

展开后内容不变，但默认不显示，不抢占页面空间。

**为什么这样改：**
- 初次使用用户需要看，用人可展开
- 熟悉后不被打扰
- 不删除信息，但减少视觉噪音

---

## 三、改后整体页面节奏

**改后数据接入页顺序（从上到下）：**

1. **主入口区** — 一眼看到"上传样本包"按钮
2. **次入口区** — 补充场景，不抢占
3. **完整性状态卡** — 三项并列+结论行
4. **接入后下一步** — 时间线，4步清楚
5. **角色说明** — 可折叠
6. **样本列表** — 已有样本查阅
7. **样本详情** — 深度信息
