# MVP试跑结果判定规则

**版本：** v1.0
**日期：** 2026-04-08
**试跑性质：** MVP模拟试跑（dry-run / conditional）

---

## 一、判定档位

| 档位 | 含义 | 说明 |
|------|------|------|
| **A** | 可进入正式MVP试跑准备 | 原型已达到MVP试跑最低要求，可开始正式试跑准备 |
| **B** | 需小修后再进入 | 有可修复的P1问题，但不影响试跑方向，修复后可达A |
| **C** | 需显著修订，不建议进入 | 存在重大问题，需重新审视原型方向或规则设计 |

---

## 二、判定矩阵

### 2.1 P0决策规则（P0一票否决）

| P0数量 | 判定结果 |
|--------|---------|
| ≥1 | **C**（直接进入C，不继续评估P1/P2） |

### 2.2 P1决策规则

| P1数量 | P1全部可带问题继续？ | 判定结果 |
|--------|---------------------|---------|
| 0 | — | 参考P2，见2.3 |
| 1-2 | 是 | **A** |
| 1-2 | 否 | **B** |
| 3-5 | 是 | **B** |
| 3-5 | 否 | **C** |
| >5 | — | **C** |

### 2.3 无P0且P1≤2时的P2参考规则

| P2情况 | 判定结果 |
|--------|---------|
| P2 ≤ 3 | **A** |
| P2 > 3 | **B**（作为观察项，可带问题进入） |

---

## 三、判定输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| P0数量 | 《MVP试跑问题登记表》 | P0类问题数量 |
| P1数量 | 《MVP试跑问题登记表》 | P1类问题数量 |
| P1是否可带问题继续 | 逐条判断 | 每条P1是否标记"可带问题继续" |
| P2数量 | 《MVP试跑问题登记表》 | P2类问题数量 |
| 冒烟测试结果 | 试跑执行日志 | 是否≥5/7 |

---

## 四、判定流程

```
开始判定
    ↓
检查P0数量
    ↓
有P0 ≥ 1 → 判定C（直接结束）
    ↓
无P0
    ↓
检查P1数量
    ↓
P1 > 5 → 判定C
P1 3-5 → 检查是否可带问题继续 → B/C
P1 1-2 → 检查是否可带问题继续 → A/B
P1 = 0 → 参考P2 → A/B
    ↓
输出最终判定
```

---

## 五、各档位后续动作

| 档位 | 后续动作 |
|------|---------|
| **A** | 可进入正式MVP试跑准备；所有问题转入《试跑后修订机制》跟踪 |
| **B** | 修复P1问题后重新判定；修复完成后按A处理 |
| **C** | 不建议进入正式试跑准备；问题转《试跑问题转任务规则》处理；重新审视原型设计 |

---

## 六、判定记录要求

每次判定必须记录：
1. P0/P1/P2数量及明细
2. 每条P1的"是否可带问题继续"判断
3. 最终档位及依据
4. 判定时间
