## 二、验证顺序

**先验证什么，再验证什么：**

```
第一轮（冒烟测试）
↓
验证A1: 周报可加载 ✅
验证A2: 台账可加载 ✅
验证A3: 领导摘要可生成 ✅
验证A4: 事项总台账可生成 ✅
验证A5: 风险清单可生成 ✅
↓
全部PASS → 进入第二轮（输出质量验证）
有ERROR → 停止，排查数据格式问题
```

```
第二轮（输出质量验证）
↓
验证B1: 领导摘要内容与周报数据吻合
验证B2: 事项台账与周报事项一致
验证B3: 风险清单与周报风险项一致
验证B4: 结论卡判断与台账状态一致
↓
全部合理 → 进入第三轮（业务验证）
有异常 → 记录问题，判断是否阻断
```

```
第三轮（业务验证/B3最终确认）
↓
验证C1: 业务方查看领导摘要，判断内容是否准确
验证C2: 业务方查看结论卡，判断准确率是否可接受
验证C3: 业务方确认全套输出是否可用
↓
业务方认可 → B3最终确认通过
业务方不认可 → 记录具体问题，评估处理方案
```

## 三、B3最终准确率如何确认

**B3条件通过的最终确认：**

### B3最终确认标准

| # | 确认项 | 确认标准 | 确认人 |
|---|--------|---------|--------|
| 1 | 领导摘要内容准确率 | 业务方判断内容与实际周报吻合 | 业务方 |
| 2 | 结论卡判断准确率 | 业务方判断黄灯/绿灯/结项判断与实际一致 | 业务方 |
| 3 | 全套输出综合评价 | 业务方判断可用性是否可接受 | 业务方 |

### B3最终确认流程

```
真实数据接入
    ↓
第一轮：技术冒烟（7/7 PASS）
    ↓
第二轮：输出质量自检（项目方自检）
    ↓ 有问题 → 记录 → 评估
    ↓ 没问题
第三轮：业务方验证（业务方确认B3）
    ↓
B3最终确认通过 / 不通过
```

### B3确认的证据要求

**B3最终确认的证据形式（优先级从高到低）：**

| 优先级 | 证据形式 | 说明 |
|--------|---------|------|
| 1 | 书面确认邮件 | 业务方书面认可 |
| 2 | 会议纪要确认 | 有记录的会议确认 |
| 3 | 消息截图+文字确认 | 飞书/微信等消息确认 |
| 4 | 口头有记录 | 有录音或文字记录的口头确认 |

## 四、哪些输出要重点对照业务判断

**真实数据验证时，重点检查以下输出：**

### 4.1 领导摘要

| 检查项 | 判断标准 |
|--------|---------|
| 周报期数是否正确 | 加载了几期周报 → 生成几期摘要 |
| 项目概述是否准确 | 业务方判断"项目数量/事项数量"是否与实际吻合 |
| 文风是否可接受 | 与业务方日常报告风格是否一致 |

### 4.2 事项总台账

| 检查项 | 判断标准 |
|--------|---------|
| 事项数量是否正确 | 台账事项数与周报事项数是否对应 |
| 项目名称是否匹配 | 台账项目名与周报提及项目是否对应 |
| 状态灯是否正确 | 黄灯/绿灯/结项判断是否与台账一致 |

### 4.3 风险清单

| 检查项 | 判断标准 |
|--------|---------|
| 风险识别是否准确 | 是否有遗漏的风险项 |
| 风险分级是否合理 | red/yellow/defer 分级是否与业务判断一致 |
| 风险描述是否准确 | 风险内容描述是否与实际情况吻合 |

### 4.4 结论卡（核心验证项）

**结论卡是B3最终确认的核心：**

| 检查项 | 判断标准 |
|--------|---------|
| 黄灯判断是否准确 | 被判为黄灯的项目，是否确实存在风险 |
| 结项判断是否准确 | 被判为结项的项目，是否确实满足结项条件 |
| 延期判断是否准确 | 被判为延期的项目，是否确实存在延期 |
| 各项判断是否有依据 | 每项判断是否在周报/台账中有对应证据 |

## 六、验证问题处理

### 6.1 常见问题及处理

| # | 问题 | 原因 | 处理方式 |
|---|------|------|---------|
| 1 | 7/7 PASS 但输出为空 | 数据JSON格式不匹配 | 检查JSON字段名 |
| 2 | 事项数量不匹配 | 周报与台账数据期数不一致 | 确认数据同步 |
| 3 | 项目名称匹配率低 | 项目命名不统一 | 优化fuzzy匹配逻辑 |
| 4 | 状态灯判断错误 | 台账status字段值不标准 | 标准化status值 |

### 6.2 问题严重度判断

| # | 严重度 | 定义 | 处理 |
|---|--------|------|------|
| A | 阻断 | 数据完全无法加载或输出全错 | 停止，修复后再验证 |
| B | 严重 | 输出大部分错误 | 记录，评估是否修复 |
| C | 轻微 | 输出有小问题 | 记录，评估是否修复 |
| D | 可忽略 | 不影响业务使用 | 记录，可接受 |
