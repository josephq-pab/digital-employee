# 演示反馈转MVP任务规则

**版本：** v1.0
**日期：** 2026-04-07
**用途：** 说明演示反馈如何转化为MVP任务，明确哪些进入P0/P1/P2

---

## 一、转化原则

**原则1：不是所有反馈都变成任务**
演示中的反馈只有一部分值得转化为MVP任务。
核心判断：是否影响"数字员工能正确判断"。

**原则2：规则问题优先确认，暂不修改**
演示中涉及Q01-Q05的规则反馈，
在确认前只能记录，不能擅自修改provisional规则。

**原则3：展示问题立即处理**
展示层问题不影响判断逻辑，可以立即处理，不进入MVP任务池。

---

## 二、反馈→任务分级

### P0：阻断MVP试跑

**定义：** 收到此类反馈后，在问题解决前不能进入MVP试跑。

**触发条件：**
1. 演示中管理者明确表示"这个判断我不能接受"
2. 且该判断涉及Q01-Q05中尚未确认的规则
3. 且无法用当前provisional规则合理解释

**典型场景：**
- "黄灯不应该只是中风险，应该升级"（涉及PMO-02核心规则）
- "这个输出格式我完全无法使用"（涉及MGT-01）
- "行业和项目的映射关系搞错了"（涉及MAP-01~04）

**转化流程：**
```
P0反馈 → 记录到规则问题待确认清单.md
       → 标注"P0-阻断MVP"
       → 触发Q01-Q05确认包重新发出
       → 门控G1/G2/G3更新为🔴
       → 等确认后才能进入MVP
```

**示例：**
```
反馈："黄灯应该直接升级，不是关注"
分析：涉及PMO-02核心规则（黄灯→中风险→关注）
判断：P0-阻断MVP
动作：记录→触发PMO确认→等确认→更新规则→再评估MVP
```

---

### P1：建议在MVP前处理

**定义：** 不阻断MVP试跑，但建议在试点启动前处理，否则影响试点质量。

**触发条件：**
1. 演示中有具体反馈
2. 不涉及Q01-Q05确认问题
3. 但影响数字员工判断准确性或覆盖率

**典型场景：**
- "汽车行业的映射不对，应该对应ledger-10"（涉及QUAL-03）
- "这个owner字段完全是错的"（涉及QUAL-05）
- "同样的事项出现在三期周报里，应该去重"（涉及QUAL-04）

**转化流程：**
```
P1反馈 → 记录到QUAL清单.md
       → 标注"P1-建议MVP前处理"
       → 在MVP试点第一期前完成修复
       → 更新QUAL状态
```

**示例：**
```
反馈："汽车行业应该映射到ledger-10"
分析：涉及行业→项目映射（QUAL-03）
判断：P1-建议MVP前处理
动作：记录→触发MAP确认→在试点前更新映射
```

---

### P2：可在MVP试跑中观察

**定义：** 不是错误，但是可以改进的点。在MVP试跑中观察是否持续出现，再决定是否处理。

**触发条件：**
1. 演示中有反馈
2. 不影响数字员工核心判断能力
3. 是优化项而非错误

**典型场景：**
- "摘要里的颜色不好看"（展示问题）
- "我想看更多项目维度的汇总"（需求扩展）
- "台账字段能不能自定义顺序"（需求扩展）

**转化流程：**
```
P2反馈 → 记录到需求池.md
       → 标注"P2-MVP中观察"
       → 在MVP试跑第一期后评估是否处理
```

## 三、不转化为任务的反馈

以下反馈只记录，不转化为MVP任务：

| 反馈类型 | 处理方式 |
|---------|---------|
| 需求扩展（一期不做） | 进入需求池 |
| 涉及未来阶段 | 进入需求池 |
| 与当前判断逻辑无关 | 记录但不跟进 |

---

## 四、任务转化检查清单

每次演示后，对每个反馈执行以下检查：

```
1. 这个反馈是否影响"数字员工能正确判断"？
   ├── 是 → 进入P0或P1
   └── 否 → 进入需求池或忽略

2. 这个反馈是否涉及Q01-Q05中尚未确认的规则？
   ├── 是 → P0（阻断MVP）
   └── 否 → 进入下一步

3. 这个反馈是否影响试点第一期的质量？
   ├── 是 → P1（建议MVP前处理）
   └── 否 → P2（可观察）

4. 这个反馈是否是一期明确不做的？
   ├── 是 → 记录到需求池，告知用户"一期不做"
   └── 否 → 进入对应分级
```

## 五、MVP任务池格式

当反馈转化为任务后，写入以下格式：

```markdown
## MVP任务池

### P0（阻断MVP试跑）
| # | 任务 | 来源反馈 | 依赖确认 | 状态 |
|---|------|---------|---------|------|
| 1 | （任务描述） | （反馈摘要） | Q01/Q03/Q04 | 待确认 |

### P1（建议MVP前处理）
| # | 任务 | 来源反馈 | 依赖 | 状态 |
|---|------|---------|------|------|
| 1 | （任务描述） | （反馈摘要） | MAP确认 | 待处理 |

### P2（MVP中观察）
| # | 任务 | 来源反馈 | 观察期 | 状态 |
|---|------|---------|-------|------|
| 1 | （任务描述） | （反馈摘要） | 试点第一期 | 观察中 |
```

| 任务级别 | 完成标志 |
|---------|---------|
| P0 | Q01-Q05相关门控变为🟢 |
| P1 | QUAL清单中该条标记为"已完成" |
| P2 | 试点第一期结束后评估是否关闭 |
