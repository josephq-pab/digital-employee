# MVP试跑评估标准

**版本：** v1.0
**日期：** 2026-04-08
**试跑性质：** MVP模拟试跑（dry-run / conditional）

---

## 一、评估维度

| # | 维度 | 评估内容 |
|---|------|---------|
| 1 | 运行稳定性 | 脚本是否稳定运行，无崩溃/报错 |
| 2 | 输出完整性 | 三件套（领导摘要/事项台账/风险清单）是否全部生成 |
| 3 | 数据准确性 | 输出内容与原始样本的匹配程度 |
| 4 | 规则正确性 | 风险判断是否合理，是否有误判/漏判 |
| 5 | 映射覆盖率 | 周报与台账的关联程度 |
| 6 | 边界合规性 | provisional标记是否完整，不得对外声称正式 |

---

## 二、严重度定义

### P0：阻断

**定义：** 试跑无法继续，输出结果完全不可用，或核心功能完全失效。

**判定标准：**
- 脚本运行报错，无法生成任何输出
- 冒烟测试 <5/7
- 三件套中有任何一件完全缺失
- 归一化事项数为0
- 存在严重的规则误判（如将申请结项判为高风险升级）

**试跑动作：** 立即停止，记录问题，转《试跑问题转任务规则》

---

### P1：重要但可继续

**定义：** 输出存在问题，但不影响全流程验证，可在问题已知的情况下继续推进正式试跑准备。

**判定标准：**
- 脚本可运行，但部分输出存在明显问题
- 冒烟测试 5-6/7
- 映射覆盖率低于预期但仍有可用输出
- 部分规则判断可能不准确，但有provisional声明
- 领导摘要/事项台账/风险清单均可生成，内容部分可用

**试跑动作：** 记录问题，不阻断试跑，可带问题进入正式试跑准备

---

### P2：可观察项

**定义：** 问题存在但不影响核心功能，作为观察项在MVP期间跟踪。

**判定标准：**
- 输出格式小瑕疵（如某字段截断）
- 映射覆盖率偏低但当前样本足够验证
- 跨期去重未实现，但当前样本期间不涉及
- 某些业务术语未与Q01对齐，但不影响判断逻辑

**试跑动作：** 记录问题，作为MVP期间优化项跟踪

---

## 三、试跑可接受条件

### 条件1：运行稳定

| 检查项 | 通过标准 |
|--------|---------|
| 脚本执行 | 无Python报错 |
| 冒烟测试 | ≥5/7 PASS |
| 三件套生成 | 全部生成 |

### 条件2：输出可用

| 检查项 | 通过标准 |
|--------|---------|
| 领导摘要 | 包含期间、总事项数、完成项、黄灯项 |
| 事项台账 | 事项数≥10，每条含来源/期间/责任人/状态 |
| 风险清单 | 风险数≥1，每条含规则依据 |

### 条件3：问题可接受

| 严重度 | 可接受数量 |
|--------|----------|
| P0 | 0 |
| P1 | ≤5（且无任何P1涉及provisional规则被删除） |
| P2 | 不限 |

---

## 四、可带问题进入正式试跑准备的条件

满足以下全部条件时，可带问题进入正式MVP试跑准备：

1. 无P0问题
2. P1问题 ≤5
3. 所有P1问题均已登记在《MVP试跑问题登记表》
4. 所有provisional边界声明完整，无缺失
5. Q01/Q03/Q04仍在推进中，未被无限期搁置

---

## 五、必须先修复后再试跑的条件

满足以下任意条件时，必须修复后再试跑：

1. 存在任何P0问题
2. 冒烟测试 <5/7
3. 三件套中有任何一件完全缺失
4. 输出内容为空或乱码
5. 脚本运行报错
