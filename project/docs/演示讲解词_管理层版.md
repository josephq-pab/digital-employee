## 一、整体定位

管理层最关心的是：这个工具对我有什么用？我看它输出能比现在更省力吗？它能提前发现问题吗？

---

## 二、讲解词

### 开场：数字员工能帮你做什么

```
"简单说：数字员工就是帮你读周报、做摘要、标重点的AI助手。

你每周收到PMO的周报，里面有几十项事项，
现在你需要花时间去识别哪些有风险、哪些需要关注。

数字员工做的，就是把这部分重复劳动自动化：
给你一份结构化的摘要，直接告诉你——
哪些事项有风险、风险等级是什么、建议怎么处理。
你拿到手的是判断结果，不需要自己再做第一遍过滤。"
```

### 核心价值：透明度

```
"数字员工输出有一个关键特点：每项判断都附带依据。

比如，系统判断"汽车行业有中风险"，
依据是：进度滞后 + 责任人自述风险。

这意味着什么？你拿到这个判断后，
如果觉得不对，可以追问——
"为什么说它滞后？依据在哪里？"

答案系统直接给你：原始文本里有"进度滞后"这四个字。

我们追求的是"AI判断可解释、可追责"，
不是给一个黑箱让你自己猜。"
```

### 核心价值：提前识别

```
"现在的周报体系，问题是风险往往在汇报中才被发现，
而不是在风险刚出现时就识别。

数字员工设计的初衷，就是把"事后发现"变成"提前识别"。
周报里有黄灯，系统默认认为这是"关注事项"，
而不是等到有人写了长篇报告才来处理。

这不是说AI比人聪明，
而是AI可以在海量的日常汇报中持续盯着，不会疲劳、不会遗漏。"
```

### 闭环能力

```
"我们设计了一个"闭环确认"机制：

第一步：系统识别出风险事项
第二步：管理者收到摘要后，可以确认、修改或推翻
第三步：确认结果进入下一期报告，形成跟踪

也就是说，这不是一个单向输出的系统，
而是一个持续跟踪、有人参与的闭环。
你的意见会沉淀下来，不会每期都重新来。"
```

### 当前阶段的诚实边界

```
"有一点必须说明清楚：

目前数字员工还是"条件性实现"阶段——
也就是说，部分规则还在等PMO确认，不算正式版本。

具体来说：术语口径、判断规则、输出格式，
都在等确认。我们不做"拿着半成品当正式版"这种事。

什么时候算正式？
等PMO确认完术语和管理层确认完格式，
那时候才进入试点，试点通过后才算MVP。

这是我们的节奏，不快也不拖。"
```

## 二、管理层常见问题应对

| 问题 | 应对 |
|------|------|
| 这个系统会不会替代PMO？ | 不会。AI做的是辅助判断，最终决策在管理者。PMO的判断经验反而更重要。 |
| 数据从哪来？ | 周报和项目台账，人工填报，不接生产系统，安全可控。 |
| 判断错了怎么办？ | 每项判断都有依据，管理者可以推翻。系统会记录你的修改，作为反馈。 |
| 什么时候能用？ | 等PMO和管理层确认完规则后，进入1-2个真实项目试点。 |
| 比你现在看周报好在哪儿？ | 1.结构化，不用自己找重点；2.有依据，可以追问；3.持续跟踪，不遗漏 |
