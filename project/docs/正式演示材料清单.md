## 六、快速定位指南（新Agent接手）

### 第一步：理解演示全貌
```
正式演示执行包总览.md → 演示目标/输入/输出/边界
```

### 第二步：理解本版限制
```
正式演示版本说明.md → 修复了什么/还有什么限制
```

### 第三步：准备演示
```
正式演示运行清单.md → 演示前16项检查
正式演示彩排记录.md → 内部彩排结果
```

### 第四步：执行演示
```
5-8分钟演示脚本.md → 讲解流程
演示讲解词_对内版.md → 对内技术讲解
演示讲解词_管理层版.md → 对管理层价值讲解
```

### 第五步：处理反馈
```
演示常见问答.md → 常见问题应对
演示版验收记录表.md → 记录反馈
演示反馈归并规则.md → 分类反馈
```

### 第六步：后续跟进
```
演示反馈收口机制.md → 反馈→行动
演示后分支处理说明.md → 进入分支A还是B
```

## 八、材料版本规则

每次正式演示前，必须：
1. 执行 `python project/scripts/run_prototype.py` 生成新时间戳输出
2. 更新本清单中的文件时间戳
3. 在彩排记录中确认文件最新
4. 在版本说明中记录本次演示版本号
