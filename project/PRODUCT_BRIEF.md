# PRODUCT_BRIEF.md

## 原型目标

构建**重点项目执行跟踪数字员工**的最小可验证原型（MVP），实现：
> 给定一个项目的结构化输入（责任人、承诺事项、进展、风险点），自动产出标准化的执行状态摘要、风险判断、建议动作和升级建议。

## 目标用户

- 项目管理部门（PMO）
- 项目负责人（自查汇报）
- 管理层（风险升级审阅）

## 本轮只解决的问题

1. 单项目输入 → 结构化输出的闭环推理链路
2. 输出严格区分"事实 / 判断 / 建议"，可人工复核
3. 最小可运行的命令行演示

## 暂不解决的问题

- 多项目汇总报告
- 前端界面或 API 服务
- 历史项目数据持久化
- 自动通知或升级推送
- 多轮对话或澄清输入
- LLM 调用（本轮基于规则推理）

## 核心假设

- 输入为结构化 JSON（或等效字典），字段含义清晰
- 风险判断基于预定义规则，非大模型生成
- 输出格式固定，便于程序消费和人工阅读
